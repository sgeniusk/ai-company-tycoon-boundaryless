# feat-025 이벤트·고용 픽셀 컷씬 — 설계

브레인스토밍 2026-06-22(Claude+사용자). 카이로소프트 Game Dev Story식으로 이벤트·고용 등 주요 순간에 픽셀 컷씬/애니메이션을 연출한다. 현 컷씬(feat-017 CutsceneDirector)은 출시·승급·상장에만 있고 이벤트·고용엔 없다.

## 목표
이벤트 발생·결과, 직원 고용 순간에 픽셀 캐릭터가 연기하는 컷씬을 추가해 "보는 맛"을 확장한다. 새 픽셀 애니 프레임으로 생동감을 살린다(사용자 결정).

## 사용자 결정 (브레인스토밍 합의)
- **규모** — C 혼합. 일반(이벤트 결과·일반 고용) = 작은 코너 컷인, 중요(위기 이벤트·특별 인재·첫 직원) = 전체 모달.
- **이벤트 컷인 시점** — C 둘 다. 발생 시 상황 연출(★B) + 선택 후 결과 반응(★A).
- **애니 방식** — 새 픽셀 애니 프레임(imagegen). 기존 포즈+코드 모션이 아니라 전용 프레임 루프.
- **멀티엔딩** — 이 트랙 뒤로 보류(별도).

## 범위·분해 (3단계, 큰 작업이라 분해)
| 단계 | 내용 | 연출 |
|---|---|---|
| **1 (MVP)** | 이벤트 **결과** 컷인 + 환호/낙담 애니 (캐릭터 3종 × 2반응) | 코너 컷인 |
| **2** | 고용 **등장** 컷인 + 이벤트 **발생** 당황 애니 | 코너 컷인 |
| **3** | 위기 이벤트·특별 인재·첫 직원 | 전체 모달 |

1단계부터 구현·검증 후 2·3으로 확장한다. 1단계를 이벤트 결과로 잡은 이유는 이벤트가 가장 자주 발생해 체감이 크기 때문이다.

## 아키텍처 (1단계 MVP)
코어(Core/Data/Systems/Save 시뮬) 무변경. UI/연출 레이어만 확장한다.

### 1. 트리거
`GameScreen.ShowEventModal`의 선택 `onClick` 핸들러(현 "이벤트 선택을 적용했습니다" 지점)에서 선택한 choice의 효과 합(자원 +/-)으로 좋음/나쁨을 판정한다. 판정 후 `CutsceneDirector.PlayEventResult(isPositive)`를 호출한다.
- 판정 기준 — choice의 효과(자원 델타) 합이 양수면 환호, 음수면 낙담. 0이거나 모호하면 중립(컷인 생략 또는 가벼운 반응).

### 2. 연출
기존 `CutsceneDirector.EnqueueMini`(능력업 'LEVEL UP!' 코너 미니 윈도우) 패턴을 확장한다. 새 컷인 종류 EventGood/EventBad를 추가한다.
- 환호 = 돈다발·반짝 + 말풍선 "대박!"
- 낙담 = 땀방울·먹구름 + 말풍선 "이런..."
- 형태 — 화면 하단/코너의 작은 컷인(A안). 오피스를 가리지 않고 흐름을 끊지 않는다. 탭 없이 자동 사라짐.

### 3. 애니
환호/낙담 각 2~3 프레임 루프. `ActorAnim`(오피스 직원, 포즈당 1스프라이트)은 그대로 두고, **컷씬 전용 프레임 애니 컴포넌트**(가칭 `CutsceneFrameAnim`)를 신설해 프레임 배열을 루프 재생한다.

### 4. 애니 파이프라인 (feat-024 크로마키 재사용)
codex exec imagegen으로 마젠타 배경 프레임 시트 생성(캐릭터 3종 × 환호/낙담 × 2~3프레임) → `Tools~/pixel_office/feat024/key_chroma.py` 키잉 → `diagnose_halo.py` halo=0 게이트 → Resources 반입(.meta GUID 보존). 프롬프트는 stdin 파이프(`echo ... | codex exec -i ref.png`).

## 비-목표 (이 단계)
- 멀티엔딩 연출 — 다음 트랙
- 2·3단계(고용·발생·모달) — 후속
- 코어 시뮬 변경 — 없음

## 검증 게이트
- 새 스프라이트 halo=0 (`diagnose_halo.py`)
- EditMode 145/145 (코어 무변경)
- PlayMode 캡처 — 이벤트 결과 컷인(환호/낙담) 연출 육안
- 단일 라이선스 확인(ps) 후 Unity 검증

## 열린 질문 (구현 계획에서 확정)
- 애니 프레임 수(2 vs 3)
- 환호/낙담 캐릭터 — 오피스 대표 1명 vs 직원 전체 반응
- GameEvents 시그널 신설 vs CutsceneDirector 직접 호출
