# feat-031 성장 아크 완성 — 설계 (정본)

타이쿤의 "작게 시작 → 또렷하게 성장" 경험을 코드·데이터·연출로 완성하는 피처다. 2026-06-30 사용자 피드백 3항목(1인 창고 시작·오피스 업그레이드 조건 해금·버튼 affordance)을 하나의 성장 아크로 통합한다. 사용자 방향 — "성장 아크 완성", "진짜 1인 시작 + 첫 영입 목표".

## 배경 — 왜 지금 휑함·풀오피스 문제가 나는가
- 시작 인재 = 3 (`../data/resources.json` talent.initial_value), 오피스 레벨 = 1(`garage_lab`, 정원 3).
- 즉 **시작부터 차고가 3/3 만석**이라 영입 여유가 없고, 늘리려면 곧장 확장부터 해야 하는 거꾸로 된 흐름이다.
- 오피스 씬은 `talent` 수만큼(0~10 클램프) 직원을 그린다(`GameScreen.RefreshOfficeScene`). 3인 + 가구 가득 = "풀 오피스" 인상.
- 확장 사다리는 이미 `unlock_requirements`로 게이팅돼 있다(레벨2=제품1, 레벨3=6개월+제품2, …). `OfficeService.GetExpandLockReason()`이 `"요건 미충족"/"자금 부족"/null`을 돌려준다. 문제는 게이팅 로직이 아니라 **시안 시연용 노출**이 항상 가용처럼 보이는 것이다.

## 목표 / 비목표
- 목표 — 1인 창고에서 시작해 첫 영입·오피스 해금·구성 채움으로 성장이 눈에 보이게 한다. 버튼이 누를 수 있게 보이게 한다.
- 비목표 — 새 데이터 콘텐츠(이벤트·제품·엔딩) 대량 추가는 이번 스코프가 아니다(별도 트랙). 디자인 재작업(feat-030 시안)은 유지한다.

## 블록 구성

### ① 진짜 1인 창고 시작 + 첫 영입 목표
- `../data/resources.json` 시작 인재 **3 → 1**(사장 1명). 오피스 레벨 1·차고(`rural_garage`) 유지. `DataImporter.ImportAll`로 SO 재시드.
- 효과 — 차고 1/3(영입 여유 2), 급여 1800→600/월(초반 무수익 구간 생존 완화), 시작 현금 10000으로 첫 영입(`recruit_base_cost` 2500) 충분.
- 첫 목표 연출 — `GuidanceService` 첫 제안을 "첫 직원을 영입해 팀을 꾸리세요"로. 첫 영입 시 기존 스폰 셀레브레이션(`SpawnCelebration`) 재사용으로 마일스톤 강조.
- 프리랜서 비용 — `FreelanceBaselineTalent`=3이라 talent 1~2 구간은 baseline 2500 평탄(추가 변경 불필요).

### ② 오피스 게이팅 = 잠긴 목표 카드 (시연 노출 분리)
- 다음 확장이 요건 미충족이면 **활성 구매 버튼이 아니라 "잠긴 목표 카드"**로 렌더한다 — 이름 + 해금 조건 진척("제품 1개 출시 시 해금") + 자물쇠. 요건 충족·자금 부족이면 "자금 부족", 둘 다 OK면 활성 CTA.
- 상태는 신설하지 않고 `OfficeService.GetExpandLockReason()`(요건 미충족/자금 부족/null) 3상태를 그대로 구동한다. 조건 진척 문구는 `ThresholdEval` 요건을 사람이 읽는 문장으로 포맷(헤드리스 헬퍼, EditMode 가능).
- 본사 이전·꾸미기 진입점도 같은 잠김/자금부족/가능 3상태로 통일. HUD 상단 "성급/꾸미기" 칩이 항상 눌릴 듯 보이지 않게 실제 가용성을 반영.
- 잠긴 다음 단계는 **보여주되 못 누르게** 해 성장 동기를 살린다(완전 숨기지 않음).

### ③ 성장이 눈에 보이는 오피스 구성
- 소품 밀도를 오피스 레벨/인재 수에 연동한다. 1인 차고는 의도적으로 휑하게(핵심 소품만), 영입·확장할수록 가구가 채워진다. `OfficeProps`에 스케일 티어(예 — talent/레벨 → 소품 개수 상한)를 추가한다.
- 결과 — 빈 바닥이 "아직 시작 단계"라는 서사로 읽히고, 성장할 때마다 화면이 차오르는 체감이 생긴다. 새 스프라이트 0(기존 소품 배치 밀도만 조정).

### ④ 버튼 affordance 보강
- 중립/secondary 알약에 눌림 단서 — 미세 그림자 또는 1px 잉크 테두리 + press 상태(살짝 눌림 + 어두워짐). `UiFactory`/`UiTheme`에 `StyleSecondaryButton` 패스 추가(CTA용 `StyleCtaButton`은 이미 있음).
- 평면처럼 보이던 secondary 톤을 입체화해 "누를 수 있는 버튼"으로 읽히게 한다.

## 검증
- `./init.sh` EditMode 베이스라인 **177/177** 유지. 시작 인재 3을 기대하는 테스트가 있으면 1로 갱신.
- `TechTreeReachabilityTests` 그린 — 봇 완주 창(승리 ≥ tier4 해금·≤ 90개월) 유지 확인. 1인 시작으로 창이 깨지면 시작 현금/첫 영입가로 미세 조정 후 재검증.
- PlayMode 캡처 — 신규 시작(1인 차고)·첫 영입 후·잠긴 오피스 카드·버튼 press 상태. 단일 Unity 라이선스(다른 프로젝트 미실행 `ps` 확인) 후 일괄.

## 영향 범위 / 리스크
- 코어·세이브 — ①의 시작값(데이터)·가이던스 문구 외 코어 로직 무변경 지향. talent 시작값은 데이터(resources.json) + SO 재시드라 세이브 모델 무변경.
- 밸런스 — 1인 시작은 초반 생산을 늦추지만 급여도 줄여 순효과가 양쪽이다. 완주 가능성은 반드시 테스트로 확인(가정 금지).
- 분업 — 새 에셋 양산 없음(③은 기존 소품 재배치). 전부 Claude 직접 구현 가능 범위.

## 블록 순서
①(시작값+밸런스 검증) → ②(게이팅 노출) → ③(구성 밀도) → ④(affordance). 각 블록 EditMode + 해당 PlayMode 캡처로 검증하고 progress.md·feature_list.json에 증거를 남긴다.
