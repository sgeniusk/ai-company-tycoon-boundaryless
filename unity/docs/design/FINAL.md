# 최종안 — 메인 화면 디자인 결정 기록 (Unity 구현 스펙)

클로드 디자인 시안 3종을 검토해 방향을 확정하고, 인터랙티브 최종 HTML로 구현한 결과다. 다음 세션의 Unity 재설계는 이 문서와 `ai-tycoon-final.html`을 정본으로 삼는다.

## 시안 검토 → 선택
3종은 이미 `탐색 → 합성 → 러프 프로토타입` 순서였다.
- `ai-tycoon-uiux-sian.html` — 진단(현 화면 5문제) + 메인 4종(A 슬림HUD / B 풀블리드 / C 사이드레일 / D 대시보드) + 센터버튼 3상태 + 온보딩 + 4박자 + 컬러/타이포 시스템.
- `ai-tycoon-main-recommended.html` — 합성·추천. **"A의 정리된 뼈대 + D의 추월 트렌드 + B의 풀블리드 오피스"**.
- `ai-tycoon-prototype.html` — 추천안의 러프 인터랙션(1월차 차고 메인).

**선택 — 추천안(A+D+B)**. 보드 자체의 결론이자 방치형 도파민("내 회사가 알아서 자란다")에 가장 맞는다. 프로토타입에 빠진 4박자 루프·보텀시트·스마트 센터버튼·컬러/타이포를 모두 담아 `ai-tycoon-final.html`로 완성했다.

## 현 Unity 화면 대비 — 무엇이 바뀌나
대조 캡처 — 현재 `Logs/shots/01-main.png`, `38-briefing-great.png` / 최종안 `shots/final-*.png`.

| 영역 | 현재 (버림) | 최종안 (대체) |
|---|---|---|
| 상단 HUD | 다크 그린 바 + 큰 랭킹 전광판(화면 25%) | **슬림 1줄 HUD**(월차·성급·꾸미기) |
| 자원 | 정체불명 칩(0★·$10K·0·100) | **자원별 색 칩 + ▲델타**(현금 코랄·이용자 민트·연산력 보라) |
| 랭킹 | 큰 전광판 블록 | **슬림 1줄 추월 트렌드 바**(#935 ▲ 추월 중 · 추월까지 N계단) |
| 오피스 | 위아래 UI에 눌림 | **풀블리드 — 화면의 주인공**. 떠오르는 숫자·추월 토스트가 여기서 터짐 |
| 하단 상태줄 | "아직 월을 넘기지 않았습니다" 중복 | 제거 → AI 비서 리본 한 줄 |
| 센터 | 평범한 출시 버튼 | **스마트 센터버튼**(상황별 출시/수령/다음 달 + 펄스·색) |
| 모달 | 풀스크린 | **보텀시트**(오피스 위로 올라옴, 맥락 유지) |

유지/강화 — 카이로소프트식 픽셀 오피스, 제품 테크트리, 월말 결산 항목(매출/인건비·연산/순익/랭킹), AI 비서, 성급 승급.

## 핵심 인터랙션 — 도파민 4박자 (다음 달 ▸ 누르면)
1. **타임랩스** — Day N/30 카운터 + 낮→밤 하늘 페이드. **2x · 스킵 필수**(반복되면 연출이 형벌).
2. **중간 이벤트 일시정지** — 'AI 시대성'(규제·정렬 등) 선택지. 결과를 오피스·평판에 반영.
3. **손익 브리핑 시트** — 큰 순이익 숫자 **틱업** + 매출/비용/랭킹 추월 + "직원들이 환호합니다". 배경에 오피스.
4. **성급 승급 컷인**(임계 돌파 시만) — 차고 → 데이터센터 ★★★. **오피스 비주얼이 실제로 변하는 것이 이 게임의 차별점**. 회전 광선 + 글로우 연출.

## 스마트 센터버튼 — 3상태
- ① **추천 액션 있음** — 코랄 + 펄스(예: 제품 출시 ▸).
- ② **즉시 보상 대기** — 골드(예: 투자 유치 보상 수령 ▸).
- ③ **할 일 없음** — 채도 낮춘 아웃라인(다음 달 ▸). 방치형의 기본 박자.

## 컬러 · 타이포
- 팔레트 — 크림 베이스(#F5ECDD) + 잉크(#2C2620). 자원 고정색 — 현금/CTA 코랄(#EE6C3D), 이용자 민트(#2FA877), 연산력 보라(#7A66D6), 보상/랭킹 골드(#E2A02E). **한 화면 강조색은 하나(=센터버튼)**. 다크 그린 HUD는 폐기.
- 타이포 — Black Han Sans(큰 숫자·성급·결산), Gowun Dodum(본문·라벨), Jua(버튼·게임 액션).

## 오피스 3레이어 깊이 — 시안과 합류
시안은 오피스를 **풀블리드 주인공**으로 키운다. 진행 중이던 고정 3레이어(뒤 가구·러그 / 직원·책상 / 앞 전경이 직원 하반신 가림) 방향과 충돌 없음 — 오히려 큰 무대에서 깊이가 더 산다. `PropSpec.Layer`(Back/Front) 일급 속성 + 미래 에셋 레이어 규칙은 그대로 진행하되, 떠오르는 숫자·토스트가 터질 **전경 여백**을 오피스 상단에 확보한다(최종안 `.floaters` 영역 참고).
