# 클로드 디자인 시안 수령 — 디자인 주도 UI/UX 재설계

클로드 디자인(claude.ai/design)에서 받은 UI/UX 시안을 구현하기 위한 폴더다. DesignSync(claude_design) MCP가 데스크톱 앱에서 인증 불가([[mcp 한계 메모]] 참조)라, 시안 HTML을 직접 받아 구현한다.

## 대상 시안 (클로드 디자인 프로젝트 7b982813-974f-4480-bce4-1f73b9427086)
3개 스탠드얼론 HTML — 사용자가 이 폴더에 붙여넣거나 내용 전달.
- `ai-tycoon-uiux-sian.html` — "AI 타이쿤 UIUX 시안" (전반 UI/UX 방향)
- `ai-tycoon-main-recommended.html` — "AI 타이쿤 메인 (추천안)" (메인 화면 추천 구성)
- `ai-tycoon-prototype.html` — "AI 타이쿤 프로토타입" (인터랙션 프로토타입)

## 새 세션 작업 흐름
1. 위 3개 HTML을 읽는다(이 폴더에 있거나 사용자가 붙여넣음).
2. 시안이 제안하는 화면 구성(HUD·도크·오피스·모달·인터랙션)을 파악.
3. 현재 Unity 화면과 매핑 — 무엇이 바뀌고 무엇이 유지되나. 캡처 `Logs/shots/01-main.png`(현 메인)·`38/39-briefing`(브리핑)과 대조.
4. superpowers:brainstorming → writing-plans → subagent-driven 으로 구현.

## 진행 중이던 직결 설계 — 오피스 3레이어 깊이 (미작성 스펙)
디자인 임포트 직전 브레인스토밍에서 합의된 것 (스펙 문서 미작성, 이 시안과 합쳐서 진행):
- 현재 오피스는 **2레이어**(소품=직원 뒤 / 직원·책상=앞) — 책상이 러그 위엔 앉지만 앞 소품이 직원을 못 가림.
- 결정 — **고정 3레이어**(뒤 가구·러그 / 직원·책상 / 앞 전경 소품이 직원 하반신 가림).
- `PropSpec`에 **`Layer` 일급 속성**(Back/Front) 추가. `OfficeProps.Populate`가 뒤/앞 레이어 둘 생성(앞=스테이지 sibling+1).
- 배치 — 뒤: 러그·벽 가구. 앞: 전경 큰 화분·소파(직원과 겹쳐 깊이).
- **미래 에셋 규칙** — 새 소품은 `Layer` 필수 지정, Codex 핸드오프에도 의도 레이어 명시.
- 메인 화면 시안이 오피스 구성을 다르게 제안할 수 있으니, 시안 먼저 보고 3레이어를 그 방향에 맞춘다.
