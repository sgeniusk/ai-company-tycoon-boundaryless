# feat-032 사무실 씬 아트 리워크 — Codex 에셋 핸드오프 (정본)

진짜 사무실 풍경을 만들기 위한 신규 픽셀 에셋 양산 스펙이다. 사용자 피드백 — "다 같이 앞만 보고 일렬은 사무실이 아니다. 칸막이·크기 다른 책상·다른 방향 보며 분주히." 현재 캐릭터/책상은 전부 정면이라, **측면·후면 방향 에셋**이 핵심이다.

## 분업 경계 (엄수)
- Codex — 이 문서의 스프라이트 **양산만**. `unity/` 밖 금지. `.cs`·`progress.md`·`feature_list.json`·`docs/feat-031-*`는 건드리지 않는다.
- Claude — 구성 코드(오피스 씬 큐비클 재작곡)·검증·정본 반영·import 후 캡처.

## 파이프라인 (재사용, 필수)
기존 크로마키 파이프라인을 그대로 쓴다 — `Tools~/pixel_office/feat024/`.
- 생성 — Codex imagegen. **흰 배경 금지**, 크로마키 배경(마젠타 `#FF00FF` 또는 녹색 `#00FF00`) 위에 스프라이트. (메모리 — 흰배경 헤일로는 후처리로 못 지운다. 크로마키가 정답.)
- 키잉 — `key_chroma.py`(키색 제거 + 키색잔여 차단 거리145 + hole-fill/CC + 경계 내부색 디컨탐 + 16색 양자화 + 하드 알파).
- 게이트 — `diagnose_halo.py`로 **경계 헤일로 0 / semi 0 / 키색잔여 0** (GATE PASS)여야 반입. 미달이면 재생성.
- 반입 — `Resources/Art/Actors/`(캐릭터)·`Resources/Art/`(가구)에 `.png` + 결정적 `.meta`(기존 GUID 보존 규칙). 기존 스프라이트와 **같은 96×96 네이티브**(가구는 기존 비율 유지).
- 원본 백업 — `Tools~/pixel_office/feat032/backup_*`.

## 스타일·일관성 요건
- 기존 정면 캐릭터(`actor_human.png` 등, 96×96)의 **팔레트·비율·식별 마커**를 그대로 유지한다. 같은 인물이 몸만 돌린 것이어야 한다(색·헤어·복장 동일).
- human = 사람(따뜻한 살구 피부, 메모리 pixel-art 표준), ai = violet 정본, robot = 로봇. 각 kind 정체성 불변.
- 앉아서 일하는 자세(책상 앞). 카이로소프트 "게임 개발 스토리" 톤.

## 에셋 매니페스트

### 1차 배치 (최우선 — 방향 다양성의 핵심)
kind = human / ai / robot 각각.
1. `actor_{kind}_side.png` — **3/4 측면, 왼쪽을 보고** 앉아 일하는 자세. (오른쪽 방향은 코드에서 hflip으로 만든다 — 우향 에셋 불필요.) 얼굴/모니터가 왼쪽을 향함.
2. `furniture_desk_side.png` — 측면 캐릭터가 앉을 **측면 책상**(모니터가 옆을 향한 책상). 기존 `furniture_desk_wood/white`와 톤 일치. 1종.

→ 1차 = 캐릭터 3장 + 책상 1장 = **4장**.

### 2차 배치 (깊이·밀집 강화, 1차 검증 후)
3. `actor_{kind}_back.png` — **뒤에서 본**(관객 반대편) 앉은 자세. 뒤통수+어깨+의자 등받이. 깊은 줄(뒤 큐비클)에 쓴다.
4. `prop_cubicle.png` — L자 큐비클 칸막이(기존 `prop_partition`보다 크고 코너). 1~2종.

→ 2차 = 캐릭터 3장 + 칸막이 1~2장.

## 산출 후 (Claude가 수행 — Codex 범위 아님)
- import → EditMode/PlayMode 캡처. 오피스 씬을 큐비클 클러스터로 재작곡 — 슬롯별 **방향(정면/좌/우/후면)** 혼합 + **책상 종류·크기 변주** + **칸막이** + 밀집 배치. `OfficeLayout.Slot`에 Orientation/DeskKind/DeskScale 필드 추가, `PlaceOfficeActor`가 방향별 스프라이트(+hflip)·측면책상·칸막이를 배치.

## 수용 기준
- 4장(1차) 모두 GATE PASS(헤일로/키색잔여 0), 96×96, 팔레트·식별 기존과 일치, 크로마키 잔흔 없음.
- 검증 시트(_contact)로 정면 대비 같은 인물 확인.
- `.cs` 무수정, 분담 경계 준수.
