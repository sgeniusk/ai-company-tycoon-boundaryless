# 비주얼 초안 의뢰 브리프 — '클로드 디자인'에 컨셉 초안을 의뢰하고 Claude가 엔진별로 양산하는 2단계 아트 파이프라인

준비 2026-06-15. 활성 트랙 Unity 포트(루트 `progress.md`는 stale, `unity/progress.md` 참조). 직전 비주얼 세션은 feat-020(오피스 아트 전면 AI 재생성). 이 문서는 '클로드 디자인'에 붙여넣을 의뢰 프롬프트 모음이다. **§0 공통 컨텍스트를 항상 맨 앞에 두고, 대상 섹션(§1~§4) 하나를 이어 붙여 한 번에 한 대상씩** 의뢰한다.

## 워크플로우 (2단계)
1. **컨셉 초안** — 이 브리프(§0 + 대상 1개)를 클로드 디자인에 전달해 초안 디자인(방향·키 이미지 컨셉)을 받는다.
2. **사용자 → Claude 전달** — 받은 초안을 Claude Code에 넘긴다.
3. **양산 분배 (Claude)**
   - 미적 앵커 **키 이미지 1컷만 → Midjourney** (미감·구도·색을 잡는 기준 프레임 전용. 양산용 아님).
   - 그 키 이미지를 스타일 레퍼런스로, 인게임 스프라이트 → **Codex imagegen + pixel-art-pipeline** (96~128px, 정본 20색, 투명 배경).
   - 캐릭터 턴어라운드/포즈 시트 → **Antigravity CLI + nano-banana**.
4. **Unity 통합·검증** — pixel-art-pipeline 가공 → `Resources/Art/*` 드롭인 → `ScreenshotCaptureTests` 캡처 대조 + 캐릭터시트 일관성 체크.

**진행 순서** — §1 → §2 → §3 → §4 순차 (사용자 승인 2026-06-15). 스타일 방향 '픽셀아트 고급화'.

## §1. 오피스 아트 확장 `→ 키 이미지 Midjourney 1컷 · 양산 Codex 픽셀 + 포즈 nano-banana`

```
[목표] 지난 작업(캐릭터 3종 · 배경 4종 · 책상 2종 · 소품 5종)을 넘어 인게임의
다양성과 생동감을 키운다. 같은 세계 · 같은 팔레트 안에서의 '확장'이지 리디자인이 아니다.

[그려줄 것]
1) 직원 3종(인간/AI/로봇) 포즈 시트 — idle / work(인간 타이핑 · AI 눈스캔 · 로봇 팔작동) /
   card_use(위 보며 손들고 골드 플래시) / cheer(양손 들고 점프) / alert(레드 강조, 놀란 자세).
   포즈는 가로 행, 캐릭터는 세로 열로 정렬.
2) 추가 배경 단계 / 시간대 — 차고 야간, 확장 데이터센터, 옥상 라운지, 글로벌 본사 로비 등
   현 4종을 잇는 1~2단계. 정면 단면 룸, 바닥 개방(직원 합성용).
3) 추가 소품 5~8종 — 서버랙, 화이트보드, 커피머신, 트로피 진열장, 대형 화분, 회의 테이블 등.

[무드/제약] 정면-약등각, 두꺼운 다크 아웃라인, 96~128px 픽셀로 떨어뜨려도 또렷한 실루엣.
[키 이미지] '확장된 사무실 한 컷'(직원 다포즈 + 신규 소품 + 새 배경)을 Midjourney 앵커로.
[산출 형식] 캐릭터=포즈×캐릭터 격자 시트 / 배경=정면 단면 룸 1컷씩 / 소품=투명배경 격자 시트.
팔레트 스와치 동봉.
```

## §2. 컷씬·이벤트 일러스트 `→ 장면별 키 이미지 Midjourney`

```
[목표] 출시 발표회 · 승급 · 상장 · 엔딩 컷씬이 현재 코드 모션뿐(전용 일러스트 0장)이다.
감정의 절정을 담은 키 비주얼을 입혀 '한 방'을 만든다.

[그려줄 것]
1) 신제품 출시 발표회 — 무대 스포트라이트, 발표하는 직원, 환호하는 객석(로봇 · AI 섞임).
2) 시리즈 투자 유치 / IPO 상장 축하 — 종 울리기, 색종이, 그래프 상승.
3) 멀티 엔딩 대표 키아트 3~5종 — 예) 글로벌 제패, 인수합병, 조용한 은퇴, 특이점 도달.
4) 위기 · 경쟁 이벤트 삽화 — 라이벌의 역습, 자금난, 서버 다운 같은 긴장 장면.

[무드/제약] 픽셀 고급화 또는 픽셀-일러스트 하이브리드. 게임 캐릭터의 식별 마커 · 팔레트 톤은
유지하되 인게임 스프라이트보다 묘사 · 연출 자유도를 높인다(미적 임팩트 우선).
[키 이미지] 각 장면을 대표하는 1컷이 곧 Midjourney 앵커다. 이 섹션은 키 이미지 비중이 가장 큼.
[산출 형식] 장면별 키 비주얼 컨셉 — 구도 · 색감 · 감정 · 조명 키워드를 풍부하게
(Midjourney 프롬프트로 옮기기 좋게). 가로/세로 컷 명시.
```
