# agy 핸드오프 — 출시 컷씬 직원 포즈 스프라이트 (feat-017)

Antigravity(agy)로 픽셀 캐릭터 포즈를 생성해, 출시 컷씬(`CutsceneDirector`)의 코드 모션을 진짜 포즈 스프라이트로 업그레이드한다. agy 비대화형은 권한 거부 이력이라 **사용자가 대화형 agy로 실행**한다.

## 목적
현재 컷씬은 v090 기본 정면 스프라이트(차렷 포즈)를 위치·스케일·회전 코드 모션으로 움직인다. "보는 맛"은 나지만 포즈가 차렷 하나뿐이다. 발표·환호·박수·점프 **포즈 스프라이트**가 있으면 훨씬 생동감 있다.

## 절대 제약 — 스타일 정합 (가장 중요)
새 포즈가 기존 v090 직원과 **도트 크기·윤곽선 두께·팔레트·비율**이 어긋나면 컷씬에서 따로 논다(배경 재작업 때 겪은 불일치와 동일 함정). 반드시 기존 캐릭터를 베이스로 포즈만 바꾼다.
- **레퍼런스 원본** — `unity/Assets/_Project/Resources/Art/UI/v090-workforce-actor-hires.png` (768×256, 3종 가로 배열, 각 256×256). 생성 스크립트 `scripts/assets/generate-v090-workforce-actor-hires.mjs`.
- **캐릭터 3종** — ① 인간(검은 머리, 민트 베스트, 노란 넥타이) ② AI(보라 모니터헤드 로봇, 청록 화면+눈) ③ 로봇(흰/회색 박스 로봇, 빨강 안테나).
- **팔레트(v090 추출, 고정)** — 윤곽선 `#1F1912`, 민트 `#5FC6A6`/`#2B6B4F`, 메탈 `#8A9AA0`/`#CDDADB`/`#485662`, 남색 `#1C303F`/`#242D35`, 청색 `#335F7A`, 보라 `#6F5BB2`/`#3F3771`, 노랑 `#F4CC70`, 살색 `#F5B884`, 크림 `#FFF7DF`, 빨강 `#D64838`. 새 색 도입 금지.
- 정면 뷰, 1px 어두운 윤곽선, 발밑 타원 그림자 유지(v090과 동일).

## 필요 포즈 (캐릭터 3종 × 각 포즈)
| 포즈 키 | 설명 | 용도 |
|---|---|---|
| `_present` | 양팔(또는 한 팔)을 위로 들어 머리 위 제품을 제시 | 발표 직원 |
| `_cheer` | 양팔 만세, 입 벌림(인간)/화면 ^^(로봇) | 객석 환호 절정 |
| `_clap` | 양손을 가슴 앞으로 모은 박수 | 객석 박수 |
| `_jump` | 무릎 굽힘→점프(살짝 웅크림 or 공중) | 객석 점프 |

→ 예: 인간 발표 = `actor_human_present`. 최소 우선순위는 **`_present`(인간) + `_cheer`(3종)**. 나머지는 여력 되면.

## 출력 사양
- 각 포즈 256×256 PNG, 투명 배경(흰 배경 생성 후 알파화도 가능).
- 아틀라스로 묶으면 v090과 동일 레이아웃(가로 N칸, 각 256) 선호 — IconAtlasImporter 재사용. 개별 PNG도 가능.
- 바이블 식별 마커(v090에 있던 결정성 확인용 마커) 동일 규약 유지.

## 반입 후 배선 (Claude 담당)
1. 새 스프라이트를 `Resources/Art/UI/`에 두고 IconLibrary에 포즈 키 등록.
2. `CutsceneDirector.MakeActor`가 상황별 포즈 키를 받게 확장(발표=`_present`, 객석=`_cheer`/`_clap` 교대).
3. 코드 모션은 보조로 유지(포즈 스프라이트 + 미세 들썩/점프). 캡처 회귀 `15-cutscene-launch.png` 재촬영.

## 검수 포인트
- v090 3종과 나란히 뒀을 때 같은 캐릭터로 읽히는가(스타일 일치).
- 포즈가 과장돼도 도트 그리드·윤곽선이 흔들리지 않는가.
