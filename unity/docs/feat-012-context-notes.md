# feat-012 컨텍스트 노트 — AI 테크트리: 분류·떡밥·해금 (설계 확정본)

사용자 방향(2026-06-11) — "제품·능력·업그레이드가 많아 분류가 필요하다. AI 세계는 무궁무진하니 더 많아질 것. 잠긴 제품은 ??? 제목 + 잠금 + 떡밥 설명으로 기대심리를 만들고, 연구가 차오르면 해금. 모든 제품·연구가 문명 시리즈처럼 트리/요건으로 연계되는 꼼꼼한 설계."

## 0. 현재 구조 진단
이미 3층 연계가 존재한다 — **능력 레벨 → 도메인 해금(unlocks_domains) → 제품(required_capabilities + domain + trust)**. 문제는 (a) UI가 평면 리스트라 구조가 안 보이고, (b) 잠긴 제품 36종이 전부 노출돼 미지의 영역이 없고, (c) 제품 간 선행 관계가 없어 트리가 2단에서 끝난다.

## 1. 설계 원칙 (문명식)
1. **항상 다음 목표가 보인다** — 어떤 화면에서든 "다음에 열릴 것"이 최소 1개 티저로 보여야 한다.
2. **발견의 사다리** — 숨김 → ??? 티저 → 실명 해금 → 출시. 연구가 진행될수록 ???가 늘고 실명으로 바뀐다.
3. **분야별 무궁무진** — 도메인 섹션 헤더에 `발견 N / 전체 M` 진행도. M에는 숨김도 포함해 "아직 못 본 게 있다"를 수치로 보여준다.
4. **트리는 깊게, 입구는 좁게** — 초반 가시 제품은 적게(차고 감성), 후반 분기는 넓게.

## 2. 노출 상태 머신 (derive-only — 세이브 필드 추가 없음)
제품별 상태를 매 갱신 시 파생 계산한다. 우선순위 순서대로 첫 매칭.
- **launched** — ActiveProducts 포함.
- **unlocked** — 도메인 해금 && required_capabilities 충족 && 선행 제품(prerequisite_products) 전부 출시. 전체 공개 + 출시 버튼.
- **revealed** — 도메인 해금 but 요건 미충족. 실명 공개 + 잠금 + 부족 요건 표기(현 GetProductLockReason 확장).
- **teaser** — 도메인 미해금이지만 "근접": 도메인을 해금하는 능력 중 하나라도 `현재 레벨 >= 해금 레벨 - 2`. 표시는 `???` 제목 + 🔒 + `teaser` 떡밥 문구 + 힌트 한 줄("힌트 — 로보틱스 계열 연구를 진행해 보세요"). 출시 불가, 상세 없음.
- **hidden** — 그 외. 리스트에 없음. 섹션 푸터에 `미발견 제품 N종` 카운터만.

티저 근접 임계(레벨-2)는 상수로 빼고 밸런스 패스에서 조정한다. 동일 상태 머신을 도메인 헤더에도 적용(도메인 자체가 teaser/hidden 가능).

## 3. 데이터 스키마 확장 (정본 ../../data/*.json — additive, React 비파괴)
- `products.json` 각 항목에 추가
  - `teaser` (string) — ??? 상태에서 보여줄 떡밥 한 줄. 예: "하늘을 나는 무언가의 도면이 책상 위에 있습니다."
  - `prerequisite_products` (string[]) — 선행 제품 id. 예: `flying_drone_taxi`는 `["autonomous_kit", "ev_os"]`.
  - `tier` (int, 1~4) — 트리 깊이 표기·정렬·검증용(1 차고, 4 미래).
- `capabilities.json` 각 항목에 `category` (string) — research/engineering/business 등 그룹 헤더용.
- 신규 미래 제품 웨이브(+12~16종, 도메인당 1~2) — 자율주행 키트, 플라잉 드론 택시, 휴머노이드 가사로봇, AGI 연구 보조, 신약 탐색 엔진, 핵융합 제어 보조 등 tier 3~4 중심. 모두 teaser 필수.
- 검증 스크립트(validate-data 또는 EditMode) — 선행 그래프 사이클 금지, 존재하지 않는 id 참조 금지, tier 단조성(선행 tier <= 자기 tier), 모든 제품 도달 가능(요구 능력 max_level로 충족 가능).

## 4. UI 설계
- **제품 팝업** — 도메인 섹션 헤더(이름+상태+`발견 N/M`+접기) > 제품 카드(launched/unlocked/revealed/teaser 4스타일) > 푸터 미발견 카운터. teaser 카드는 어둡게+??? 굵게+떡밥 이탤릭.
- **능력 팝업** — category 그룹 헤더. 각 능력 카드에 "다음 레벨이 여는 것" 미리보기(도메인/제품 teaser 연결) 한 줄 — 연구 동기를 직접 보여준다.
- **업그레이드 팝업** — 기존 일반/자동화 2그룹 유지, 카드에 tier 뱃지만 추가.
- **해금 모먼트** — teaser→revealed 전환 시 ToastRibbon("새 제품 발견 — 플라잉 드론 택시!") + SpawnCelebration 재사용. 월 정산 후 상태 전환 검사(derive 비교).
- 분량 — 모든 신규 텍스트는 UiTheme 폰트 위계(최소 26) 준수.
