사용자 방향(2026-06-13) — "게임개발 스토리처럼 사무실 연출을 비주얼적으로 완성하자. 오래된 게임이니 우리는 더 발전한다. 기존 레이아웃은 유지하되 연출력은 확 올려 아기자기하고 보는 맛을 늘린다." dynamic workflow + ultracode로 트랙별 구현·검증.

## 0. 진단 — 현재 vs 카이로소프트 격차 (실증)
feat-016~018로 정면 단면 픽셀룸 4종 + 폰 cover 고해상도 + 벽 디테일은 완성됐다. 실게임 캡처(`Logs/shots/01c-phone-2400.png`·`10-dopamine.png`) 기준 4가지가 빈다.
1. **밀도** — 직원이 바닥 한 줄(`min(talent,6)`)로 서 있고 책상은 배경에 룸당 2~3개. 중앙·상단 워크스테이션이 없어 휑함.
2. **생기** — 스크림 `0.38`(`GameScreen.cs:385`)이 오피스 전체를 회색으로 덮어 픽셀 배경 채도를 죽임.
3. **상시 juice** — `WorkLoop` 이모트(36px, 3.5~7.5초)는 있으나 말풍선·직원별 리워드 플로팅·열일 불꽃이 없어 일꾼별 생명감 부재. juice가 중앙(매출/축하)에 몰림.
4. **앰비언트** — 배경이 정적 PNG라 방 자체가 정지(모니터·LED·조명 무동작).

## 1. 사용자 결정 (3 체크포인트)
- **캐릭터 배치 → 2열 원근.** 앞줄(큰 액터)+뒷줄(작은 액터) 2단 깊이 + 책상 클러스터에 착석. 정면 단면 레이아웃·cover 프레이밍은 유지.
- **밝기 → 그라데이션/로컬 스크림.** 오피스 중앙 무대는 거의 풀밝기(생기↑), 상단 HUD·하단 가이던스 리본 뒤에만 스크림을 깔아 폰트26 가독성 유지.
- **룸 애니 → 풀세트.** 정적 PNG 위 가벼운 절차 애니 오버레이로 방을 살아있게.
