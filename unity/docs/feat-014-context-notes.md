# feat-014 컨텍스트 노트 — 경영 탭: 인재·사무실·전략 (설계 초안)

사용자 방향(2026-06-11, 에디터 플레이 직후 아이디어 덤프) — "채용엔 종류가 있고 인재도 고르고. 사무실 크기로 인원 제한. 경영 탭에 사무실 업그레이드·본사 이전. 업그레이드는 자본력으로 하는 것 — 경영 탭으로 바꿔 CM·경쟁사 흠집내기 같은 전략 활동. 제품 누르면 기대수익/실제수익. GPU는 TOPS 표기. 인재 능력치가 개발력으로. 성급별 해금 — 2성 연구·제품 확장, 4성 데이터센터·저탄소 사옥, 5성 랜드마크 본사. 아이디어는 무궁무진."

## 0. 핵심 — 전부 React 정본 데이터가 이미 있다 (포팅이지 발명이 아님)
| 사용자 아이디어 | React 정본 소스 | 비고 |
|---|---|---|
| 인재 종류·고르는 채용 | `data/agent_types.json` **22종** (kind human/ai, rarity, stats 8축, 외형 기술) | React hireAgentViaChannel + 채용 채널 |
| 인재 능력치 → 개발력 | agent stats(research/engineering/...) | 연구 비용 할인·데이터 생산 등으로 환산 |
| 사무실 크기 인원 제한 | `data/office_expansions.json` **6단계** (`hire_capacity` 3→...) | 차고형 연구실부터 |
| 본사 이전 | `data/company_locations.json` **5곳** (유지비 모디파이어, 인재 풀, 채용 할인) | 강원 산골 차고 → ... |
| 경쟁사 흠집내기 등 전략 | `data/rival_events.json` 5종 + `data/strategy_cards.json` 19종 | 리스크/효과 구조 기존재 |
| 성급별 해금 | `data/company_stages.json` 6단계 | 성급 = 스테이지, 해금 게이트로 활용 |

## 1. 경영 탭 신설 (하단 도크 개편)
- 현 [제품·능력·업그레이드] → **[제품·연구·경영]**.
  - 연구 = 현 능력 탭 (채용·GPU 증설 카드는 경영으로 이동).
  - **경영 탭** = ① 인재(채용·로스터) ② 시설(사무실 업그레이드·본사 이전·데이터센터) ③ 전략(자본 활동 — 마케팅 캠페인·경쟁사 흠집내기·기존 업그레이드 통합).
- 기존 업그레이드/자동화는 경영 탭 '전략·투자' 그룹으로 흡수 (id·세이브 불변, 표시 위치만).

## 2. 인재 시스템 (agent_types 포팅, 최소 버전)
- **채용 = 후보 3택1** — 누르면 후보 3명(시드 파생, 사무실 인재 풀 가중)이 카드로 — 이름·종류(사람/AI)·능력치·월급. 고르면 talent +1 + 로스터 등록.
- **개발력 환산 (derive-only)** — 로스터 합산 스탯이 보너스로: research → 능력 강화 비용 -x%, engineering → 제품 강화 비용 -x%, operations → 자동화 +, growth → 이용자 +%. 시작은 2~3축만 (research/engineering 우선).
- **정원** — `office_expansions.hire_capacity`. 정원 초과 시 채용 잠금 + "사무실이 좁습니다" 사유.
- 세이브 — 로스터(고른 agent_type id 목록)는 신규 필드 v5 (구세이브 빈 로스터 = 익명 talent, 무손실).

## 3. 시설 (office_expansions + company_locations 포팅)
- **사무실 업그레이드** — 6단계, 정원·장식 슬롯·월간 효과. 오피스 스테이지 비주얼 연동(가구 밀도/배경 — 기존 office_scene 에셋 재사용 우선).
- **본사 이전** — 5곳. 유지비 모디파이어(MonthController 비용 곱), 인재 풀(채용 후보 가중), 해금 요건.
- **성급 게이트** — 2성: 사무실 2단계+연구 슬롯, 4성: 데이터센터 건립(연산력 월간 +, GPU 증설 상위 티어)·저탄소 사옥, 5성: 랜드마크 본사(비주얼+위신 보너스). company_stages 요건에 매핑.

## 4. 전략 활동 (자본으로 하는 것)
- 마케팅 캠페인(기존 social_media_campaign 등 업그레이드 재배치) + **경쟁사 견제** — 리스크 액션: 비용+신뢰 하락 리스크 vs 경쟁사 모멘텀 감소(MarketService.CompetitorStates에 효과). rival_events 데이터 톤 재사용.
- 쿨다운/월 1회 제한으로 스팸 방지.

## 5. 표기 폴리시 (소소하지만 체감 큰 것)
- **GPU → TOPS 표기** — 연산력 칩/카드에 "연산력 1,250 TOPS"처럼 단위 표기 (resources.json 표시명 활용, 숫자 의미 불변).
- **제품 기대수익 vs 실제수익** — 제품 카드에 "예상 월 매출 $X" (레벨·신뢰 멀티 반영 파생), 월 정산 리본에 실제와 대비. 이미 derive 함수 존재(ApplyMonthly 분해).
