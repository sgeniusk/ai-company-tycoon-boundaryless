# feat-016 컨텍스트 노트 — 오피스 비주얼 전면 재작업 (진짜 사무실 픽셀아트)

사용자 피드백(2026-06-13) — "저 오피스는 이제 바뀌어야 하지 않을까. 전혀 오피스 같지도 않고. 정말 사무실을 픽셀아트로 그려내야 할 것 같은데."

## 0. 문제 진단
현재 배경(`office.png` + Codex 생성 `office_growth/datacenter/landmark`)은 **떠 있는 등각 나무 마루 플랫폼 + 측면 빌딩 + 녹색 픽셀 화분** 구도다. 이건 "사무실"이 아니라 추상적 디오라마 무대처럼 보인다. 직원 액터·가구가 그 위에 얹혀도 **실내 사무실 공간감이 전혀 없다**.
- 근본 한계 — office.png 자체가 카이로소프트 "게임 개발 스토리"의 사무실 룸이 아니라, 떠 있는 판때기 위에 오브젝트를 흩뿌린 형태다. Codex가 이 구도를 앵커로 3종을 더 만들면서 **잘못된 방향을 증폭**했다.

## 1. 목표 방향 (사용자 의도)
**진짜 사무실 픽셀아트.** 카이로소프트 게임 개발 스토리처럼 — 벽·바닥·창문이 있는 **실내 룸**, 책상이 격자로 놓이고 직원이 그 앞에 앉아 일하는 톱다운/정면 혼합 뷰. "회사 안"에 들어와 있는 공간감.
- 픽셀아트 — 진짜 도트(픽셀) 스타일. 현재의 벡터 평면 일러스트가 아니라.
- 성급 사다리 유지 — 차고(좁고 허름) → 스타트업 오피스 → 데이터센터 통합 → 랜드마크 본사. 단 **전부 "방 안" 구도**로 통일.

## 2. 결정 필요 (사용자 체크포인트)
1. **뷰 앵글** — 톱다운(바닥 위주) vs 정면 단면(벽+바닥, 인형의 집 단면) vs 현행 등각 유지하되 룸으로. → 액터(v090 직원)는 정면 스프라이트라 **정면 단면 룸**이 가장 자연스러울 듯.
2. **액터 재사용 여부** — 현 v090 직원 3종 스프라이트를 그대로 쓸지, 룸 구도에 맞춰 재생성할지. 재사용하면 룸은 직원이 앉을 책상 라인을 가진 배경이어야 함.
3. **생성 경로** — (a) Codex 절차 생성(현 한계 — 진짜 픽셀아트 디테일 어려움) (b) 사용자 대화형 agy/이미지 툴 (c) 외부 픽셀아트 에셋. agy 비대화형은 권한 거부 이력.

## 3. 영향 범위
