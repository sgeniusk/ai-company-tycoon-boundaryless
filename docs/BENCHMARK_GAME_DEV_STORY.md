# Game Dev Story iOS 벤치마크

## 목적

AI 컴퍼니 타이쿤의 1차 감각 벤치마크는 카이로소프트의 Game Dev Story iOS 버전입니다. 목표는 복제가 아니라, 모바일 경영 시뮬레이션으로서의 압축감, 읽기 쉬움, 빠른 보상 루프, 회사 성장 판타지를 분석해 AI 회사 소재에 맞게 변환하는 것입니다.

## 확인한 기준 자료

- Apple App Store의 Game Dev Story 설명은 인기 조합을 찾고, 최신 플랫폼을 대상으로 개발하며, 프로그래머부터 사운드 엔지니어까지 다양한 직군의 스태프를 운용하는 게임으로 소개합니다.
- Kairosoft iOS 공식 페이지는 Game Dev Story를 "게임 회사 시뮬레이션"으로 소개하며, 회사를 운영하고 콘솔을 개발하고 스태프를 고용해 밀리언셀러 히트를 노리는 구조를 강조합니다.
- Game Dev Story의 공개 게임플레이 설명은 좌측 메뉴로 회사를 운영하고, 출시 게임의 판매량을 확인하며, 채용, 개발 방향, 속도, 아이템, 사무실 확장, 리뷰 점수와 판매량이 맞물리는 구조를 보여줍니다.

## 가져올 감각

### 1. 작은 회사가 커지는 사무실 판타지

Game Dev Story는 거대한 산업을 다루지만 화면은 작은 사무실과 직원 단위로 시작합니다. 이 대비가 중요합니다.

AI 컴퍼니 타이쿤 적용:

- 처음에는 작은 AI 도구 팀으로 시작합니다.
- 제품이 늘고 능력이 올라가면 회사 단계가 바뀝니다.
- MVP에서는 실제 사무실 애니메이션보다 "회사 단계", "활성 제품", "인재", "회사 기록"으로 먼저 성장감을 전달합니다.

### 2. 조합의 재미

Game Dev Story의 핵심 쾌감은 장르, 방향, 플랫폼, 직원 능력의 조합에서 나옵니다.

AI 컴퍼니 타이쿤 적용:

- AI 능력 + 산업 분야 + 제품 포지션 조합으로 바꿉니다.
- 예: 언어 Lv.1 + 개인 생산성 = AI 글쓰기 비서.
- 예: 언어 Lv.2 + 고객 지원 + 신뢰 40 = 고객지원 챗봇.
- 장기적으로는 "능력 조합이 예상 밖의 산업을 연다"는 느낌을 강화합니다.

### 3. 출시 후 즉시 피드백

Game Dev Story는 제품 완성 후 리뷰 점수, 판매량, 순위가 나오기 때문에 플레이어가 방금 한 선택을 빠르게 평가할 수 있습니다.

AI 컴퍼니 타이쿤 적용:

- 제품 출시 후 화제성, 이용자, 매출, 데이터, 신뢰 변화가 즉시 보여야 합니다.
- 월 진행 후 "이번 달 성과 보고"가 필요합니다.
- 이후 마일스톤에서 제품별 "시장 반응 점수" 또는 "AI 평가단 리뷰"를 추가합니다.

### 4. 짧은 입력, 긴 상상

카이로소프트 게임은 버튼 하나를 눌렀을 뿐인데 플레이어 머릿속에서는 회사가 바쁘게 굴러갑니다.

AI 컴퍼니 타이쿤 적용:

- 플레이어 조작은 간단해야 합니다: 출시, 연구, 채용, 투자, 다음 달.
- 결과 문구는 상상력을 자극해야 합니다.
- 예: "AI 글쓰기 비서가 크리에이터 커뮤니티에서 입소문을 탔습니다."
- 예: "연산력 부족으로 응답 지연이 발생했습니다. 신뢰가 흔들립니다."

### 5. 가벼운 유머와 산업 패러디

Game Dev Story는 게임 산업 유머와 패러디가 강점입니다.

AI 컴퍼니 타이쿤 적용:

- AI 업계의 과장 광고, GPU 품귀, 오픈소스 경쟁, 환각 사고, 기업 계약, 규제 조사 같은 현대적 사건을 가볍지만 날카롭게 다룹니다.
- 실제 회사명이나 특정 인물을 그대로 희화화하지 않고, "업계에서 있을 법한 상황"으로 만듭니다.

## 그대로 가져오면 안 되는 것

- Game Dev Story의 UI, 픽셀 아트, 사무실 구성, 텍스트, 고유 시스템을 그대로 복제하지 않습니다.
- "게임 개발사" 구조를 "AI 회사"에 억지로 치환하지 않습니다.
- AI 회사의 독특한 핵심인 연산력, 데이터, 신뢰, 안전, 자동화, 도메인 확장은 반드시 독립적인 재미로 설계합니다.

## AI 회사 버전 변환표

| Game Dev Story 요소 | AI 컴퍼니 타이쿤 변환 |
|---|---|
| 게임 장르/타입 조합 | AI 능력/산업 분야/제품 포지션 조합 |
| 플랫폼 선택 | 시장/도메인/배포 채널 선택 |
| 직원 직군 | 연구원, 엔지니어, 안전 담당, 영업, 운영 자동화 |
| 개발 품질 | 제품 완성도, 신뢰, 모델 성능 |
| 리뷰 점수 | 시장 반응, 언론 평가, 사용자 만족, 기업 신뢰 |
| 판매량 | 이용자, 매출, 계약, 데이터 증가 |
| 콘솔 개발 | 자체 플랫폼, 에이전트 생태계, AI 운영체제 |
| 사무실 확장 | 회사 단계, 조직 규모, 자동화 수준 |
| 명예의 전당 | 산업 표준, 글로벌 AI 플랫폼, Boundaryless 단계 |

## MVP 설계 반영

다음 마일스톤부터 MVP의 화면과 시스템은 이 원칙을 따릅니다.

1. 첫 화면은 운영 화면이어야 합니다.
2. 버튼은 적고 결과는 풍부해야 합니다.
3. 제품 출시 후 즉시 변화가 보여야 합니다.
4. 잠긴 제품은 "왜 못 하는지"가 바로 보여야 합니다.
5. 월 진행은 항상 작은 드라마를 만들어야 합니다.
6. 숫자만 오르는 게임이 아니라 "우리 회사가 뭘 해냈는지"가 기록되어야 합니다.

## 우선 개선 과제

- 제품 출시 카드에 "예상 시장 반응" 추가.
- 월간 보고서 패널 추가.
- 제품별 누적 이용자/매출 표시.
- 회사 단계 표시.
- 인재를 단순 숫자가 아니라 직군 카드로 확장.
- Game Dev Story의 리뷰 점수에 해당하는 "AI 시장 반응 점수" 설계.
