# 로그라이트 덱빌딩 전환 설계

작성일: 2026-05-15  
대상 버전: `v0.12.0-alpha`

## 결정

`AI Company Tycoon: Boundaryless`의 핵심 장르 정의를 다음으로 전환한다.

> 로그라이트 덱빌딩 AI 회사 경영 게임

기존의 카이로소프트식 작은 회사 운영 감각은 유지한다. 다만 한 번 클리어하면 동기가 약해지는 문제를 해결하기 위해, 실패한 런도 다음 창업에 남는 메타 진행으로 바꾼다. 제품 개발은 단순 시간 경과가 아니라 카드 선택과 작은 개발 퍼즐의 점수로 결과가 달라지게 한다.

## 흥행 가설

- Game Dev Story식 경영 압축감은 첫 진입과 이해가 빠르다.
- 덱빌딩은 “이번 런의 회사 성격”을 매번 다르게 만든다.
- 로그라이트 메타는 실패를 보상으로 바꿔 재시작 이유를 만든다.
- AI 회사 소재는 카드 효과를 자연스럽게 확장한다. 예: 프롬프트, GPU, 레드팀, 오픈소스, 자동화, 규제 대응.
- 개발 퍼즐은 숫자만 보는 경영을 손맛 있는 의사결정으로 바꾼다.

## v0.12 범위

이번 버전은 완성형 덱빌더가 아니라 “방향 전환의 작동 가능한 뼈대”다.

- 전략 카드 데이터와 시작 덱.
- 손패, 드로우 더미, 버림 더미.
- 카드 사용으로 자원, 개발 진행, 완성도 변화.
- 3x3 개발 퍼즐 생성과 자동 해결 프로토타입.
- 런 보상인 창업 통찰.
- 창업 통찰로 메타 해금 후 새 런 시작.
- 덱/퍼즐/메타 메뉴.
- 저장/불러오기 호환.
- 데이터 검증과 단위 테스트.

## 시스템 구조

시뮬레이션 상태는 계속 `GameState`가 단일 원천이다.

- `data/strategy_cards.json`: 카드 이름, 비용, 효과, 태그, 해금 조건.
- `data/meta_unlocks.json`: 메타 해금 비용, 시작 보너스, 해금 카드.
- `src/game/deckbuilding.ts`: 덱 생성, 카드 사용, 드로우 처리.
- `src/game/development-puzzle.ts`: 프로젝트별 3x3 이슈 보드와 점수 계산.
- `src/game/meta-progression.ts`: 런 보상, 메타 해금, 새 런 시작.
- `src/components/MenuPanels.tsx`: 덱 메뉴 UI.

렌더러나 UI가 룰을 소유하지 않는다. UI는 버튼을 통해 시스템 함수만 호출한다.

## 개발 퍼즐 방향

현재 퍼즐은 자동 선택형 프로토타입이다. 보드는 3x3이며 각 칸은 `UX`, `버그`, `연산 병목`, `안전 리스크`, `모델 한계`, `시장 포지션`, `데이터 품질`, `사용감`, `자동화` 같은 이슈다.

다음 버전에서는 플레이어가 직접 3~4개 이슈를 선택하고, 에이전트 능력치와 카드 시너지가 점수에 반영되게 만든다. 점수는 제품의 출시 리뷰, 월간 성장, 사고 위험에 연결한다.

## 하네스 검증 포인트

- 덱 시스템 엔지니어: 카드가 자원/프로젝트에 실제 영향을 주는지 검증.
- 로그라이트 메타 엔지니어: 실패가 다음 런의 선택지로 이어지는지 검증.
- 퍼즐 메커닉 엔지니어: 퍼즐 점수가 제품 결과를 바꾸는지 검증.
- 밸런스 시뮬레이션 엔지니어: 카드가 무한 자원 루프를 만들지 않는지 검증.
- 리텐션/LTV 에이전트: “다음 런에서 새 카드를 써보고 싶다”가 보이는지 검증.
- 솔로 개발 스코프 에이전트: 그래픽 없이도 재미를 만드는지, 콘텐츠 폭발을 막는지 검증.

## 성공 기준

- 새 게임 시작 시 시작 덱과 손패가 생성된다.
- 카드 사용이 개발 중인 프로젝트의 진행도/완성도를 바꾼다.
- 개발 퍼즐 해결이 프로젝트 품질과 진행도를 바꾼다.
- 실패 또는 완료된 런이 창업 통찰을 만든다.
- 창업 통찰로 메타 해금을 하면 다음 런의 덱에 새 카드가 들어간다.
- `npm test`, `npm run validate:data`, `npm run build`가 통과한다.

## 다음 설계 질문

v0.12.1에서는 “개발 퍼즐을 어느 정도로 플레이어가 직접 조작하게 할지”가 핵심이다.

추천 방향은 카드와 퍼즐을 분리하지 않는 것이다. 카드를 먼저 사용해 보드의 칸을 약화시키거나, 퍼즐을 해결해 카드 보상을 받는 식으로 두 시스템을 한 루프로 묶어야 한다.
