# 로드맵 — AI Company Tycoon: Boundaryless

작성일: 2026-05-16

## 현재 상태

현재 버전은 `v0.15.0-alpha`다. 프로젝트 방향은 **카이로소프트식 AI 회사 경영 + 로그라이트 덱빌딩 + 선택형 개발 이슈 대응 + 실패 후 재도전 메타 + 10년 캠페인 엔딩**으로 조정했다. 그래픽 에셋을 제외한 핵심 프로토타입은 다음 기반을 갖췄다.

- 첫 제품 개발과 출시.
- 첫 AI 모델 제품인 파운데이션 모델 v0.
- 출시 평가, 성장 경로 선택, 경쟁사 압박.
- 10개월 MVP 진행도.
- 업적/런 목표와 보상.
- 시작 덱, 손패, 드로우/버림 더미.
- 전략 카드 사용으로 개발 진행도/완성도 변화.
- 직원/에이전트 배치를 기본 개발력으로 삼는 제품 개발.
- 선택형 개발 이슈 대응.
- 카드가 이슈 대응 난이도, 점수, 선택 칸 수에 영향을 주는 카드-이슈 시너지.
- 출시 후 카드 보상 3택1.
- 덱 편집 토큰으로 카드 제거/강화.
- 반도체, 모빌리티, 로봇, 커피 체인, 장난감 산업으로 이어지는 경계 없는 확장.
- 제품 개발 시작 전 직접 고르는 직원 배치.
- 직원 조합이 개발 기간보다 완성도와 출시 리뷰에 더 크게 영향을 주는 출시 산식.
- 보유 아이템을 확인하는 인벤토리 패널.
- 실제 고용 직원과 프로젝트 보드가 보이는 사무실 화면.
- 사무실 확장 단계와 고용 한도.
- 사무실 장식 슬롯과 배치/보관.
- 배치된 장식 아이템의 개발 품질 보너스.
- 첫 10분 행동 루프: 고용, 제품 시작, 카드/이슈 대응, 출시, 성장 선택, 사무실 정비, 경쟁 대응.
- 가이드 카드의 단계별 진행률과 현재 목표 표시.
- 로봇공학 연구 후 고용 가능한 로봇형 개발 인력.
- 초반 8개 경쟁사와 12개월/24개월 신규 강력 경쟁사 등장 기반.
- 창업 통찰, 메타 해금, 새 런 시작.
- 런 결과에서 대표 제품, 대표 카드, 경쟁사 압박, 통찰 보상 확인.
- 새 런 시작 시 최근 런 기록 저장.
- 덱 메뉴에서 최근 런 기록 확인.
- 10분 알파 시뮬레이션 하네스.
- 결과 화면에서 다음 런 미리보기까지 이어지는 로그라이트 폐곡선.
- 결과 화면의 다음 런 브리핑: 예상 통찰, 해금 후보, 첫 행동 3단계.
- 새 런 시작 직후 덱 메뉴의 온보딩 브리핑과 최근 런 기록 안내.
- 데스크톱 한 화면형 게임 셸: 좌측 자원/명령, 중앙 사무실, 우측 메뉴 패널.
- 운영/성장/시장으로 나뉜 압축 메뉴 레일.
- 10년 캠페인 시계: 1턴 1개월, 120개월 최종 평가.
- 1년차부터 10년차까지 이어지는 연간 심사, 목표 진행도, 통과/미달 보상.
- 연간 심사 후 다음 1년 동안 적용되는 후속 지시와 월간 모멘텀.
- 연간 심사 직후 다음 해 운영 지시를 3택1로 고르는 선택형 전략 루프.
- 선택한 연간 운영 지시가 다음 제품 출시의 카드 보상 후보를 편향하는 덱 시너지.
- 덱 보상 화면에서 현재 연간 운영 지시의 카드 보상 편향 태그 표시.
- `?scenario=reward-bias`로 연간 지시 보상 편향 덱 화면을 바로 여는 QA 진입점.
- 보상 카드별로 현재 연간 지시와 맞는 태그를 보여 주는 `지시 보너스` 배지.
- 연간 운영 지시를 다음 제품, 연구, 경쟁 대응 추천으로 확장하는 `연간 전략실`.
- `?scenario=annual-strategy`로 연간 전략실을 바로 여는 QA 진입점.
- 낮/밤 사무실 분위기 전환.
- 한국 시골 차고에서 시작해 지방 창업센터, 판교, 서울, 글로벌 캠퍼스로 이전하는 지역 성장 구조.
- 사람 직원 기반 AI 운용 한도.
- 사람 직원 6종, AI 에이전트 13종, 로봇 인력 3종으로 확장된 장기 고용 풀.
- 복지, 제조, 하드웨어, 모빌리티를 포함한 37종 아이템 풀.
- 아이템과 연결되는 사무실 사물 21종과 픽셀아트 아이콘 훅 18종.
- 캠페인 단계별 콘텐츠 추천: 차고, 초기 스타트업, 스케일업, 기업/산업 확장, 프런티어.
- 고용 화면의 사람/AI/로봇 필터와 상점 카테고리 필터.
- 현재 단계에 맞는 추천 고용 후보와 추천 아이템 표시.
- 실제 미접속 일수만큼 게임 일수 손익을 반영하는 오프라인 정산 팝업.
- 경쟁사별 대응 플랜.
- 경쟁사 선점 제품에 맞춘 추천 카드, 추천 제품, 추천 연구.
- 경쟁사 위협과 모멘텀을 낮추는 카운터 카드.
- 제품 메뉴의 산업별 필터.
- 성장 경로 월간 효과.
- 제품 업그레이드.
- 저장/불러오기와 깨진 저장 복구.
- 런 결과 랭크와 다음 행동 추천.
- 자동 밸런스 시뮬레이션 하네스.

## 개발 원칙

- 사용자에게 보여 주는 보고서와 로드맵은 한국어로 작성한다.
- 한 번 개선할 때마다 버전을 붙이고 변경 기록을 남긴다.
- 플레이어가 느끼는 재미를 먼저 고정하고, 그래픽은 그 다음에 붙인다.
- 매 버전은 테스트, 데이터 검증, 빌드를 통과해야 한다.
- 큰 기능은 반드시 QA 시나리오나 시뮬레이션 하네스로 반복 검증한다.
- Vercel 프로덕션 배포는 큰 버전업 또는 사용자가 공개 확인을 요청한 경우에만 진행한다.

## 단기 계획

### v0.15.0-alpha — 연간 전략실

상태: 완료.

목표: 연간 운영 지시가 월간 보너스와 카드 보상 편향을 넘어, 플레이어의 다음 행동 후보까지 이끌게 만든다.

구현:

- 활성 연간 지시의 보상 편향 태그를 읽어 제품 후보를 점수화한다.
- 같은 태그를 연구 후보와 경쟁 대응 플랜 점수에도 반영한다.
- 회사 화면의 연간 심사 카드 안에 제품, 연구, 경쟁 대응 추천을 표시한다.
- `annual-strategy` QA 시나리오로 신뢰형 지시가 전략실 추천을 만드는 상태를 재현한다.

완료 기준:

- 활성 지시가 없거나 보상 편향 태그가 없으면 전략실 조언을 만들지 않는다.
- `신뢰 복리 프로그램` 지시는 신뢰, 안전, 기업 태그를 표시한다.
- 신뢰형 지시는 고객지원 챗봇, 안전/엔터프라이즈 연구, 관련 경쟁 대응을 추천한다.
- 회사 화면에서 연간 전략실이 표시된다.

### v0.14.9-alpha — 보상 카드 지시 보너스 배지

상태: 완료.

목표: 보상 후보가 연간 지시와 왜 맞는지 카드 단위로 설명한다.

구현:

- 전략 카드와 활성 연간 지시의 보상 편향 태그 교집합을 계산한다.
- 일치 태그를 한글 라벨로 변환해 `지시 보너스` 문구를 만든다.
- 보상 카드에 지시 보너스 배지를 표시한다.
- 보상 편향 QA 시나리오와 덱빌딩 테스트로 카드별 표시 모델을 검증한다.

완료 기준:

- 지시와 태그가 일치하는 보상 카드는 일치 라벨을 반환한다.
- 일치하지 않는 카드는 배지를 표시하지 않는다.
- `reward-bias` 시나리오에서 보너스 배지가 표시된다.

### v0.14.8-alpha — 연간 지시 보상 편향 QA 시나리오

상태: 완료.

목표: 보상 편향 표시를 수동 플레이 없이 브라우저에서 바로 확인할 수 있게 한다.

구현:

- QA 시나리오 목록에 `reward-bias`를 추가한다.
- 1년차 연간 심사 후 `신뢰 복리 프로그램`을 선택한 상태를 만든다.
- 해당 상태에서 카드 보상 3택1을 생성하고 덱 메뉴로 진입한다.
- URL 파싱에서 `?scenario=reward-bias`를 지원한다.

완료 기준:

- `reward-bias` 시나리오가 덱 메뉴로 열린다.
- 활성 지시가 `신뢰 복리 프로그램`이고 보상 편향 태그를 가진다.
- 카드 보상 후보에 신뢰/안전/기업 태그 보상이 포함된다.
- 관련 테스트와 브라우저 QA가 통과한다.

### v0.14.7-alpha — 카드 보상 편향 표시

상태: 완료.

목표: v0.14.6에서 추가한 연간 지시의 카드 보상 편향을 플레이어가 보상 선택 전에 이해할 수 있게 만든다.

구현:

- 활성 연간 지시의 `rewardBiasTags`를 한글 라벨로 요약하는 읽기 모델을 추가한다.
- 카드 보상 3택1이 떠 있을 때 덱 화면에 현재 지시의 보상 편향 안내를 표시한다.
- 신뢰형 지시가 신뢰, 안전, 기업 태그 쪽으로 보상을 기울인다는 설명을 테스트로 고정한다.

완료 기준:

- 덱 UI용 보상 편향 요약 모델이 활성 지시 제목과 태그 라벨을 제공한다.
- 카드 보상 화면에서 연간 지시의 보상 편향 설명이 표시된다.
- 관련 테스트와 빌드가 통과한다.

### v0.14.6-alpha — 연간 지시와 카드 보상 연결

상태: 완료.

목표: 연간 운영 지시가 월간 자원 보너스에만 머물지 않고, 다음 제품 출시 후 받는 카드 보상 방향까지 바꾸게 만든다.

구현:

- 연간 운영 지시 선택 데이터에 `reward_bias_tags`를 추가한다.
- 선택한 지시를 활성 지시로 저장할 때 카드 보상 편향 태그를 함께 보존한다.
- 카드 보상 랭킹이 제품 태그, 리뷰 점수, 보유 카드 수에 더해 연간 지시 태그를 반영한다.
- 저장/로드와 상태 무결성 검사가 `rewardBiasTags`를 다룬다.
- 신뢰형 지시 선택 후 신뢰/안전/기업 태그 카드가 보상 후보 상위에 뜨는지 테스트한다.

완료 기준:

- 선택한 연간 지시에 보상 편향 태그가 저장된다.
- 저장/로드 후에도 보상 편향 태그가 유지된다.
- 카드 보상 3택1이 연간 지시 태그에 따라 달라진다.
- 관련 테스트와 빌드가 통과한다.

### v0.14.5-alpha — 연간 심사 후 3택1 운영 지시

상태: 완료.

목표: 연간 심사를 단순 평가/보상 이벤트가 아니라, 다음 해 회사 운영 방향을 플레이어가 직접 고르는 전략 분기점으로 만든다.

구현:

- `annual_directive_choices.json`에 재사용 가능한 연간 운영 지시 풀을 만든다.
- 심사 통과/미달과 연차 범위에 맞는 후보 3개를 제안한다.
- 플레이어가 선택한 지시가 기본 지시를 대체하고 다음 12개월 월간 효과로 적용된다.
- 회사 화면에서 후보별 월간 효과와 추천 메뉴를 한눈에 비교할 수 있게 한다.
- 24개월 이상 자동 진행하면서 첫 연간 지시 선택까지 검증하는 시뮬레이션 하네스를 추가한다.
- 데이터 검증과 상태 무결성 검사가 선택 풀과 대기 선택 상태를 검사한다.

완료 기준:

- 12개월차 심사 후 3개의 다음 해 운영 지시 후보가 생성된다.
- 선택한 지시가 활성 지시로 반영되고 대기 선택 상태는 정리된다.
- 장기 시뮬레이션이 24개월 이상 진행하며 최소 1회 지시 선택을 처리한다.
- QA 시나리오, 데이터 검증, 빌드가 통과한다.

### v0.14.4-alpha — 연간 심사 후속 지시와 다음 해 모멘텀

상태: 완료.

목표: 연간 심사가 단발 보상으로 끝나지 않고, 다음 해의 운영 방향과 경쟁사 반응으로 이어지게 만든다.

구현:

- 심사 결과에 따라 `annualDirective` 상태를 생성한다.
- 활성 지시의 월간 효과를 경제 계산에 합산한다.
- 지시 결과로 경쟁사 모멘텀과 최근 행동을 갱신한다.
- 회사 화면에 다음 해 지시, 월간 효과, 추천 메뉴, 만료 월을 표시한다.
