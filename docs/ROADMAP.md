# 로드맵 — AI Company Tycoon: Boundaryless

작성일: 2026-05-15

## 현재 상태

현재 버전은 `v0.11.0-alpha`다. 그래픽 에셋을 제외한 핵심 프로토타입은 다음 기반을 갖췄다.

- 첫 제품 개발과 출시.
- 출시 평가, 성장 경로 선택, 경쟁사 압박.
- 10개월 MVP 진행도.
- 업적/런 목표와 보상.
- 성장 경로 월간 효과.
- 제품 업그레이드.
- 저장/불러오기와 깨진 저장 복구.
- 런 결과 랭크와 다음 행동 추천.
- 자동 밸런스 시뮬레이션 하네스.

## 개발 원칙

- 사용자에게 보여 주는 보고서와 로드맵은 한국어로 작성한다.
- 한 번 개선할 때마다 버전을 붙이고 변경 기록을 남긴다.
- 플레이어가 느끼는 재미를 먼저 고정하고, 그래픽은 그 다음에 붙인다.
- 매 버전은 테스트, 데이터 검증, 빌드를 통과해야 한다.
- 큰 기능은 반드시 QA 시나리오나 시뮬레이션 하네스로 반복 검증한다.

## 단기 계획

### v0.11.1-alpha — 한국어 몰입과 위험 피드백

목표: 현재 화면에서 가장 눈에 띄는 몰입 저해 요소를 제거한다.

구현:

- 영어로 남아 있는 이벤트 제목, 본문, 선택지를 한국어화한다.
- 현금이 0 미만일 때 상단 또는 월간 보고에 재무 위험 경고를 표시한다.
- 현금 압박 상태에서 추천 행동을 자동화, 비용 절감, 투자 유치 중 하나로 연결한다.
- `?scenario=commercial`과 `?scenario=arc`에서 경고가 잘 보이는지 확인한다.

완료 기준:

- 이벤트 데이터에 영어 노출이 남지 않는다.
- 현금 위험 경고가 테스트와 브라우저 QA로 검증된다.
- `npm run harness:gate` 통과.

### v0.11.2-alpha — 제품 업그레이드의 재미 강화

목표: 제품 업그레이드가 단순 레벨 증가가 아니라 경영 선택처럼 느껴지게 한다.

구현:

- 제품별 업그레이드 이름을 추가한다.
- 업그레이드 방향을 최소 2종으로 나눈다. 예: 이용자 성장형, 수익화형, 안정성형.
- 업그레이드가 리뷰, 신뢰, 이용자, 매출 중 서로 다른 축에 영향을 주게 한다.
- 제품 카드에 현재 업그레이드 성격을 표시한다.

완료 기준:

- 최소 3개 제품에 업그레이드 선택지가 있다.
- 업그레이드 선택이 월간 보고에 반영된다.
- 제품 업그레이드 테스트와 상용 시뮬레이션 통과.

### v0.11.3-alpha — 자동화와 회복 루프

목표: 실수하거나 현금이 말라도 다시 살아날 방법을 만든다.

구현:

- 재무 위기 상태에서 자동화/비용 절감 중심의 회복 목표를 추가한다.
- 자동화 업그레이드의 체감 효과를 키운다.
- 투자 유치나 긴급 대출 이벤트를 추가한다.
- 실패 직전 상태를 위한 QA 시나리오를 추가한다.

완료 기준:

- 현금 마이너스 상태에서도 회복 경로가 최소 2개 보인다.
- 밸런스 시뮬레이터가 회복 전략을 검증한다.
- 실패/회복 QA 시나리오 추가.

### v0.11.4-alpha — 경쟁사 대응과 시장 사건

목표: 경쟁사가 단순 점수 배경이 아니라 전략적 압박으로 느껴지게 한다.

구현:

- 경쟁사별 카운터 전략을 추가한다.
- 경쟁사가 선점한 제품군에 들어갈 때 패널티/보너스를 더 명확히 만든다.
- 경쟁사 이벤트를 한국어로 확장한다.
- 랭킹 변화가 타임라인과 월간 보고에 더 잘 보이게 한다.

완료 기준:

- 경쟁사별 대응 선택이 최소 3종 이상 있다.
- 경쟁사 선점이 제품 출시 평가에 명확히 반영된다.
- 경쟁 QA 시나리오 통과.

### v0.12.0-alpha — 10분 플레이 알파 완성

목표: 처음 켠 플레이어가 10분 안에 “AI 회사를 키웠다”는 감각을 얻도록 만든다.

구현:

- 첫 10분 흐름을 고정한다.
- 튜토리얼 문구를 줄이고, 행동 버튼 중심으로 안내한다.
- 첫 출시, 첫 성장 선택, 두 번째 제품, 첫 위기, 런 결과까지 자연스럽게 이어지게 조정한다.
- 모바일/좁은 화면에서 회사 패널을 다시 정리한다.

완료 기준:

- 신규 플레이어 기준 첫 30초 목표 이해.
- 10분 안에 첫 제품 출시와 성장 경로 선택 가능.
- `commercial`, `fresh`, `project`, `release`, `strategy`, `arc` QA 시나리오 통과.

### v0.14.0-alpha — 콘텐츠 확장

완료 기준:

- 최소 3개 성장 전략이 서로 다른 플레이 결과를 만든다.
- 합성 플레이테스트에서 “다음 런 이유”가 명확하다는 평가를 받는다.

## 다음 추천 작업

가장 먼저 진행할 작업은 `v0.11.1-alpha — 한국어 몰입과 위험 피드백`이다.

이유:

- 현재 브라우저 QA에서 영어 이벤트가 가장 눈에 띄는 몰입 저해 요소다.
- 현금 마이너스는 좋은 긴장 요소지만, 경고와 회복 버튼이 없으면 불공정하게 느껴질 수 있다.
- 그래픽 작업 전에 텍스트와 상태 피드백을 정리하면 이후 UI/픽셀아트 작업이 덜 흔들린다.

## 현재 차단 요소

- GitHub 원격 저장소가 아직 설정되어 있지 않다. `git remote -v` 결과가 비어 있으므로 push와 Vercel 자동 배포는 원격 연결 이후 진행해야 한다.
- Playwright가 현재 로컬 Node REPL 환경에 없어 자동 스크린샷 QA가 제한된다. 당분간은 Chrome Computer Use와 수동 브라우저 QA를 병행한다.
