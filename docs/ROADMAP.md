# 로드맵 — AI Company Tycoon: Boundaryless

작성일: 2026-05-15

## 현재 상태

현재 버전은 `v0.12.1-alpha`다. 프로젝트 방향은 **카이로소프트식 AI 회사 경영 + 로그라이트 덱빌딩 + 개발 퍼즐**로 전환했다. 그래픽 에셋을 제외한 핵심 프로토타입은 다음 기반을 갖췄다.

- 첫 제품 개발과 출시.
- 출시 평가, 성장 경로 선택, 경쟁사 압박.
- 10개월 MVP 진행도.
- 업적/런 목표와 보상.
- 시작 덱, 손패, 드로우/버림 더미.
- 전략 카드 사용으로 개발 진행도/완성도 변화.
- 플레이어가 직접 고르는 3x3 개발 퍼즐.
- 카드가 퍼즐 난이도, 점수, 선택 칸 수에 영향을 주는 카드-퍼즐 시너지.
- 창업 통찰, 메타 해금, 새 런 시작.
- 성장 경로 월간 효과.
- 제품 업그레이드.
- 저장/불러오기와 깨진 저장 복구.
- 런 결과 랭크와 다음 행동 추천.
- 자동 밸런스 시뮬레이션 하네스.

## 개발 원칙

- 사용자에게 보여 주는 보고서와 로드맵은 한국어로 작성한다.
- 한 번 개선할 때마다 버전을 붙이고 변경 기록을 남긴다.
- 플레이어가 느끼는 재미를 먼저 고정하고, 그래픽은 그 다음에 붙인다.
- 매 버전은 테스트, 데이터 검증, 빌드를 통과해야 한다.
- 큰 기능은 반드시 QA 시나리오나 시뮬레이션 하네스로 반복 검증한다.

## 단기 계획

### v0.12.1-alpha — 카드와 퍼즐의 직접 조작

상태: 완료.

목표: 덱빌딩과 개발 퍼즐을 “보이는 장식”이 아니라 제품 개발의 핵심 손맛으로 만든다.

구현:

- 퍼즐 칸을 플레이어가 직접 선택할 수 있게 한다.
- 카드가 퍼즐 난이도, 선택 가능 칸, 점수 배율에 영향을 주게 한다.
- 퍼즐 점수를 출시 리뷰와 월간 성과에 더 명확히 연결한다.
- `?scenario=deck` 브라우저 QA를 정식 체크리스트에 넣는다.

완료 기준:

- 플레이어가 제품 개발 중 카드 1장과 퍼즐 1회를 직접 사용할 수 있다.
- 퍼즐 결과가 진행도, 완성도, 출시 평가 중 최소 2개 지표에 반영된다.
- 카드-퍼즐 상호작용 테스트가 있다.
- `npm run harness:gate` 통과.

### v0.12.2-alpha — 카드 보상과 덱 편집

목표: 매 런이 다른 회사가 되도록 카드 보상, 제거, 강화 선택을 추가한다.

구현:

- 제품 출시 후 카드 보상 3택1을 제공한다.
- 약한 카드를 제거하거나 강화하는 덱 편집 루프를 추가한다.
- 카드 희귀도와 태그 시너지를 UI에 표시한다.
- 런 결과에 “이번 회사의 대표 카드”를 기록한다.

완료 기준:

- 첫 출시 후 카드 보상을 선택할 수 있다.
- 카드 제거/강화가 저장/불러오기에 보존된다.
- 덱이 너무 커지거나 한 카드만 반복되는 상황을 밸런스 테스트로 점검한다.

### v0.12.3-alpha — 실패 화면과 메타 진행 강화

목표: 망한 런도 “다음엔 더 잘할 수 있다”는 강한 재시작 이유를 만든다.

구현:

- 실패/성공/10개월 종료 화면을 분리한다.
- 런 기록에 대표 제품, 대표 카드, 경쟁사 압박, 실패 원인을 저장한다.
- 창업 통찰 획득식을 화면에 설명한다.
- 메타 해금 화면을 별도 패널로 정리한다.

완료 기준:

- 실패 후 새 런 시작까지 2클릭 이하.
- 플레이어가 어떤 조건 때문에 통찰을 얻었는지 읽을 수 있다.
- 메타 해금이 다음 런의 카드/자원에 반영된다.

### v0.12.4-alpha — 경쟁사와 덱 카운터

목표: 경쟁사가 카드 선택과 제품 개발 전략을 흔드는 압박으로 작동하게 한다.

구현:

- 경쟁사별 카운터 카드 또는 대응 카드 태그를 추가한다.
- 경쟁사가 선점한 제품군의 퍼즐 난이도나 출시 평가를 조정한다.
- 경쟁 이벤트 보상에 카드 획득/제거 선택을 포함한다.
- 랭킹 변화가 덱 전략과 연결되도록 설명한다.

완료 기준:

- 최소 3개 경쟁사가 서로 다른 카드/퍼즐 압박을 만든다.
- 경쟁 대응 선택이 카드 또는 메타 진행에 영향을 준다.
- 경쟁 QA와 덱 QA 시나리오 통과.

### v0.13.0-alpha — 10분 로그라이트 알파 완성

목표: 처음 켠 플레이어가 10분 안에 “한 런을 끝내고 다음 런에서 새 선택지를 써보고 싶다”는 감각을 얻도록 만든다.

구현:

- 첫 10분 흐름을 고정한다.
- 튜토리얼 문구를 줄이고, 행동 버튼 중심으로 안내한다.
- 첫 출시, 첫 카드 사용, 첫 퍼즐, 첫 성장 선택, 첫 위기, 런 결과까지 자연스럽게 이어지게 조정한다.
- 모바일/좁은 화면에서 회사 패널을 다시 정리한다.

완료 기준:

- 신규 플레이어 기준 첫 30초 목표 이해.
- 10분 안에 첫 제품 출시, 카드 사용, 퍼즐 해결, 성장 경로 선택 가능.
- `commercial`, `fresh`, `project`, `deck`, `release`, `strategy`, `arc` QA 시나리오 통과.

## 다음 추천 작업

가장 먼저 진행할 작업은 `v0.12.2-alpha — 카드 보상과 덱 편집`이다.

이유:

- v0.12.1에서 카드와 퍼즐이 연결되었으니, 이제 매 런의 덱이 달라지는 보상 구조가 필요하다.
- 출시 후 카드 보상 3택1이 들어가야 “다음 한 달”과 “다음 런”의 동기가 더 선명해진다.
- 덱 편집이 생기면 미니맥스 플레이어가 파고들 조합 공략의 뼈대가 생긴다.

## 현재 차단 요소

- GitHub 원격 저장소가 아직 설정되어 있지 않다. `git remote -v` 결과가 비어 있으므로 push와 Vercel 자동 배포는 원격 연결 이후 진행해야 한다.
- Playwright가 현재 로컬 Node REPL 환경에 없어 자동 스크린샷 QA가 제한된다. 당분간은 Chrome Computer Use와 수동 브라우저 QA를 병행한다.
