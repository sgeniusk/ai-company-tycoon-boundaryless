# v0.28-alpha 제작 보고서 — 조합형 제품 프로젝트 루프

작성일: 2026-05-17

## 목표

이번 단계의 목표는 v0.27에서 만든 제품 아이디어 조합실을 “보여주는 기능”에서 “실제 게임 진행 기능”으로 끌어올리는 것이다. 플레이어가 소재, 제품 타입, 파격 옵션을 조합하면 그 결과가 정식 개발 프로젝트가 되고, 기존 제품 리뉴얼도 즉시 업그레이드가 아니라 개발 기간과 출시 결과를 거치게 만들었다.

## 구현 요약

- `generatedProducts`를 `GameState`에 추가했다.
- 세이브 버전을 `8`로 올리고 생성 제품 저장/불러오기 복구를 추가했다.
- `createProductDefinitionFromConcept()`를 추가해 조합 결과를 정식 제품 데이터로 변환한다.
- `startProductConceptProject()`를 추가해 조합형 제품 개발 프로젝트를 시작한다.
- `startProductRenewalProject()`를 추가해 기존 제품을 리뉴얼 프로젝트로 개발한다.
- 리뉴얼 프로젝트는 `releaseName`을 사용해 `AI 글쓰기 비서 v2` 같은 출시명으로 결과를 남긴다.
- 월간 경제, 출시 리뷰, 카드 보상, 덱 퍼즐, 런 요약, 사무실 HUD, 제품 메뉴가 생성 제품을 정식 제품처럼 읽도록 조정했다.
- 제품 메뉴에 `조합 개발 시작`과 `리뉴얼 개발` 버튼을 추가했다.

## 플레이 변화

- 이제 “전기차 + 차량 + 초개인화” 같은 조합을 실제 프로젝트로 개발할 수 있다.
- 출시 전에는 활성 제품에 들어가지 않고 개발 중 프로젝트로 보인다.
- 완성 후에는 활성 제품, 리뷰, 카드 보상, 매출/이용자/데이터 루프에 들어간다.
- 기존 제품은 한 번 출시하고 끝나는 것이 아니라 리뉴얼 프로젝트를 통해 v2, 리뉴얼, 파생 라인으로 성장한다.

## 하네스 에이전트 리뷰

### Executive Producer Agent

- 판정: 통과
- 사유: v0.27의 P1 백로그였던 “후보 표시만 됨” 문제가 닫혔다. 10분 MVP에서 제품 선택의 결과가 실제 진행으로 이어진다.

### Game Designer Agent

- 판정: 통과
- 사유: 새 제품 개발과 대표 제품 리뉴얼 사이에 선택지가 생겼다. AI 회사가 산업을 넘나드는 핵심 판타지가 더 직접적으로 플레이에 반영된다.

### Systems Architect Agent

- 판정: 통과
- 사유: 생성 제품을 상태에 보관하고, 기존 정적 제품 목록과 합치는 `getAvailableProductDefinitions()`를 단일 조회 경로로 둔 점이 확장에 유리하다.

### QA Agent

- 판정: 통과
- 확인: 조합형 제품 출시, 리뉴얼 출시, 생성 제품 저장 복구 테스트 통과. 데이터 검증과 빌드 통과.

### Balance Agent

- 판정: 조건부 통과
- 사유: 조합형 제품 비용과 요구 역량은 아직 1차 공식이다. 후속 버전에서 고점 조합과 리뉴얼 비용 곡선을 시뮬레이션해야 한다.

### UX Agent

- 판정: 통과
- 사유: 제품 메뉴에서 조합 결과를 바로 개발 시작할 수 있어 기능의 의도가 명확해졌다. 다만 조합실이 길어지는 문제는 접힘/상세 패널로 한 번 더 압축할 필요가 있다.

### Retention / LTV Agent

- 판정: 통과
- 사유: 같은 회사라도 조합형 제품 선택에 따라 다른 포트폴리오가 생긴다. 반복 런의 빌드 다양성에 직접 기여한다.

### Shareability Agent

- 판정: 통과
- 사유: “AI 커피 프랜차이즈”, “초개인화 전기차” 같은 출시명이 실제 타임라인과 리뷰에 남아 스크린샷 소재가 된다.

### Solo Dev Scope Agent

- 판정: 통과
- 사유: 새 조합 결과를 별도 에셋 없이 데이터/상태 확장으로 처리했다. 콘텐츠 폭은 넓히되 코드 부담은 비교적 작다.

## 백로그

- P1: 조합형 제품과 리뉴얼 프로젝트의 비용/매출 밸런스를 장기 시뮬레이션에 포함한다.
- P1: 제품 조합실을 접힘형 또는 탭형으로 압축해 한 화면 게임 UI에 더 맞춘다.
- P2: 생성 제품명 변형 규칙과 출시 카피를 더 다양화한다.
- P2: 리뉴얼 방식별 효과 차이를 제품 레벨 보너스에 반영한다.
- P3: 조합형 제품을 즐겨찾기/후보 보관함에 넣는 기능을 검토한다.
