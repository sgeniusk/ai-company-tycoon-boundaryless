# v0.32-alpha 제작 보고서 — 로그라이트 재시작 설계실

작성일: 2026-05-17

## 목표

이번 단계의 목표는 한 판이 끝나거나 실패 직전에 “다음 판은 어떻게 다르게 갈까?”가 바로 보이게 만드는 것이다. 단순히 새 런 버튼만 두지 않고, 이번 런의 대표 제품, 대표 카드, 위험 신호, 창업 통찰 보상, 추천 메타 해금, 시작 덱을 한 화면에서 비교하도록 만들었다.

## 구현 요약

- `getNextRunSetupPlan()` 추가.
  - 현재 런 번호, 다음 런 번호, 현재 통찰, 예상 보상, 시작 가능 통찰을 계산한다.
  - 현금 흐름 붕괴, 신뢰도 위험, 자동화 부족, 첫 출시 지연, 경쟁사 모멘텀 과열을 회복 경고로 만든다.
  - 대표 제품, 대표 카드, 최대 압박 경쟁사를 요약한다.
- 메타 해금 추천을 카테고리화했다.
  - 품질/신뢰
  - 성장/리텐션
  - 운영 자동화
  - 경쟁/기업
  - 하드웨어/연구
  - 경계 확장
- 덱 메뉴 상단에 `다음 런 설계실`을 추가했다.
  - 안전 재시작
  - 추천 해금
  - 현재 덱 유지
- `restart-setup` QA 시나리오를 추가했다.
- 도우미 캐릭터 `미나`가 10개월 이후 첫 덱 메뉴 진입 때 다음 런 설계실을 안내한다.

## 플레이 변화

- 실패한 런도 그냥 끝나는 것이 아니라 “현금 흐름이 무너졌으니 자동화로 복구”, “신뢰가 낮으니 평가 하네스를 열자” 같은 다음 판 목적을 만든다.
- 성공 런은 성장/런칭 계열 해금으로 다음 확장 런을 준비할 수 있다.
- 시작 덱 선택이 흩어진 버튼이 아니라 빠른 시작의 일부로 읽힌다.
- 로그라이트 메타가 추상 보상이 아니라 새 회사의 첫 전략 결정처럼 보인다.

## 하네스 에이전트 리뷰

### Executive Producer Agent

- 판정: 통과
- 사유: v0.32 목표였던 재시작 동기 강화가 덱 메뉴 상단의 실제 선택으로 구현됐다.

### Game Designer Agent

- 판정: 통과
- 사유: 실패 원인과 다음 런 선택이 연결되어 “망했지만 배운다”는 로그라이트 감각이 생겼다.

### Systems Architect Agent

- 판정: 통과
- 사유: 추천 계산은 `getNextRunSetupPlan()` 순수 함수로 분리했고, UI는 계산 결과만 렌더링한다.

### QA Agent

- 판정: 통과
- 확인: 집중 테스트, 전체 하네스, Browser DOM QA, 데스크톱/모바일 스크린샷 QA를 통과했다.

### Balance Agent

- 판정: 조건부 통과
- 사유: 추천 해금은 현재 점수 기반이라 안전하다. 다만 장기적으로 특정 해금이 항상 1순위가 되는지 확인해야 한다.

### UX Agent

- 판정: 조건부 통과
- 사유: 데스크톱에서는 첫 화면에 보인다. 모바일에서는 가로 넘침은 없지만 덱 메뉴 내부 스크롤이 필요하다.

### Retention / LTV Agent

- 판정: 통과
- 사유: 새 런을 누르기 전에 보존할 빌드, 새로 열 해금, 복구 목표가 보여 재도전 이유가 명확해졌다.

### Shareability Agent

- 판정: 통과
- 사유: `복구 런 설계`, `현금 흐름 붕괴`, `런칭 플레이북 해금`처럼 스크린샷으로 상황이 설명된다.

### Solo Dev Scope Agent

- 판정: 통과
- 사유: 새 에셋 없이 기존 메타 해금, 시작 덱, 런 기록 데이터를 재조합해 반복 플레이 가치를 높였다.

### Deck System Engineer

- 판정: 통과
- 사유: 시작 덱과 해금 카드가 다음 런 선택으로 직접 이어진다.

### Roguelite Meta Engineer

- 판정: 통과
- 사유: 통찰 보상, 해금 지출, 스타터 덱 선택이 하나의 재시작 루프로 묶였다.

### Balance Simulation Engineer

- 판정: 조건부 통과
- 사유: 무한 자원 체인은 없다. 다음 단계에서는 10년 시뮬레이션에서 해금 추천 분포를 기록해야 한다.

## 백로그

- P1: 모바일 덱 메뉴에서 다음 런 설계실 요약을 더 위로 끌어올리거나 별도 접힘 요약을 둔다.
- P1: 해금 추천 분포를 장기 시뮬레이션에 기록해 특정 메타 해금 독주를 점검한다.
- P2: 빠른 시작을 누르기 전 “이번 선택으로 추가되는 카드” 미리보기를 표시한다.
- P2: 다음 런 시작 전 창업 배경이나 초반 사무실 선택 후보를 붙인다.
