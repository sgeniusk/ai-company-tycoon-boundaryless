# v0.21-alpha 제작 보고서 — 20인 검증과 보조 패널 압축

작성일: 2026-05-17

## 목표

`v0.20-alpha`를 실제 평가 가능한 빌드로 만들었으므로, 이번 단계는 평가와 화면 밀도를 동시에 개선하는 것이었다. 핵심은 세 가지다.

- 20인 페르소나 테스트를 데이터와 코드 하네스로 승격한다.
- 우측 보조 정보가 웹페이지처럼 길게 쌓이지 않도록 탭 구조로 압축한다.
- 공개 알파 빌드에서 큰 JS 청크 경고가 줄어들도록 Vite 청크 분리를 시작한다.

## 구현

### 20인 페르소나 하네스

- `data/playtest_personas.json`을 12명에서 20명으로 확장했다.
- 남성 10명, 여성 10명 균형을 데이터 검증 규칙에 반영했다.
- 각 페르소나에 `benchmark`, `concern`을 추가해 단순 역할명이 아니라 평가 관점과 비교 대상을 함께 보게 했다.
- `runPersonaPlaytestReview()`를 추가해 v0.21 알파 후보를 점수, 판정, 우선순위, 페르소나별 메모로 요약한다.

### QA 시나리오

- `?scenario=persona20`을 추가했다.
- 브라우저에서 열면 로그 메뉴가 바로 열리고, 타임라인 상단에 `v0.21 20인 페르소나` 결과와 핵심 개선 우선순위가 표시된다.

### 우측 보조 패널 압축

- 기존 우측 보조 패널을 `목표 / 회사 / 월간 / 결과` 탭 구조로 바꿨다.
- 출시 결과나 10년 엔딩, 런 결과가 생기면 결과 탭으로 자동 이동한다.
- 평소에는 목표 탭 중심으로 시작해 사무실 장면과 보조 정보가 서로 덜 경쟁하게 했다.

### 빌드 설정

- `vite.config.ts`에 `manualChunks`를 추가했다.
- React 계열은 `react-vendor`, JSON 데이터는 `game-data` 청크로 분리한다.

## 하네스 에이전트 판단

- Executive Producer: v0.21은 큰 기능 확장보다 플레이테스트 가능성을 높이는 후속 버전으로 적절하다.
- UX Agent: 우측 패널 탭화로 한 화면 게임감이 강해졌다. 다음은 메뉴 패널 내부의 긴 회사 개요를 더 줄여야 한다.
- Synthetic Playtester Agent: 20인 페르소나가 코드 하네스에 연결되어 반복 검증 가능한 형태가 되었다.
- Solo Dev Scope Agent: 페르소나 보고서는 데이터 기반이라 유지 부담이 낮다.
- Retention / LTV Agent: 첫 5분 보상 연출이 아직 다음 우선순위로 남아 있다.
