# v0.30-alpha 제작 보고서 — 도우미 캐릭터 튜토리얼

작성일: 2026-05-17

## 목표

이번 단계의 목표는 새 기능이 늘어날수록 플레이어가 “어디를 눌러야 하는지”를 잃지 않게 만드는 것이다. 웹페이지 설명문을 더 붙이는 대신, 게임 화면 안에서 도우미 캐릭터 `미나`가 처음 열린 시스템을 짧게 소개하고 관련 메뉴로 데려가도록 만들었다.

## 구현 요약

- `TutorialGuide` 타입과 `getTutorialGuide()` 순수 선택 함수를 추가했다.
- `seenTutorials`를 `GameState`에 추가하고 세이브 버전을 9로 올렸다.
- `dismissTutorialGuide()`로 읽음 처리를 중앙 시뮬레이션 상태에 기록한다.
- 저장/불러오기와 로그라이트 새 런에서도 이미 본 튜토리얼이 유지된다.
- 안내 트리거를 7개로 구성했다.
  - 차고 창업 시작
  - 첫 팀원 고용
  - 제품 아이디어 조합실
  - 개발 프로젝트 시작
  - 출시 보상 카드
  - 사무실 성장 플래너
  - 경쟁사 압박
- `App.tsx`에 고정 도우미 말풍선을 추가했다.
- 오프라인 정산 모달과 도우미 안내가 겹치지 않게 처리했다.

## 플레이 변화

- 첫 화면에서 “에이전트 고용”으로 자연스럽게 이어진다.
- 새 기능이 등장해도 우측 콘솔을 뒤지는 부담이 줄어든다.
- 반복 플레이에서는 이미 본 안내가 다시 뜨지 않아 로그라이트 재시작 흐름을 방해하지 않는다.
- 미나의 초상과 말풍선이 화면 안에 고정되어 게임 내 튜토리얼처럼 보인다.

## 하네스 에이전트 리뷰

### Executive Producer Agent

- 판정: 통과
- 사유: 신규 시스템이 많아진 현재 빌드에서 첫 10분 MVP 진입 장벽을 낮춘다.

### Game Designer Agent

- 판정: 통과
- 사유: 안내가 제품, 덱, 사무실, 경쟁이라는 핵심 루프를 순서대로 노출한다.

### Systems Architect Agent

- 판정: 통과
- 사유: 튜토리얼 선택은 순수 함수, 읽음 처리는 `GameState` 단일 출처에 둬 확장 가능하다.

### QA Agent

- 판정: 통과
- 확인: 튜토리얼 순서, 저장 복원, 새 런 보존, UI 계약, 전체 하네스 통과.

### Balance Agent

- 판정: 통과
- 사유: 경제 수치에는 영향이 없고 초반 행동 선택만 선명하게 만든다.

### UX Agent

- 판정: 통과
- 사유: 화면 스크롤 없이 하단 말풍선에서 기능 소개와 메뉴 이동이 가능하다.

### Synthetic Playtester Agent

- 판정: 통과
- 사유: 신규 플레이어와 5분 플레이어가 첫 행동을 이해하기 쉬워졌다.

### Retention / LTV Agent

- 판정: 통과
- 사유: 첫 세션 이탈을 줄이고, 이미 본 안내를 반복하지 않아 재시작 피로를 낮춘다.

### Shareability Agent

- 판정: 조건부 통과
- 사유: 도우미 캐릭터가 생겨 화면 기억점은 늘었다. 단, 추후 감정 표현과 이벤트 컷인은 더 필요하다.

### Solo Dev Scope Agent

- 판정: 통과
- 사유: 새 그래픽 에셋 없이 CSS 초상과 데이터 규칙으로 구현했다.

### Deck System Engineer

- 판정: 통과
- 사유: 카드 보상 안내가 덱 메뉴로 직접 연결되어 덱 시스템 발견성이 좋아졌다.

### Roguelite Meta Engineer

- 판정: 통과
- 사유: 새 런에서도 본 튜토리얼을 유지해 반복 플레이의 마찰을 줄였다.

### Puzzle Mechanics Engineer

- 판정: 보류
- 사유: 개발 프로젝트 안내에서 퍼즐을 언급하지만, 퍼즐 튜토리얼은 별도 단계가 더 필요하다.

### Balance Simulation Engineer

- 판정: 통과
- 사유: 수치 변경 없이 상태 저장만 추가했으며 하네스 시뮬레이션이 통과했다.

## 백로그

- P1: 첫 10분 루프 전용 튜토리얼 목표표를 미나 안내와 연결한다.
- P1: 카드 보상, 개발 퍼즐, 리뉴얼 출시 안내를 더 세분화한다.
- P2: 미나 초상 상태를 상황별 표정/제스처로 분리할 수 있는 에셋 슬롯을 마련한다.
- P2: 이미 본 안내를 다시 볼 수 있는 도움말 로그를 기록 메뉴에 추가한다.
- P3: 경쟁사 대형 사건과 제품 출시 순간에 도우미 리액션을 추가한다.
