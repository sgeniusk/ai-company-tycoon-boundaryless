# v0.29-alpha 제작 보고서 — 사무실 성장 플래너

작성일: 2026-05-17

## 목표

이번 단계의 목표는 사무실 확장과 장식 배치가 단순 상점 기능처럼 보이지 않게 만드는 것이다. 플레이어가 “지금은 사람을 더 뽑기 위해 확장해야 하는가, 지역을 옮겨야 하는가, 장식 시너지를 완성해야 하는가”를 한 화면에서 판단할 수 있도록 성장 플래너를 추가했다.

## 구현 요약

- `getOfficeGrowthPlan()` 추가
  - 현재 사무실 단계, 고용 슬롯, 장식 슬롯, 월간 효과 계산
  - 다음 사무실 확장 후보와 증가 슬롯 계산
  - 다음 지역 이전 후보와 AI 운용/월비 변화 계산
  - 활성 사무실 시너지와 다음 시너지 후보 계산
  - 다음 시너지 완성에 필요한 장식 추천
  - 현재 병목에 맞는 우선 행동 추천
- 상점 콘솔에 `사무실 성장 플래너` 패널 추가
  - 다음 확장과 다음 지역 이전을 같은 줄에서 비교
  - 다음 장식 조합의 진행 조건 표시
  - 추천 장식을 바로 구매하거나 배치 가능
- 타입 확장
  - `OfficeGrowthPlan`
  - `OfficeGrowthExpansionChoice`
  - `OfficeGrowthRelocationChoice`
  - `OfficeGrowthDecorRecommendation`
  - `OfficeGrowthPrimaryAction`

## 플레이 변화

- 팀 수용 인원이 꽉 차면 사무실 확장이 명확한 우선 행동으로 뜬다.
- 장식 슬롯이 남아 있으면 다음 시너지를 완성할 장식을 추천한다.
- 사무실 확장과 지역 이전이 같은 성장 선택지로 비교된다.
- 장식 구매가 “효과 숫자만 보고 사는 것”에서 “다음 공간 조합을 완성하는 선택”으로 바뀐다.

## 하네스 에이전트 리뷰

### Executive Producer Agent

- 판정: 통과
- 사유: 사무실 성장 루프가 상점 하위 기능에서 회사 성장 의사결정으로 올라왔다.

### Game Designer Agent

- 판정: 통과
- 사유: 고용 한도, 장식 슬롯, 지역 이전, 시너지 완성이 서로 다른 성장 압박을 만든다.

### Systems Architect Agent

- 판정: 통과
- 사유: 사무실 성장 판단은 `simulation.ts`의 순수 계산 함수에 두고, UI는 결과 표시와 버튼 실행만 담당한다.

### QA Agent

- 판정: 통과
- 확인: 성장 플래너 테스트와 상점 콘솔 레이아웃 계약 테스트 통과.

### Balance Agent

- 판정: 조건부 통과
- 사유: 추천 점수는 1차 공식이다. 후속 장기 시뮬레이션에서 특정 장식만 과도하게 추천되는지 봐야 한다.

### UX Agent

- 판정: 통과
- 사유: 다음 행동이 `사무실 확장`, `추천 장식 구매`, `지역 이전`처럼 명확한 문장으로 나온다.

### Retention / LTV Agent

- 판정: 통과
- 사유: 사무실이 커지는 과정이 제품 출시 사이의 중간 목표가 된다.

### Shareability Agent

- 판정: 보류
- 사유: 시스템은 좋아졌지만 시각적으로 “사무실이 변했다”는 장면은 아직 더 강화해야 한다.

### Solo Dev Scope Agent

- 판정: 통과
- 사유: 새 에셋 없이 기존 데이터에서 추천과 비교를 계산해 체감 가치를 올렸다.

## 백로그

- P1: 사무실 확장 단계별 오브젝트 밀도와 배치 변화를 더 강하게 보이게 한다.
- P1: 장식 교체를 자동 추천하거나 원클릭으로 스왑하는 기능을 검토한다.
- P2: 추천 장식 점수를 장기 밸런스 시뮬레이션에 포함한다.
- P2: 지역 이전과 사무실 확장의 월간 손익 차이를 더 읽기 쉽게 표시한다.
- P3: 사무실 성장 플래너를 회사 메뉴에도 축약 표시한다.
