# v0.34.14-alpha 제작 보고서 — 인사 후폭풍의 제품 개발 영향

작성일: 2026-05-18

## 목표

v0.34.13에서 인사 사건을 방치하면 직원 수치, 자원, 경쟁사 모멘텀이 흔들리게 됐다. 하지만 제품을 개발 중인 회사 입장에서는 가장 체감되는 손실이 “지금 만들던 제품이 망가진다”는 점이다. 이번 패스는 미해결 인사 사건을 제품 일정과 완성도 손실로 연결해, 월 넘기기 전 인사 대응의 우선순위를 더 분명하게 만드는 것이다.

## 이번 구현

- 미해결 인사 사건 후폭풍이 현재 배치된 제품 프로젝트를 찾아 진행도와 완성도를 깎는다.
- 번아웃, 스카우트, 계약 불만의 프로젝트 손실값을 `data/balance.json`에 추가했다.
- 후폭풍 기록에 `projectImpactLabel`을 추가해 어떤 제품이 얼마나 손상됐는지 남긴다.
- 월간 보고에 `인사 후폭풍` 요약을 추가했다.
- 하단 자원 변화량 계산에 후폭풍 자원 손실도 포함했다.
- 사무실 중앙 경보가 최근 월간 후폭풍을 우선 노출한다.
- 에이전트 콘솔의 최근 후폭풍 카드에 프로젝트 영향 라벨을 추가했다.
- `staff-aftermath` QA 시나리오가 프로젝트 손실과 월간 보고 요약까지 검증하도록 확장됐다.

## 하네스 에이전트 검토

- Executive Producer Agent: P1 해결. “방치하면 실제 개발 일정이 밀린다”는 결과가 생겨 다음 달 진행의 긴장감이 커졌다.
- Game Designer Agent: P1 해결. 돈을 아끼고 사건을 넘기는 선택이 단순 페널티가 아니라 출시 품질 손실로 이어진다.
- Systems Architect Agent: P2 양호. 손실 수치는 JSON 밸런스 데이터로 이동했고, 기존 `GameState`와 월간 보고 구조를 확장했다.
- QA Agent: P1 해결. 프로젝트 손실, 월간 보고, QA 시나리오, 레이아웃 계약을 테스트로 고정했다.
- Balance Agent: P2 관찰. 진행도 손실은 월간 기본 진행량보다 작게 둬 치명상보다는 일정 압박으로 작동한다.
- UX Agent: P2 해결. 사무실 경보, 월간 보고, 에이전트 콘솔에서 같은 원인을 반복 노출해 손실 이유를 추적할 수 있다.
- Retention / LTV Agent: P2 양호. 직원 케어 실패가 런의 이야기로 남아 다음 판에서 더 나은 운영을 시도하게 만든다.
- Shareability Agent: P2 양호. “프로젝트 완성도 손실”이 기록으로 남아 스크린샷 설명력이 좋아졌다.
- Solo Dev Scope Agent: P2 양호. 별도 사고 시스템을 만들지 않고 기존 인사 사건 후폭풍에 얹었다.
- Deck System Engineer: P3 관찰. 다음 단계에서 인사 리스크를 줄이는 카드나 직원 케어 카드가 있으면 덱빌딩 선택과 더 강하게 연결된다.
- Roguelite Meta Engineer: P3 관찰. 방치로 손상된 프로젝트가 실패 런의 회고 문구나 다음 런 통찰로 이어지면 반복 플레이 가치가 커진다.

## 남은 작업

- 프로젝트별 위험도를 제품 메뉴에서 미리 예보하기.
- 인사 사건 대응 카드, 복지 아이템, 사무실 장식이 후폭풍 손실을 완화하도록 연결하기.
- 월간 보고를 “돈/사용자/품질/인사/경쟁” 5개 라인으로 더 게임스럽게 재정리하기.
- 인사 후폭풍을 20인 페르소나 테스트에 다시 넣어 스트레스 수위를 확인하기.
