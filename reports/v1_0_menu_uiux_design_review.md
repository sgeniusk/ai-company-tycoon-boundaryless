# 디자인 검토 — 게임 메뉴 IA + 오피스 우선 UI/UX (v1.0 재설계)

작성일: 2026-06-02
작성자: Claude Code (디자인 검토 패스 / 편집장)
승인 대기: 사용자 검토 후 구현. visual/additive — 시뮬·틱·세이브·데이터·경제 로직 불변, 메뉴 **내용은 재사용** 위치만 이동.

## 요약 (핵심 결정)

- 우측 상시 `menu-layout` 컬럼 제거 → 메뉴는 **하단 런처 바에서 팝업**으로. 오피스가 기본 전체 폭.
- **운영 / 꾸미기 / 기록을 IA에서 명확히 분리**.
  - **운영** = 턴·진행 흐름. 항상 보이는 "다음 행동" 칩(가이드 1개) + 런처 '운영' 그룹.
  - **꾸미기** = 오피스 장식. 지금 `상점`에 에이전트 장비와 섞여 묻힘 → **오피스 전용 별도 버튼**(오피스 모서리)으로 분리, 상점(구매)과 구분.
  - **기록** = 단일 타임라인 → **타임라인 / 도감 / 업적** 서브탭으로 세부화.
- 팝업은 **v0.98 오버레이 인프라 재사용**(닫기/Esc/초기 포커스/어포던스/reduced-motion), 한 번에 1개.

## 진단 (코드 근거)

- `app-shell` 그리드 `"top top top" / "stage stage menu" / "resources commands menu"` — `menu-layout`(우측)이 항상 한 메뉴 콘텐츠를 큰 패널로 점유(스크린샷 "회사 단계…").
- 오피스 위 오버레이 — `office-hud`(GameChrome:984), `OfficeActionSlots`(:1000/1991: 고용→agents·개발→products·전략→deck·**꾸미기→shop**), `OperationCommandPanel`(:1007), `RivalIncidentBanner`(:1006).
- `stage-side`(:1101) — 가이드/런치임팩트/디브리프 등 보조 패널.
- **`ShopPanel`("아이템 상점", MenuPanels:3178)이 에이전트 장비 + 사무실 물건(장식·구획·zone)을 한 메뉴에 혼재** → 꾸미기가 상점에 묻힘.

## 제안 메뉴 IA

기존 8메뉴(menu.ts)의 **내용은 유지**, 접근/그룹/세부화만 재구성.

| 그룹 | 항목 | 접근 | 내용(기존 패널 재사용) | 변경 |
|------|------|------|----------------------|------|
| 운영(core) | 회사 | 런처 | CompanyPanel(단계/승급/지표) | 그대로 |
| 운영 | 제품 | 런처 + "개발" | ProductsPanel(출시) | 그대로 |
| 운영 | 덱 | 런처 + "전략" | DeckPanel(런/퍼즐 카드) | 그대로 |
| 성장(operations) | 에이전트 | 런처 + "고용" | AgentsPanel(능력치/고용) | 그대로 |
| 성장 | 연구 | 런처 | ResearchPanel(AI 능력) | 그대로 |
| 성장 | 상점 | 런처 | ShopPanel **장비/구매 뷰** | 오피스 물건은 꾸미기로 |
| 시장(meta) | 경쟁 | 런처 | CompetitionPanel(시장/라이벌) | 그대로 |
| 시장 | **기록** | 런처 | **타임라인/도감/업적 서브탭** | 세부화 |
| (오피스 전용) | **꾸미기** | **오피스 버튼** | ShopPanel **사무실 물건/구획/배치 뷰** | 상점에서 분리 |

- **꾸미기 분리**: ShopPanel을 `상점`(장비·구매)과 `꾸미기`(사무실 물건 배치·구획·zone) **두 진입점/뷰**로. 콘텐츠는 이미 ShopPanel 안에 둘 다 있으니(officeItems/placedOfficeItems/zonePlan vs agent items) **렌더 분기만** — 게임 로직 불변. 꾸미기는 오피스를 직접 바꾸므로 오피스 전용 버튼이 맥락상 맞음.
- **기록 세부화**: TimelinePanel(타임라인) + 도감(discoveredArchetypeIds/discoveredPayoffIds, 기존 collection-lite) + 업적(unlockedAchievements) → `기록` 팝업의 3 서브탭. 콘텐츠 재사용.

## 하단 런처 바

- `menu.ts`의 3그룹(운영/성장/시장)을 **구분선/라벨로 시각 분리**. 각 버튼 = 아이콘(menuIconIds) + 라벨.
- 클릭 → `setActivePopupMenu(id)`. 현재 활성 팝업 버튼은 눌림 상태 표시.
- 모바일 = 가로 스크롤(그룹 구분 유지), 또는 2줄 래핑.

## 운영 vs 꾸미기 (분리)

- **운영** — 좌하단 "다음 행동" 칩 1개(가이드 최상위 추천). 클릭 시 해당 운영 팝업(회사/제품/덱). 기존 `OperationCommandPanel`+가이드를 이 칩으로 축약. **턴 진행의 핵심 흐름**.
- **꾸미기** — 오피스 우상 모서리 **전용 버튼 🎨**. 클릭 시 꾸미기 팝업(사무실 물건 배치/구획). **오피스를 직접 꾸미는 행동**이라 오피스 컨텍스트에 둠. 운영과 위치·아이콘·의미 모두 구분.
- 기존 4슬롯(고용/개발/전략/꾸미기) 중 고용/개발/전략은 런처가 흡수(중복 제거), 꾸미기만 오피스 전용 버튼으로 승격.

## 팝업 시스템

- 신규 `MenuPopupModal` — v0.98 패턴 재사용: `role="dialog"` `aria-modal`, 닫기 버튼 + Esc + 초기 포커스 + hover/active/focus-visible + reduced-motion. 한 번에 1개.
- 상태: App.tsx `activePopupMenu: MenuId | "decor" | null`(기본 null = 오피스 클리어). 모든 `onOpenMenu`/`setActiveMenu` 콜사이트 → `setActivePopupMenu`. 닫기/Esc = null. **새 GameState 필드 없음**.
- 콘텐츠 = `renderMenuContent(activePopupMenu, …)`(decor는 ShopPanel의 오피스 뷰). 사이즈: 데스크톱 중앙 카드(min(900px,90vw)), 모바일 풀시트.

## 가이드/온보딩

- `getTutorialGuide(gameState, activeMenu)`가 activeMenu 의존 → 팝업 닫힘(null)일 때 기본 컨텍스트(예 "company")로 가이드 산출. 추천 행동은 "다음 행동" 칩에 표시, 클릭 시 해당 팝업 오픈(기존 targetMenu 라우팅 재사용).
