# v0.24-alpha 제작 보고서 — 게임 화면형 레이아웃 압축

작성일: 2026-05-17

## 목표

이전 빌드는 기능은 늘었지만 화면 구조가 웹페이지처럼 세로로 펼쳐지는 인상이 강했다. v0.24는 “한 화면 안에서 회사를 운영하는 게임”처럼 보이도록 PC 기준 HUD, 사무실, 메뉴 패널을 다시 배치하는 데 집중했다.

## 구현

- 왼쪽 세로 자원판을 제거하고 하단 자원 HUD로 이동했다.
- 명령 버튼을 하단 고정 컨트롤 스트립으로 압축했다.
- 중앙 사무실 영역을 더 크게 확보했다.
- 우측 메뉴는 고정 게임 패널 안에서만 스크롤하도록 유지했다.
- 태블릿/모바일 폭에서도 페이지 전체가 길게 흐르지 않도록 `100dvh` 기반 고정 뷰포트를 적용했다.
- 좁은 화면에서 가로 잘림이 생기지 않도록 핵심 그리드 자식에 `min-width: 0`을 명시했다.

## 하네스 에이전트 판단

- UX Agent: PC 기준으로 한 화면 안에서 사무실, 결과, 메뉴가 함께 보인다. 웹페이지 느낌이 줄었다.
- Game Designer Agent: 자원과 명령이 하단 HUD로 내려가 플레이 화면의 중심이 사무실로 돌아왔다.
- Solo Dev Scope Agent: 컴포넌트 구조를 크게 갈아엎지 않고 CSS 레이아웃 계약을 먼저 고정해 유지 부담이 낮다.
- QA Agent: 데스크톱과 모바일 캡처를 확인했다. 모바일은 임시 압축안이며, 추후 전용 하단 탭/접이식 메뉴가 필요하다.

## 남은 작업

- 모바일은 아직 “쾌적한 모바일 게임 UI”가 아니라 “잘리지 않는 압축 화면” 단계다.
- 우측 메뉴는 다음 단계에서 아이콘형 탭과 상세 패널 분리로 더 게임답게 줄여야 한다.
- 사무실 화면 안에 직원/프로젝트 상태를 더 많이 직접 표시해 메뉴 의존도를 낮춘다.
