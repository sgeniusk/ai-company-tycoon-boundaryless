# Alpha v0.4.1 인력 조합 시너지 제작 보고서

작성일: 2026-05-17  
범위: `v0.4-alpha — 회사 운영과 공간 성장` 2차  
목표: 사람 직원, AI 에이전트, 로봇 인력이 서로 다른 역할을 갖고, 섞어 고용했을 때 프로젝트 성과가 달라지게 만든다.

## 구현 요약

- `data/workforce_synergies.json`를 추가해 인력 조합 시너지를 데이터로 분리했다.
- `getWorkforceSynergySummary`를 추가해 현재 팀 또는 프로젝트 투입 팀 기준으로 활성 조합, 다음 후보, 프로젝트 보너스를 계산한다.
- 제품 개발 예측과 실제 월간 개발 진행에 인력 조합 보너스를 반영했다.
- 에이전트 화면 상단에 `팀 조합` 패널을 추가해 현재 조합 상태와 다음 후보를 표시했다.
- 데이터 검증 하네스가 인력 조합 시너지의 필수 필드, 요구 인력 종류, 스탯 효과, 프로젝트 효과를 검사한다.

## 추가된 조합

- `사람-AI 감독 루프`: 사람 직원이 방향을 잡고 AI 에이전트가 반복 작업을 맡아 진행과 완성도를 보조한다.
- `사람-로봇 현장 운영`: 물리 산업과 제조/테스트 루프에 강한 조합이다.
- `AI-로봇 자동화 파이프라인`: AI 계획과 로봇 실행을 묶어 자동화형 프로젝트 속도를 올린다.
- `풀스택 회사 셀`: 사람, AI, 로봇이 모두 있는 후반 산업 확장형 독립 팀이다.

## 에이전트 리뷰

### Executive Producer Agent

- P0/P1 없음.
- 0.4의 “회사 운영과 공간 성장” 중 인력 역할 차별화가 실제 프로젝트 성과에 연결됐다.

### Game Designer Agent

- P1 없음.
- 사람은 AI 운용과 감독, AI는 반복 작업, 로봇은 물리/자동화 확장이라는 역할 차이가 더 선명해졌다.
- 다음에는 조합별 전용 사건이나 제품군 보너스를 붙이면 좋다.

### Systems Architect Agent

- P0/P1 없음.
- 인력 조합은 JSON 데이터와 순수 계산 함수로 분리되어 확장 가능하다.
- 프로젝트 예측과 실제 진행이 같은 계산 경로를 공유한다.

### QA Agent

- P0/P1 없음.
- 단위 테스트, 관련 회귀, 데이터 검증, 빌드, 브라우저 렌더링을 통과했다.
- 남은 P2: 팀 조합 후보 진행 라벨의 `human`, `ai_agent` 같은 내부 키를 한국어 라벨로 바꿀 필요가 있다.

### Balance Agent

- P1 없음.
- 보너스는 조합을 맞출 때만 적용되어 무조건 특정 인력만 고용하는 단일 전략을 줄인다.
- 후반 로봇 해금 후 진행 속도 누적이 과해지는지는 10년 시뮬레이션으로 추적해야 한다.

### UX Agent

- P1 없음.
- 에이전트 화면에서 “왜 사람도 필요하고 로봇도 필요한지”를 한눈에 설명하기 시작했다.
- 첫 화면에서 조합이 비활성일 때 다음 후보가 보이는 점은 초반 학습에 유리하다.

## 판정

`v0.4.1-alpha`는 완료로 본다. 이제 회사 성장은 공간 확장뿐 아니라 팀 구성의 질적 차이로도 표현된다.
