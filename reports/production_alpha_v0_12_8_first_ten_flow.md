# Alpha v0.12.8 제작 보고서 — 첫 10분 행동 흐름 압축

작성일: 2026-05-16

## 요약

이번 버전은 기능을 새로 늘리기보다, 지금까지 만든 시스템을 플레이어가 순서대로 밟게 만드는 작업이다. 기존에는 고용, 제품, 덱, 출시, 성장 선택, 상점, 사무실, 경쟁 메뉴가 각각 존재했지만, 첫 플레이어가 무엇을 먼저 눌러야 하는지 한눈에 잡기 어려웠다.

v0.12.8은 가이드 카드를 `첫 10분 루프`로 확장해 `고용 → 제품 시작 → 카드/이슈 대응 → 출시 → 성장 선택 → 사무실 정비 → 경쟁 대응`의 7단계를 항상 보여준다.

## 구현 내용

- `getFirstTenMinutePlan` 추가.
- `getFirstTenMinuteProgress` 추가.
- 첫 10분 단계별 완료/활성 상태 계산.
- 활성 프로젝트가 있으면 바로 다음 달이 아니라 `카드/이슈 대응`을 먼저 안내.
- 개발 이슈를 해결한 뒤에는 `출시까지 진행` 안내.
- 성장 선택 이후에는 `상점에서 사무실 정비` 안내.
- 사무실 확장/장식 후에는 `경쟁사 대응 플랜 확인` 안내.
- 가이드 카드 UI에 첫 10분 단계표와 진행률 표시.
- `?scenario=flow` QA 시나리오 추가.

## 에이전트 리뷰

### Executive Producer Agent

- P0/P1 없음.
- 기능량이 늘어난 상태에서 플레이어가 따라갈 주 루프를 다시 고정했다.

### Game Designer Agent

- P0/P1 없음.
- 카드/이슈 대응이 제품 개발의 선택형 손맛으로 더 먼저 드러난다.
- P2: 다음 버전에서는 이 흐름이 실제 10분 안에 런 결과까지 이어지는지 더 강하게 시뮬레이션해야 한다.

### Systems Architect Agent

- P0/P1 없음.
- 가이드 단계는 순수 함수로 분리했고 UI는 계산 결과만 렌더링한다.
- P2: 장기적으로는 플레이어가 실제로 메뉴를 방문했는지 추적하는 온보딩 상태가 필요할 수 있다.

### QA Agent

- P0/P1 없음.
- 가이드 순서, 활성 단계, `flow` QA 시나리오 테스트가 통과했다.

### UX Agent

- P0/P1 없음.
- 새 플레이어가 첫 화면에서 “다음에 할 일”과 “전체 흐름”을 같이 볼 수 있다.
- P2: 모바일에서 7단계 로드맵이 길어질 수 있어 계속 브라우저 QA가 필요하다.

### Retention / LTV Agent

- P0/P1 없음.
- 첫 10분 목표가 끝나기 전에 사무실 정비와 경쟁 대응이라는 다음 욕망이 보인다.

### Shareability Agent

- P0/P1 없음.
- `첫 10분 루프 86%`, `경쟁 대응` 같은 상태가 스트리밍/스크린샷에서 현재 목표를 설명한다.

### Solo Dev Scope Agent

- P0/P1 없음.
- 새 콘텐츠 없이 기존 시스템의 연결성을 높인 고효율 작업이다.

### Deck System Engineer

- P0/P1 없음.
- 덱/이슈 대응이 첫 제품 개발 중 노출되어 덱빌딩 레이어가 뒤늦게 발견되는 문제를 줄인다.

### Roguelite Meta Engineer

- P0/P1 없음.
- 첫 런의 압축된 흐름이 이후 실패/재도전 루프를 이해하는 발판이 된다.
