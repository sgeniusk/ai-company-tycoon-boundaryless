# v0.34.13-alpha 제작 보고서 — 인사 사건 미대응 후폭풍

작성일: 2026-05-18

## 목표

인사 사건은 경고와 대응 선택지까지 갖췄지만, 플레이어가 무시하고 `다음 달`을 눌렀을 때의 결과가 약했다. 이번 목표는 인사 사건을 실제 운영 압박으로 만들고, 방치했을 때 직원/자원/경쟁 구도가 흔들리는 것을 기록과 UI에 남기는 것이다.

## 이번 구현

- 인사 사건 카드에 `방치 시` 후폭풍 미리보기를 추가했다.
- 미해결 사건을 다음 달로 넘기면 최대 2명의 핵심 인사 사건이 후폭풍으로 적용된다.
- 번아웃 방치는 체력 -14, 충성 -4를 적용한다.
- 스카우트 방치는 충성 -12, 회사 화제성 -1, 해당 경쟁사 모멘텀 +8을 적용한다.
- 계약 불만 방치는 충성 -8, 현금 손실, 신뢰 -1을 적용한다.
- 후폭풍은 `recentStaffIncidentResolutions`에 `isAftermath` 기록으로 남기되, UI에서는 대응 기록과 분리해 `최근 인사 후폭풍` 패널로 보여준다.
- 후폭풍 패널을 사건 목록보다 위에 배치해 다음 달을 넘긴 결과가 먼저 보이도록 했다.
- 회사 기록의 공유 순간 카드는 후폭풍을 `스카우트 방어`가 아니라 `스카우트 후폭풍`으로 표시한다.
- `staff-aftermath` QA 시나리오를 추가했다.

## 하네스 에이전트 검토

- Executive Producer Agent: P2 해결. 인사 사건이 다음 달 운영 판단과 연결되어 월 넘기기 버튼의 의미가 커졌다.
- Game Designer Agent: P1 해결. 해결 비용을 아끼는 선택과 방치 페널티 사이의 전략 긴장이 생겼다.
- Systems Architect Agent: P2 양호. 기존 인사 사건 계산을 재사용하고, 후폭풍 기록은 기존 로그 타입에 `isAftermath`로 확장했다.
- QA Agent: P1 해결. 방치 미리보기, 월간 후폭풍 적용, 해결 후 후폭풍 방지, QA 시나리오, 레이아웃 계약을 테스트로 고정했다.
- Balance Agent: P2 관찰. 스카우트 후폭풍은 모멘텀 +8로 강하게 설정했다. 월간 경쟁사 모멘텀 감쇠가 있어 실제 다음 달 잔존 압박은 완화된다.
- UX Agent: P2 해결. 후폭풍은 대응 결과와 다른 색의 패널로 분리되어 원인을 읽기 쉽다.
- Retention / LTV Agent: P2 양호. 핵심 인재 사건을 무시하면 손실 기록이 남아 다음 판에서 더 신경 쓰게 된다.
- Shareability Agent: P2 해결. 회사 기록에 “스카우트 후폭풍” 같은 스크린샷 문구가 남는다.
- Solo Dev Scope Agent: P2 양호. 별도 사건 저장소를 만들지 않고 기존 로그 구조를 확장했다.
