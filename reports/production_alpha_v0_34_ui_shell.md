# v0.34-alpha 제작 보고서 — 그래픽 전 UI 셸 전환

작성일: 2026-05-17

## 반영한 디자인 분석

Claude 디자인 외주 분석의 핵심 판정은 “내용은 게임인데, 옷은 어드민 페이지”였다. 따라서 이번 버전은 기능을 더 넣기보다 첫 화면이 게임 화면처럼 읽히는지에 집중했다.

## 이번 구현

- 데스크톱 `1366×768`, 모바일 `390×844` 기준 셸 변수를 추가했다.
- 상단 상태바에 TOP3 경쟁사 HUD를 붙여 라이벌 압박을 첫 화면에 노출했다.
- 사무실 플레이필드 안에 `이번 달 목표` 스트립과 바로가기 버튼을 추가했다.
- 하단 명령줄에 전략 손패, 덱/버림 수, 카드 비용을 항상 표시했다.
- 자원 HUD에 전월 변화량과 위험 자원 강조를 추가했다.
- 레이아웃 계약 테스트에 v0.34 디자인 게이트를 추가했다.

## 하네스 에이전트 검토

- Executive Producer Agent: P1 해결. 다음 행동이 첫 화면에 고정되어 10분 MVP 흐름이 더 선명해졌다.
- Game Designer Agent: P1 해결. 카드, 경쟁사, 월간 목표가 동시에 보여 덱빌딩/경영/경쟁 축이 한 화면에서 연결된다.
- UX Agent: P1 일부 해결. 화면은 더 게임답게 조여졌지만, 메뉴 4+4 위계와 모바일 5탭은 다음 패스가 필요하다.
- Deck System Engineer: P1 해결. 손패가 하단 고정 HUD로 올라와 카드가 보조 메뉴가 아니라 핵심 조작처럼 보이기 시작했다.
- Solo Dev Scope Agent: P2 양호. 그래픽 에셋 없이 CSS와 기존 데이터만 활용해 화면 체감 개선을 만들었다.

## 다음 작업
