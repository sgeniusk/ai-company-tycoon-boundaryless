# Alpha v0.12.6 제작 보고서 — 경쟁사 대응 플랜과 카운터 카드

작성일: 2026-05-16

## 요약

이번 버전은 경쟁사를 단순 랭킹 배경이 아니라, 덱과 제품 선택을 흔드는 압박으로 바꾸는 첫 단계다. 경쟁사가 선점한 제품과 성장 경로 충돌을 계산해 대응 카드, 대응 제품, 필요한 연구를 추천한다.

## 구현 내용

- `rival-counters` 모듈 추가.
- 경쟁사 압박 점수, 충돌 등급, 선점 제품, 대응 도메인 계산.
- 추천 대응 카드, 추천 제품, 추천 연구 산출.
- 신규 카드 3장 추가: `경쟁사 벤치마크룸`, `시장 리포지셔닝`, `상호운용 방어막`.
- 카드 효과 `rival_score_delta`, `rival_momentum_delta` 추가.
- 카운터 카드 사용 시 최상위 압박 경쟁사의 위협과 모멘텀 감소.
- 경쟁 메뉴에 상위 대응 플랜, 랭킹 대응 힌트, 경쟁사별 추천 대응 표시.
- 덱 메뉴에 최우선 경쟁 대응 조언 표시.
- `?scenario=counter` QA 시나리오 추가.

## 에이전트 리뷰

### Executive Producer Agent

- P0/P1 없음.
- 경쟁 화면이 “읽기 전용 정보판”에서 다음 행동을 고르는 화면으로 한 단계 이동했다.

### Game Designer Agent

- P0/P1 없음.
- 경쟁 압박이 카드와 제품 개발 선택으로 연결된다.
- P2: 다음 단계에서는 경쟁 대응이 출시 평가나 첫 10분 튜토리얼 흐름에도 자연스럽게 엮여야 한다.

### Systems Architect Agent

- P0/P1 없음.
- 경쟁 대응 계산은 순수 함수로 분리했고, 카드 효과는 데이터 검증기에 등록했다.
- P2: 장기적으로 경쟁사별 고유 카운터 규칙은 별도 JSON으로 빼면 더 확장하기 쉽다.

### QA Agent

- P0/P1 없음.
- 경쟁 대응 계획, 카드 실행, QA 시나리오 테스트가 통과했다.

### Balance Agent

- P0/P1 없음.
- 카운터 카드는 라이벌 점수와 모멘텀을 낮추지만 자원 비용이 있어 무료 방어가 아니다.
- P2: 장기 런에서 카운터 카드 반복 사용이 경쟁 압박을 너무 쉽게 제거하지 않는지 시뮬레이션이 필요하다.

### UX Agent

- P0/P1 없음.
- 대응 플랜이 카드/제품/연구로 나뉘어 있어 다음 행동이 읽힌다.
- P2: 모바일에서 경쟁 카드가 길어질 수 있으므로 v0.12.7에서 첫 10분 동선과 함께 재점검한다.

### Retention / LTV Agent

- P0/P1 없음.
- “이번에는 챗지오디를 견제하는 빌드”처럼 다음 런 목표를 만들기 쉬워졌다.

### Shareability Agent

- P0/P1 없음.
- `챗지오디 대응 플랜`, `시장 리포지셔닝` 같은 문구는 스크린샷에서 게임의 소재성을 잘 보여준다.

### Solo Dev Scope Agent

- P0/P1 없음.
- 새 그래픽이나 대형 이벤트 시스템 없이 데이터와 UI만으로 경쟁사의 플레이 영향력을 키웠다.

### Deck System Engineer

- P0/P1 없음.
- 카드가 개발 진행뿐 아니라 경쟁사 수치를 건드리기 시작했다.
- P2: 카드 보상 후보에 카운터 카드가 더 자연스럽게 등장하는 흐름이 필요하다.

### Roguelite Meta Engineer

- P0/P1 없음.
- 실패 후 다음 런에서 “어떤 라이벌을 견제할지”라는 메타 목표가 생겼다.

### Balance Simulation Engineer

- P0/P1 없음.
- 현재 카운터 효과는 단발성이고 비용이 있어 즉시 무한 루프 위험은 낮다.
