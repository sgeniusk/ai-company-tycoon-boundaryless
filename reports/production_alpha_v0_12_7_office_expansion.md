# Alpha v0.12.7 제작 보고서 — 사무실 확장과 회사 꾸미기

작성일: 2026-05-16

## 요약

이번 버전은 회사가 커지는 감각을 숫자만이 아니라 공간으로도 느끼게 만드는 작업이다. 작은 차고형 연구실에서 시작해 사무실을 확장하면 고용 한도가 늘어나고, 회사 꾸미기 아이템을 배치하면 실제 제품 개발 예측에 보너스가 붙는다.

목표는 단순 장식이 아니라 “이제 사람이 더 필요하다 → 더 큰 사무실이 필요하다 → 좋은 장비와 분위기가 제품 완성도를 밀어준다”는 타이쿤식 성장 감각을 만드는 것이다.

## 구현 내용

- `office_expansions.json` 데이터 추가.
- 차고형 연구실, 스타트업 스위트, 성장층 오피스, 캠퍼스 연구동 단계 추가.
- `GameState.office`에 현재 사무실 단계와 배치 장식 상태 저장.
- 사무실 단계별 고용 한도와 장식 슬롯 적용.
- 사무실이 꽉 차면 추가 고용을 막고 확장을 안내.
- 회사/사무실 아이템 자동 배치, 보관, 재배치 기능 추가.
- 배치된 장식만 글로벌 개발 스탯 보너스로 반영.
- 상점 UI에 사무실 확장, 배치된 장식, 보관 중 장식 패널 추가.
- 사무실 화면에 사무실 이름, 고용 한도, 장식 슬롯 표시.
- `?scenario=office` QA 시나리오 추가.

## 에이전트 리뷰

### Executive Producer Agent

- P0/P1 없음.
- 고용, 상점, 사무실 화면이 하나의 성장 루프로 연결됐다.
- P2: 다음 버전에서는 첫 10분 동선에서 “왜 상점에 가야 하는지”를 더 일찍 보여줘야 한다.

### Game Designer Agent

- P0/P1 없음.
- 사무실 확장이 경영 판타지를 강화한다. 고용 한도는 플레이어에게 다음 목표를 주고, 장식 슬롯은 구매 아이템의 의미를 만든다.
- P2: 장식 아이템마다 더 뚜렷한 전략 색깔이 생기면 덱/직원 조합과 더 잘 맞물린다.

### Systems Architect Agent

- P0/P1 없음.
- 확장 단계는 JSON 데이터로 분리했고, 장식 보너스는 배치 상태를 통해 계산한다.
- 저장 버전 상향과 기존 저장 보강을 처리했다.
- P2: 향후 장식 종류가 늘면 `decor_categories` 같은 분류 데이터가 필요할 수 있다.

### QA Agent

- P0/P1 없음.
- 고용 한도, 확장 구매, 장식 슬롯, 배치 보너스, QA 시나리오 테스트가 통과했다.
- Headless Chrome DOM 기준으로 사무실 확장 UI가 렌더링되는 것을 확인했다.

### Balance Agent

- P0/P1 없음.
- 초반 한도 3명은 “더 고용하고 싶다”는 압박을 만들고, 5명 확장은 첫 목표로 충분히 읽힌다.
- P2: 장식 보너스가 필수 구매로 굳지 않도록 아이템 비용과 효과를 다음 시뮬레이션에서 본다.

### UX Agent

- P2: 상점 정보량이 늘었으므로 v0.12.8에서 첫 플레이 흐름용 안내 우선순위를 정리한다.

### Synthetic Playtester Agent

- P0/P1 없음.
- 신규 플레이어는 고용이 막혔을 때 “사무실을 넓히면 된다”는 목표를 얻는다.

### Retention / LTV Agent

- P3: 확장 순간에 짧은 축하 연출이나 타임라인 문구를 강화하면 공유성이 더 좋아진다.

### Solo Dev Scope Agent

- P2: 특정 카드 전략과 궁합이 있는 장식 태그를 나중에 추가할 수 있다.

### Roguelite Meta Engineer

- P0/P1 없음.

- P2: 메타 해금으로 시작 사무실 슬롯이나 할인 효과를 주는 선택지는 다음 단계 후보로 남긴다.

### Puzzle Mechanics Engineer

- P0/P1 없음.

- P3: 개발 이슈 대응이 살아날 때 장식이 선택 칸/난이도에 간접 영향을 줄 수 있다.
