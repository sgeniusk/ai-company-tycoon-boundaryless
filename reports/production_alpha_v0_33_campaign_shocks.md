# v0.33-alpha 제작 보고서 — 10년 캠페인 시장 충격

작성일: 2026-05-17

## 목표

이번 단계의 목표는 10년 캠페인이 단순히 120개월을 반복하는 구조로 느껴지지 않게 만드는 것이다. 3년차, 6년차, 9년차에 시장 전체를 흔드는 충격을 넣어 플레이어가 제품, 연구, 경쟁 대응을 다시 생각하게 했다.

## 구현 요약

- `data/campaign_shocks.json` 추가.
  - 3년차: `파운데이션 모델 전쟁`
  - 6년차: `연산 공급 대란`
  - 9년차: `경계 없는 산업 붐`
- `applyDueCampaignShocks()` 추가.
  - 해당 월에 도달하면 충격을 한 번만 적용한다.
  - 자원 변화, 경쟁사 모멘텀, 타임라인 로그, 충격 이력을 함께 갱신한다.
- `getCampaignShockForecast()` 추가.
  - 다음 충격, 완료 수, 현재/예정 상태, 추천 제품/역량/분야를 계산한다.
- 회사 메뉴 상단에 `시장 충격 예보` 패널을 추가했다.
  - 적용 완료 또는 다음 예정 상태를 표시한다.
  - 제품 대응, 연구 대응, 경쟁 대응 버튼으로 바로 이동한다.
- 10년 시뮬레이션 하네스가 시장 충격을 마일스톤으로 기록한다.
- 세이브 버전을 10으로 올리고, 저장 복구와 상태 무결성 검사에 `campaignShockHistory`를 추가했다.
- 데이터 검증 하네스가 시장 충격의 제품, 산업, 역량 참조를 검사한다.

## 플레이 변화

- 3년차에는 파운데이션 모델 경쟁이 시작되어 연산 비용과 신뢰 압박이 생긴다.
- 6년차에는 연산 공급 대란으로 반도체, 로봇, 최적화 연구 가치가 올라간다.
- 9년차에는 자동차, 커피 프랜차이즈, 장난감 같은 경계 없는 산업 확장이 엔딩 전 막판 선택지로 부각된다.
- 충격이 타임라인과 회사 패널에 남기 때문에 플레이어가 “왜 갑자기 어려워졌는지”를 읽을 수 있다.

## 하네스 에이전트 리뷰

### Executive Producer Agent

- 판정: 통과
- 사유: v0.33 목표였던 캠페인 중반 변곡점이 3개 고정 이벤트로 구현됐다.

### Game Designer Agent

- 판정: 통과
- 사유: 시장 충격이 단순 설명이 아니라 자원, 경쟁사, 추천 대응을 바꿔 전략적 긴장을 만든다.

### Systems Architect Agent

- 판정: 통과
- 사유: 충격 정의는 JSON 데이터, 적용은 순수 시뮬레이션 함수, 표시는 UI 예보 함수로 분리되어 있다.

### QA Agent

- 판정: 통과
- 확인: 집중 테스트, 전체 하네스, 데이터 검증, 빌드, 브라우저 DOM QA, 데스크톱/모바일 스크린샷 QA를 통과했다.

### Balance Agent

- 판정: 조건부 통과
- 사유: 충격 수치가 치명적이지 않게 설계되어 복구 가능하다. 다만 6년차 연산 공급 대란은 장기 밸런스에서 더 관찰해야 한다.

### UX Agent

- 판정: 조건부 통과
- 사유: 데스크톱에서는 회사 메뉴 첫 화면에서 충격 예보가 보인다. 모바일에서는 회사 개요 아래로 이어져 다음 UI 압축 단계에서 요약 위치를 더 끌어올려야 한다.

### Retention / LTV Agent

- 판정: 통과
- 사유: 3/6/9년차 변곡점이 “다음엔 이 충격 전에 준비하자”는 반복 플레이 동기를 만든다.

### Shareability Agent

- 판정: 통과
- 사유: `파운데이션 모델 전쟁`, `연산 공급 대란`, `경계 없는 산업 붐`은 스크린샷과 후기 문장으로 설명하기 쉽다.

### Solo Dev Scope Agent

- 판정: 통과
- 사유: 새 그래픽 에셋 없이 데이터와 기존 패널 구조만으로 장기 캠페인 리듬을 강화했다.

### Deck System Engineer

- 판정: 조건부 통과
- 사유: 시장 충격이 추천 카드/역량 방향을 암시하지만, 아직 특정 카드가 충격 대응으로 자동 강조되지는 않는다.

### Roguelite Meta Engineer

- 판정: 통과
- 사유: 실패 원인에 시장 충격 이력이 남기 때문에 다음 런 설계실에서 향후 충격 대비 빌드로 확장할 수 있다.

### Balance Simulation Engineer

- 판정: 조건부 통과
- 사유: 120개월 시뮬레이션에서 세 충격은 모두 발생한다. 다음 단계에서는 충격별 생존율과 복구 경로를 수치화한다.
