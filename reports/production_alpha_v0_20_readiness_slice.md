# v0.20-alpha 제작 보고서 — 플레이테스트 후보 슬라이스

작성일: 2026-05-17

## 목표

이번 작업은 `v0.15.7`부터 `v0.20`까지를 하나의 큰 슬라이스로 묶어, 그래픽 에셋 이전 단계의 알파 플레이테스트 후보를 만드는 것이다. 핵심은 경쟁사 시즌, 10년 엔딩, 산업 확장, QA 하네스가 실제 플레이 루프 안에서 서로 이어지게 하는 데 있다.

## 단계별 구현

### v0.15.7-alpha — 시즌 과제 보상/압박

- 시즌 대응 과제가 월간 진행에 직접 영향을 주게 했다.
- 미해결 과제는 관련 경쟁사의 모멘텀을 올리고 `시즌 압박` 로그를 남긴다.
- 완료된 과제는 신뢰, 이용자, 데이터 보상을 제공한다.

### v0.16.0-alpha — 한 화면 UI 압축

- 중앙 보조 패널의 긴 카드가 서로 겹치지 않도록 카드 단위 `max-height`와 내부 스크롤을 추가했다.
- 레이아웃 계약 테스트에 stage-side 카드 스크롤 조건을 추가했다.

### v0.17.0-alpha — 10년 엔딩 기록 강화

- 새 런을 시작할 때 10년 캠페인 엔딩 랭크, 엔딩명, 생존 연수를 런 기록에 저장한다.
- 덱/메타 화면의 최근 런 기록에 10년 엔딩 정보를 표시한다.

### v0.18.0-alpha — 경계 확장 목표

- `getBoundarylessExpansionGoals`를 추가했다.
- 제품 화면에 반도체, 로봇, 모빌리티, AI 카페, 장난감 산업의 잠김/진출 가능/출시 완료 상태를 보여준다.
- 후반 산업 확장이 단순 제품 목록이 아니라 장기 목표로 보이게 했다.

### v0.19.0-alpha — 알파 준비도 하네스

- `evaluateAlphaReadiness`를 추가했다.
- 모든 성장 경로의 10개월 상용 루프, 10년 캠페인, 상태 무결성, 엔딩 랭크를 통합 평가한다.
- v0.20 후보 점수와 게이트 결과를 계산한다.

### v0.20-alpha — 플레이테스트 후보 QA

- `?scenario=readiness` QA 시나리오를 추가했다.
- v0.20 알파 준비도 결과를 바로 브라우저에서 열 수 있게 했다.

## 하네스 에이전트 검토

### Executive Producer Agent

- P0/P1 없음.
- v0.20은 “사용자가 실제로 엔딩까지 평가할 수 있는 알파 후보”에 가까워졌다.

### Game Designer Agent

- P0/P1 없음.
- 경쟁사 시즌, 과제, 산업 확장, 10년 엔딩이 서로 연결되면서 장기 목표가 분명해졌다.
- 다음 단계에서는 시즌 과제 보상 수치를 플레이테스트 기반으로 조정해야 한다.

### Systems Architect Agent

- P0/P1 없음.
- 새 기능은 파생 계산 중심으로 구현해 저장 호환성 위험을 낮췄다.
- `boundaryless-expansion.ts`와 readiness 하네스가 별도 모듈로 분리되어 확장 가능하다.

### QA Agent

- P0/P1 없음.
- `?scenario=readiness`가 추가되어 v0.20 후보 상태를 브라우저에서 직접 점검할 수 있다.

### Balance Simulation Engineer

- P0/P1 없음.
- `evaluateAlphaReadiness`가 상용 루프, 10년 캠페인, 상태 무결성, 엔딩을 한 번에 확인한다.
- P2: 보상/압박 숫자는 아직 러프하며, 다음 플레이테스트 후 조정이 필요하다.

### UX Agent

- P0/P1 없음.
- 중앙 패널 겹침은 완화됐다.
- P2: 우측 회사 메뉴의 정보량은 여전히 많다. v0.21 이후 탭/접기/우선순위 카드 방식으로 더 줄이는 것이 좋다.

## 판정

`v0.20-alpha`는 그래픽 에셋 이전의 플레이테스트 후보로 진행 가능하다.
