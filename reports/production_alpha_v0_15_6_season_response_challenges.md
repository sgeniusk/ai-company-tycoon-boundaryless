# v0.15.6-alpha 제작 보고서 — 시즌 대응 과제

작성일: 2026-05-17

## 목표

이번 단계의 목표는 경쟁사 시즌 브리프를 실제 행동 목표로 연결하는 것이다. 이전 버전에서 “올해 누가 들어왔고 누가 가장 압박하는지”를 보여줬다면, 이번 버전에서는 플레이어가 바로 대응할 수 있는 과제를 받도록 만들었다.

## 구현 내용

- `getCompetitionSeasonChallenges`를 추가했다.
- 최대 압박 경쟁사에 대한 `압박 대응` 과제를 생성한다.
- 올해 신규 진입 경쟁사가 있으면 `신규 경쟁자 파동 대응` 과제를 생성한다.
- 각 과제는 설명, 보상 예상, 방치 위험, 추천 제품, 추천 대응 카드를 포함한다.
- 회사 현황의 시장 시즌 패널에 첫 대응 과제로 이동하는 버튼을 추가했다.
- 경쟁 메뉴 상단에 시즌 대응 과제 목록을 표시했다.

## 플레이 효과

- 경쟁 시즌 정보가 단순 관찰이 아니라 “지금 무엇을 해야 하는가”로 이어진다.
- 대응 카드와 추천 제품이 같이 보여 덱빌딩, 제품 개발, 경쟁사 대응 루프가 더 잘 묶인다.
- 후반 경쟁사 파동이 도착했을 때 플레이어가 손 놓고 당하는 느낌이 줄어든다.

## 하네스 에이전트 검토

### Executive Producer Agent

- P0/P1 없음.
- v0.15.5의 시즌 브리프가 행동 목표까지 이어져 10년 캠페인의 장기 몰입이 강화됐다.

### Game Designer Agent

- P0/P1 없음.
- 카이로소프트식 “다음 경진대회/다음 목표” 감각과 경쟁사 압박이 더 잘 연결됐다.
- 다음 단계에서는 과제 완료 시 실제 카드 보상 또는 시장 점유율 회복이 들어가야 한다.

### Systems Architect Agent

- P0/P1 없음.
- 새 저장 필드 없이 현재 상태에서 과제를 파생 계산한다. 저장 호환성 위험이 낮다.

### Deck System Engineer

- P0/P1 없음.
- 대응 과제가 추천 대응 카드를 보여주므로 덱빌딩의 목적성이 조금 더 살아났다.
- 향후 과제 보상으로 대응 카드 선택지를 제공하면 루프가 더 강해진다.

### UX Agent

- P0/P1 없음.
- 경쟁 메뉴에서 시즌 상황과 대응 과제를 한 번에 읽을 수 있다.
- P2: 10년 압축 QA 화면의 중앙 보조 패널은 여전히 정보량이 많아 겹쳐 보이는 구간이 있다. v0.16 UI 정리에서 우선 처리한다.

### Solo Dev Scope Agent

- P0/P1 없음.
- 파생 계산과 기존 UI 재사용으로 구현 범위를 작게 유지했다.

## 다음 과제

- 대응 과제 완료 여부를 연간 심사 또는 카드 보상 편향에 반영한다.
- v0.16에서 메인 화면 사이드 패널을 접기/탭/우선순위 카드 방식으로 압축한다.
