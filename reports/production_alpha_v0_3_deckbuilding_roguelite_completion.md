# Alpha v0.3 덱빌딩/로그라이트 완성 보고서

작성일: 2026-05-17  
범위: `v0.3-alpha — 덱빌딩과 로그라이트 깊이`  
목표: 한 번 깬 뒤에도 다른 빌드로 다시 하고 싶게 만드는 전략 덱, 메타 해금, 시작 덱 선택 기반을 완성한다.

## 구현 요약

- 덱 아키타입 데이터를 `data/deck_archetypes.json`로 분리했다.
- 시작 덱 데이터를 `data/starter_decks.json`로 분리했다.
- 현재 덱의 카드 태그, 강화 카드, 보상 선택 이력을 바탕으로 빌드 아키타입을 계산하는 `getDeckArchetypeSummary`를 추가했다.
- 메타 해금 상태에 따라 다음 런 시작 덱을 잠금/해금 상태로 보여 주는 `getAvailableStarterDecks`를 추가했다.
- `resetRunWithMetaUnlocks`가 선택한 시작 덱으로 새 런을 열 수 있게 확장했다.
- 덱 화면에 `현재 빌드` 패널과 `다음 런 시작 덱` 선택 영역을 추가했다.
- 저장 데이터 복원 시 `starterDeckId`를 보존하도록 했다.
- 데이터 검증 하네스가 덱 아키타입과 시작 덱의 필수 필드, 카드 참조, 메타 해금 참조를 검사한다.

## 플레이 변화

- 플레이어는 지금 덱이 `신뢰 하네스`, `런칭 과열`, `자동화 운영`, `라이벌 카운터`, `연구 복리` 중 어디에 가까운지 즉시 볼 수 있다.
- 다음 카드 보상에서 어떤 태그를 노리면 빌드가 더 선명해지는지 표시된다.
- 실패하거나 성공한 런 이후 메타 해금에 따라 다른 시작 덱을 골라 새 회사를 열 수 있다.
- 같은 10년 캠페인을 반복해도 초반 카드 구성이 달라져 다른 성장 루트를 시도할 이유가 생겼다.

## 에이전트 리뷰

### Executive Producer Agent

- P0/P1 없음.
- `v0.3-alpha`의 핵심 목표인 반복 플레이 동기는 충족했다.
- 다음 버전은 첫 10분 안내와 출시 보상 연출을 강화해야 한다.

### Game Designer Agent

- P1 없음.
- 카드가 단순 보정치가 아니라 회사 전략 정체성으로 읽히기 시작했다.
- 다음 과제는 아키타입별 승리 조건과 대표 이벤트를 더 강하게 묶는 것이다.

### Systems Architect Agent

- P0/P1 없음.
- 아키타입과 시작 덱은 JSON 데이터로 분리되어 확장 가능하다.
- `GameState.roguelite.starterDeckId`가 단일 출처로 저장된다.

### QA Agent

- P0/P1 없음.
- 단위 테스트, 데이터 검증, 프로덕션 빌드, 브라우저 렌더링을 통과했다.
- 남은 리스크는 시작 덱 버튼을 언제 눌러도 새 런이 시작되는 UX 정책이다. 현재는 의도적으로 빠른 반복 플레이를 우선했다.

### Balance Agent

- P1 없음.
- 시작 덱은 초반 방향성을 바꾸지만, 메타 해금과 카드 사용 조건 때문에 즉시 압도적 우위가 되지는 않는다.
- 장기적으로는 아키타입별 10년 시뮬레이션 승률 비교가 필요하다.

### UX Agent

- P1 없음.
- 덱 화면에서 현재 빌드가 보이므로 “내가 무슨 전략을 하는지”가 더 명확해졌다.
- 우측 패널 밀도는 여전히 높으므로 `v0.16`에서 슬림 패널 작업이 필요하다.

### Retention / LTV Agent

- P1 없음.
- 실패/성공 후 시작 덱 선택이 생겨 다음 런의 이유가 강해졌다.
- 다음 과제는 해금 덱별 대표 업적과 수집 목표다.

### Solo Dev Scope Agent

- P0/P1 없음.
- 그래픽 에셋 없이 데이터와 기존 덱 UI를 재사용해 가치 대비 구현 부담이 낮다.
- 다음 콘텐츠 증가는 JSON 템플릿으로 유지해야 한다.

## 판정

`v0.3-alpha`는 완료로 본다. 아직 그래픽 에셋이나 사운드가 없는 단계지만, 덱빌딩과 로그라이트 반복 동기의 최소 골격은 실제 플레이 화면과 테스트 하네스에 연결됐다.
