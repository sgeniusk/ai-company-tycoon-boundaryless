# Alpha v0.12.3 제작 보고서 — 카드 보상과 덱 편집

작성일: 2026-05-15

## 요약

이번 버전은 출시 후의 보상을 “숫자 상승”이 아니라 다음 제품 전략을 바꾸는 카드 선택으로 전환했다. 첫 제품을 완성하면 카드 보상 3택1이 열리고, 플레이어는 선택한 카드를 덱에 추가하거나 편집 토큰으로 기존 카드를 제거/강화할 수 있다.

## 구현 내용

- 제품 출시 후 `pendingCardReward`를 생성한다.
- 보상 후보는 제품 태그, 카드 태그, 현재 덱 보유량, 리뷰 점수를 기준으로 3장 선정한다.
- 보상 선택 시 선택 카드는 버림 더미에 추가되고 보상 이력이 기록된다.
- 출시마다 덱 편집 토큰을 1개 지급한다.
- 덱 편집 토큰으로 카드 1장 제거 또는 카드 1종 강화를 수행한다.
- 강화 카드는 긍정 효과가 25% 상승하고 카드 사용/퍼즐 보정에 반영된다.
- 덱 메뉴에 카드 보상, 덱 편집, 강화 상태, 카드 수량을 표시한다.
- 저장 버전 `4`에서 카드 보상/편집/강화 상태를 안전하게 저장하고 복구한다.
- `?scenario=reward` 브라우저 QA 시나리오를 추가했다.

## 에이전트 리뷰

### Executive Producer Agent

- P0/P1 없음.
- v0.12.2의 출시 루프 뒤에 즉시 보상 선택이 붙어 “다음 한 달”의 목표가 생겼다.

### Game Designer Agent

- P0/P1 없음.
- 카드 보상 3택1은 같은 제품 출시라도 다른 덱 빌드로 이어지게 한다.
- P2: 보상 후보가 아직 데이터 카드 수에 제한되어 있어, v0.15 콘텐츠 확장 전까지 반복 변주는 제한될 수 있다.

### Systems Architect Agent

- P0/P1 없음.
- 런 상태는 `GameState.roguelite` 안에서 단일 출처로 관리되고, 저장 복구도 새 필드를 보강한다.
- P2: 다음 단계에서 “대표 카드”와 “런 기록”은 별도 런 히스토리 타입으로 분리하는 편이 좋다.

### QA Agent

- P0/P1 없음.
- 카드 보상, 선택, 제거, 강화, 저장 복구, QA 시나리오 테스트가 추가됐다.

### Balance Agent

- P0/P1 없음.
- 편집 토큰은 출시마다 1개라 초반 무한 압축은 어렵다.
- P2: 좋은 보상 카드가 반복되면 덱이 너무 빨리 강해질 수 있어 보상 가중치와 희귀도 풀을 계속 점검해야 한다.

### UX Agent

- P0/P1 없음.
- 보상 선택과 덱 편집이 같은 패널에 있어 출시 후 행동이 눈에 들어온다.
- P2: 카드 효과 텍스트가 길어지므로 그래픽 에셋 전에도 카드 상세 접기/툴팁이 필요할 수 있다.

### Synthetic Playtester Agent

- 신규 플레이어: “출시 후 바로 고를 것이 생겨서 다음 목표가 선명하다.”
- 타이쿤 팬: “제품 조합 공략에 카드 빌드 공략이 붙기 시작했다.”
- 미니맥스 플레이어: “제거와 강화가 생겨 덱 압축 실험을 할 수 있다.”
- 캐주얼 플레이어: “카드를 고르는 보상은 이해하기 쉽지만, 강화/제거 우선순위는 더 알려주면 좋겠다.”
- 혹평형 리뷰어: “아직 결과 화면이 약해서 한 런의 이야기를 캡처하고 싶지는 않다.”
