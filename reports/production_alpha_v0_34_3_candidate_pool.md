# v0.34.3-alpha 제작 보고서 — 월간 채용 후보 풀

작성일: 2026-05-17

## 목표

채용 방식은 생겼지만 후보가 항상 전체 목록으로 보이면 경영 선택의 긴장감이 약하다. 이번 패스는 “이번 달에 누구를 데려올 수 있는가”를 제한해, 공채/경력 채용/헤드헌터가 실제로 다른 창구처럼 느껴지게 하는 것이다.

## 이번 구현

- `getRecruitmentCandidatePool()`을 추가했다.
- 후보 풀은 월, 지역, 회사 별 등급, 채용 방식, 에이전트 종류, 희귀도, 능력치 합, 해금 조건을 기준으로 결정된다.
- 공채는 지역 사람 직원과 초반 공통 인재를 우선한다.
- 경력 채용은 실무형 사람 직원과 AI 에이전트를 더 고르게 보여준다.
- 헤드헌터는 희귀도와 능력치가 높은 후보를 우선하되, 2성 전에는 사람 후보 중심으로 제한한다.
- 이미 고용한 인력은 후보 풀에서 빠진다.
- 에이전트 콘솔에 후보 수, 지역 인재풀, 다음 후보 갱신 월을 표시했다.

## 하네스 에이전트 검토

- Executive Producer Agent: P1 해결. 채용 루프가 “전체 도감에서 하나 고르기”에서 “이번 달 후보 중 판단하기”로 전환됐다.
- Game Designer Agent: P1 해결. 후보 풀이 제한되면서 좋은 후보를 지금 잡을지, 다음 달까지 기다릴지의 판단이 생겼다.
- Systems Architect Agent: P1 해결. 후보 풀은 기존 에이전트 데이터와 시뮬레이션 함수만 사용하며, UI는 `candidateIds`를 받아 표시만 한다.
- Balance Agent: P2 관찰. 월 갱신이 후보 운을 만들지만, 아직 후보 보류/스카우트 예약 비용은 없다. 추후 연봉 협상과 함께 재검증한다.
- UX Agent: P1 해결. 후보 수와 다음 갱신 월이 보여 채용 화면의 맥락이 명확해졌다.
- Solo Dev Scope Agent: P2 양호. 새 데이터 폭증 없이 기존 데이터 점수화로 구현했다.

## 남은 작업

- 우수 후보를 한 달 보류하는 예약금/스카우트 킵 기능.
- 직원 성장, 연봉 협상, 이직/퇴사 리스크.
- 지역 이전 시 후보 풀이 바뀌는 체감을 튜토리얼/속보로 더 분명히 보여주기.
- 20인 페르소나 테스트에서 채용 화면의 이해도 재검증.
