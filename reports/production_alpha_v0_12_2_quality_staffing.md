# Alpha v0.12.2 제작 보고서 — 완성도 중심 출시와 직원 배치

작성일: 2026-05-15

## 요약

이번 버전은 제품 출시를 “개발 시간이 다른 제품을 기다리는 구조”에서 “직원 배치, 카드, 퍼즐, 아이템이 완성도를 가르는 구조”로 바꾸는 작업이다. 플레이어는 제품 개발 시작 전에 투입 에이전트를 직접 고르고, 선택한 팀의 예상 완성도와 리뷰 등급을 확인한 뒤 프로젝트를 시작한다.

## 구현 내용

- 제품 개발 시작 함수가 선택된 에이전트 ID를 받도록 확장.
- 빈 직원 선택, 이미 배치된 직원, 3명 초과 배치를 검증.
- 월간 개발 진행도는 안정적인 2개월 기준으로 정리.
- 직원 능력치는 개발 기간보다 시작 완성도, 월 완성도 상승, 출시 리뷰 점수에 더 크게 반영.
- 제품 카드에 선택 팀, 예상 개월, 예상 완성도, 예상 리뷰, 월 완성도 상승 표시.
- 상점 화면에 인벤토리 요약과 보유 아이템 목록 추가.
- 사무실 화면에 실제 고용 에이전트 스프라이트, 이름표, 활성 프로젝트 보드 추가.
- `?scenario=staffing` 브라우저 QA 시나리오 추가.

## 에이전트 리뷰

### Executive Producer Agent

- P0/P1 없음.
- 게임 화면이 이전보다 “실제 조작 가능한 경영 게임”에 가까워졌다.

### Game Designer Agent

- P0/P1 없음.
- 완성도 중심 구조가 카드/퍼즐/직원/아이템을 하나의 출시 결과로 묶는다.
- P2: 다음 버전에서 출시 후 카드 보상 3택1이 필요하다.

### Systems Architect Agent

- P0/P1 없음.
- 제품 프로젝트 예측과 시작 로직은 `GameState`와 데이터 정의를 기준으로 동작한다.
- P2: 직원별 제품 태그 선호도와 시너지 태그를 데이터로 분리하면 확장성이 좋아진다.

### QA Agent

- P0/P1 없음.
- 신규 테스트가 명시적 배치, 안정적인 개발 기간, 완성도 기반 리뷰 차이를 검증한다.

### Balance Agent

- P0/P1 없음.
- 개발 기간 차이를 줄여 초반 리듬을 안정화했고, 고점은 완성도로 이동했다.
- P2: 카드/퍼즐을 과하게 쓰면 첫 제품 S랭크가 쉬워질 수 있으므로 다음 밸런스 시뮬레이션에서 확인한다.

### UX Agent

- P0/P1 없음.
- 제품 카드 안에서 팀 선택과 예상 결과가 바로 보여 판단 근거가 생겼다.
- P2: 모바일에서 제품 카드가 길어지므로 추후 접기/요약 UI가 필요하다.

### Synthetic Playtester Agent

- 신규 플레이어: “누굴 넣으면 뭐가 달라지는지”가 보인다.
- 타이쿤 팬: 직원 배치가 생기면서 게임데브스토리식 조합 감각에 가까워졌다.
- 미니맥스 플레이어: 완성도 예측 수치가 있어 조합 실험의 출발점이 생겼다.
- 혹평형 리뷰어: 아직 카드 보상과 덱 편집이 없어 반복 플레이 변주는 부족하다고 평가.
