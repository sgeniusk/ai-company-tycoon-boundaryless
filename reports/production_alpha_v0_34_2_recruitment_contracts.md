# v0.34.2-alpha 제작 보고서 — 채용 방식과 연봉 계약

작성일: 2026-05-17

## 목표

에이전트 고용이 단순 구매 버튼처럼 느껴지는 문제를 줄이고, 회사 운영 게임답게 “싸게 많이 뽑을지, 비싸게 검증 인재를 데려올지”를 선택하게 만든다.

## 이번 구현

- 에이전트 채용 방식 3종을 추가했다: `공채`, `경력 채용`, `헤드헌터`.
- 공채는 지역의 사람 직원 채용 할인과 낮은 유지비를 살린다.
- 경력 채용은 채용비와 연봉이 늘지만 모든 능력치에 보정이 붙는다.
- 헤드헌터는 가장 비싸고 월 고정비가 높지만 강한 능력치 보정을 제공하며 `검증 필요` 리스크를 표시한다.
- 고용된 인력은 채용 경로, 계약 품질, 리스크, 월 유지비, 능력치 보정을 저장한다.
- 월간 비용 계산은 기존 추상 인재 급여에 실제 계약 유지비를 더해 경력 채용의 고정비 압박을 반영한다.
- 우측 에이전트 콘솔은 좁은 게임 화면 안에서 카드가 읽히도록 도감 리스트를 1열로 압축했다.

## 하네스 에이전트 검토

- Executive Producer Agent: P1 해결. 채용 메뉴가 다음 제작 단계의 회사 운영축으로 명확해졌다.
- Game Designer Agent: P1 해결. 초반 차고에서는 싼 공채, 성장 후에는 경력/헤드헌터를 쓰는 전략 긴장이 생겼다.
- Systems Architect Agent: P1 해결. 채용 채널, 계약 비용, 능력치 보정은 시뮬레이션 함수로 분리했고 기존 `hireAgent`는 공채 래퍼로 유지했다.
- Balance Agent: P2 관찰. 경력 채용과 헤드헌터가 능력치를 올리지만 월 급여 압박이 커져 무한 우월 전략은 아직 보이지 않는다. 후보 풀 갱신이 들어가면 재검증 필요.
- UX Agent: P1 해결. 채용 방식 선택, 계약 배지, 위험 라벨이 한 화면에서 보인다. 2열 카드 가독성 문제는 1열 리스트로 수정했다.
- Solo Dev Scope Agent: P2 양호. 별도 후보 생성 시스템까지 확장하지 않고 기존 에이전트 데이터 위에 계약 계층을 얹었다.

## 남은 작업

- 후보 풀을 월/지역/회사 등급에 따라 갱신해 공채와 헤드헌터의 체감 차이를 더 키운다.
- 직원 성장, 이직/퇴사 리스크, 연봉 협상 이벤트를 작게 추가한다.
- 실제 사무실 좌석과 채용 한도, 직원 배치 상태를 더 촘촘히 연결한다.
- 20인 페르소나 테스트를 v0.34.2 기준으로 다시 돌린다.
