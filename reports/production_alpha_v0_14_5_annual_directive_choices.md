# Alpha v0.14.5 제작 보고서 — 연간 심사 후 3택1 운영 지시

작성일: 2026-05-16

## 요약

v0.14.5는 연간 심사 이후 플레이어가 다음 해의 운영 방향을 직접 고르는 버전이다. 이전 버전은 심사 결과에 따라 자동 지시가 붙었지만, 이번 버전부터는 통과/회복 결과에 맞는 후보 3개가 나타나고 플레이어가 월간 보너스와 추천 메뉴를 비교해 선택한다.

이 변화는 10년 캠페인을 “매년 평가받고 다음 해 운영 전략을 고르는 게임”으로 강화한다. 실패해도 회복 전략을 선택할 수 있고, 통과하면 제품 출시, 신뢰, 연구, 자동화 같은 다른 성장 전략으로 갈라질 수 있다.

## 구현 내용

- `data/annual_directive_choices.json` 추가.
- 통과/회복 결과와 연차 범위에 따라 후보 3개를 고르는 선택 생성 로직 추가.
- `pendingAnnualDirectiveChoices` 저장 상태와 레거시 저장 복구 추가.
- `chooseAnnualDirective`로 선택한 후보가 활성 연간 지시를 대체하도록 구현.
- 회사 화면의 연간 심사 카드 안에 `다음 해 운영 지시 3택1` UI 추가.
- 24개월 이상 자동 진행하고 지시 선택까지 처리하는 `runAnnualDirectiveSimulation` 추가.
- 데이터 검증 하네스와 상태 무결성 검사를 선택형 지시까지 확장.
- `?scenario=review` QA 시나리오가 3개 후보를 노출하는지 테스트로 고정.

## 하네스 관점 리뷰

- Executive Producer: 연간 심사가 다음 행동으로 이어져 10년 캠페인의 장기 목표 밀도가 높아졌다.
- Game Designer: 자동 보상보다 선택의 맛이 생겼고, 통과/미달 모두 전략 분기를 만든다.
- Systems Architect: 선택 후보는 JSON 데이터로 분리되어 새 연차/새 산업 확장 지시를 쉽게 추가할 수 있다.
- QA Agent: 저장 복구, 시나리오 QA, 장기 시뮬레이션이 모두 선택 상태를 확인한다.
- Balance Agent: 월간 효과는 작게 나눠져 있으며, 현금/유저/신뢰/연구/자동화 사이에 선택 비용이 생긴다.
- Retention / LTV Agent: 매년 “다음 해는 어떤 회사로 갈까”라는 복귀 이유가 생겼다.
- Shareability Agent: 연간 심사 결과와 3택1 지시는 스크린샷으로 설명하기 쉬운 순간이다.
- Solo Dev Scope Agent: 새 화면을 만들지 않고 기존 회사 패널 안에 붙여 유지비를 낮췄다.
- Deck System Engineer: 아직 카드와 직접 연결되지는 않지만, 향후 특정 지시가 카드 보상 풀을 바꾸는 확장점이 생겼다.
- Roguelite Meta Engineer: 실패한 해도 회복형 지시를 고르며 다음 런/다음 해 이야기를 만들 수 있다.
- Balance Simulation Engineer: 24개월 장기 시뮬레이션으로 최소 첫 선택까지 자동 검증한다.

## 발견 이슈

- P0: 없음.
- P1: 없음.
- P2: 연간 지시 선택이 현재는 즉시 경쟁사 모멘텀을 다시 조정하지 않는다. 다음 밸런스 단계에서 선택 지시별 경쟁사 반응을 별도 튜닝할 수 있다.
- P2: 선택 후보는 현재 결정론적 우선순위다. 향후 회사 전략, 덱, 경쟁사 상태에 따라 후보 풀이 바뀌면 반복 플레이성이 더 강해진다.
- P3: 선택 버튼 이후 작은 선택 완료 애니메이션이나 사운드 훅이 있으면 체감 보상이 좋아진다.
