# v0.56 Blind Playtest Plan

Status: 준비됨 / 실제 테스트 미실시
작성일: 2026-05-21
대상: `v0.56-alpha-playtest-slice-lock`

## 목적

실제 사람이 설명 없이 첫 20~30분을 플레이했을 때 아래 세 가지가 게임으로 읽히는지 검증한다.

- 첫 출시가 보상감 있는 순간으로 보이는가
- 카드, 경쟁사, 직원 사건이 결과를 바꾸는 것으로 보이는가
- 첫 10분 이후 다음 목표가 1년차 연간 심사로 자연스럽게 이어지는가
- `인력 조합` 패널에서 사람 직원, AI 에이전트, 로봇 인력의 차이를 읽는가

## P0 판정

다음 중 하나라도 발생하면 `v0.56-alpha`에서 먼저 고친다.

- 첫 3분 안에 다음 행동을 못 찾는다.
- 첫 10분 안에 첫 제품 출시로 가지 못한다.
- 카드, 개발 이슈, 결과 리본이 결과를 바꿨다는 느낌이 없다.
- 출시 결과가 왜 좋은지 나쁜지 이해하지 못한다.
- 출시 후 `보상 카드 선택`, `성장 분기 선택`, `다음 달 진행` 중 다음 행동을 고르지 못한다.
- `첫 보상 바로 선택` 또는 `성장 분기 바로 선택`을 보고도 다음에 눌러야 할 행동으로 이해하지 못한다.
- 덱 패널의 `첫 출시 보상 도착` 스포트라이트를 보고도 3장 중 1장을 고르는 순간으로 이해하지 못한다.
- `보상 선택 완료` 리본을 보고도 선택한 카드가 덱에 들어간 것을 이해하지 못한다.
- `성장 분기 선택 완료` 리본을 보고도 다음 달 월간 보너스와 연간 심사 방향을 이해하지 못한다.
- `심사까지 진행`을 보고도 연간 심사로 넘어가는 다음 행동이라고 이해하지 못한다.
- `다음 해 지시 선택 완료` 리본을 보고도 선택한 연간 지시, 월간 보너스, 추천 메뉴, `추천 메뉴 열기`/`2년차 시작` 버튼을 이해하지 못한다.
- `2년차 운영 시작` 카드를 보고도 이번 달 보너스와 연간 지시 효과가 실제로 적용됐다고 이해하지 못한다.
- 연구 메뉴의 `연간 지시 추천 연구` 카드와 `바로 연구` 버튼을 보고도 다음 행동을 고르지 못한다.
- `연구 완료` 리본을 보고도 레벨 상승, 해금 시장, 제품 후보가 보상으로 이어졌다고 이해하지 못한다.
- 제품 메뉴의 `연구가 연 제품 후보` 런치패드를 보고도 해금 시장, 다음 제품 후보, `필요 연구 보기`, 연구 메뉴의 `바로 연구`, 착수 후 `덱 열기`, 신제품 이슈 결과, 신제품 출시 결과를 이해하지 못한다.
- `인력 조합` 패널을 보고도 사람/AI/로봇 차이가 전혀 읽히지 않는다.
- 화면 정보량 때문에 주요 행동을 포기한다.

## 운영 규칙

- 외부 진행자나 AGY에게 맡길 때는 `reports/playtests/v0_56_blind_playtest_request_packet.md`의 요청 문구와 체크리스트를 먼저 전달한다. AGY에 바로 붙여 넣을 메시지는 `reports/playtests/v0_56_blind_playtest_agy_outbox.md`를 사용한다.
- 발송 전에는 `npm run qa:blind-readiness`로 세션 파일이 아직 `예정`이고 요약 게이트가 `대기`인지 확인한다.
- 게임 설명 없이 URL만 제공한다.
- 플레이 중 질문에 답하지 않는다.
- 실제 세션 후에는 해당 기록 파일의 `Status`를 정확히 `완료`로 바꾸고, `P0:`에는 `없음` 또는 구체적인 P0를 기록한다.
- 테스터 프로필, `날짜/시간`, `환경`, 첫 10초/3분/10분/15분/20분/30분 관찰 칸, 종료 질문 5개 답변은 필수 기록이다.
- 막힌 시간, 클릭한 위치, 읽은 문구, 무시한 패널을 기록한다.
- 첫 10초에는 미나의 첫 안내와 `opening-fantasy-signal` 중 무엇을 먼저 읽는지 기록한다.
- 첫 20분에는 `인력 조합`, `사람 직원`, `AI 에이전트`, `로봇 인력` 문구가 역할 차이로 읽히는지 기록한다.
- 첫 3분에는 `추천 첫 제품` 카드와 `첫 제품 개발 시작` 버튼을 발견했는지 기록한다.
- 첫 15분에는 `첫 개발 이슈` 카드, `자동 선택 이슈 해결` 버튼, `이슈 해결 결과` 리본이 카드 체감에 도움됐는지 기록한다.
- 첫 출시 직후에는 `보상 카드 선택`, `성장 분기 선택`, `다음 달 진행` 리본을 봤는지와 어느 버튼을 눌렀는지 기록한다.
- 결과/가이드 영역의 `첫 보상 바로 선택`과 `성장 분기 바로 선택`을 자연스럽게 눌렀는지 기록한다.
- 덱 패널에서는 `첫 출시 보상 도착` 스포트라이트를 보고 3장 중 1장을 고르는 행동으로 이해했는지 기록한다.
- 보상 선택 후에는 `보상 선택 완료` 리본과 선택한 카드가 덱에 들어갔다는 문구를 읽는지 기록한다.
- 성장 분기 선택 후에는 `성장 분기 선택 완료`, `다음 달부터 월간 보너스`, `연간 심사까지` 문구를 읽는지 기록한다.
- 연간 심사 전에는 `심사까지 진행`을 눌렀는지, 이 버튼을 연간 심사로 가는 행동으로 이해했는지 기록한다.
- 연간 심사 후에는 `다음 해 지시 선택 완료`, `월간 보너스`, `추천 메뉴`, `추천 메뉴 열기`, `2년차 시작` 문구를 읽는지와 선택한 연간 지시, 누른 다음 행동을 기록한다.
- 2년차 첫 달에는 `2년차 운영 시작`, `이번 달 보너스`, `연간 지시 효과`, `한 달 더 운영` 문구가 연간 지시의 실제 적용으로 읽히는지 기록한다.
- 2년차 연구 메뉴에서는 `연간 지시 추천 연구`와 `바로 연구` 버튼이 다음 행동으로 읽히는지 기록한다.
- `바로 연구` 이후에는 `연구 완료`, `해금 시장`, `제품 후보`가 실제 보상으로 읽히는지 기록한다.
- 제품 메뉴로 이동한 뒤에는 `연구가 연 제품 후보`, `해금 시장`, `다음 제품 후보`, `필요 연구 보기`, 연구 메뉴의 `제품 후보 필요 연구`와 `바로 연구`, 착수 리본의 `덱 열기`, 신제품 이슈 결과, 신제품 출시 결과가 다음 행동으로 읽히는지 기록한다.
- 첫 제품 출시 전후 반응은 별도 메모한다.
- 실제 세션 전에는 결과를 채우지 않는다.
