# Synthetic Playtest — Alpha v0.4.0 Systems Prototype

Date: 2026-05-15

## Test Setup

Simulated tester panel reviewed the current prototype loop before final graphics assets.

## Persona Findings

### New Player

Result: Understands the screen is a management game. Product menu now gives a first-action cue when no agent is hired.

Request: Add a stronger reward beat after the first release.

### Tycoon Fan

Result: Recognizes the Kairosoft-style loop: staff, items, product development, monthly advance, review score.

Request: Add more feedback moments during project completion and release.

### Min-Max Player

Result: Immediately sees the opening economy as the main optimization puzzle.

Risk: Current optimal path may become "first product, then wait" unless second-product choices get more attractive.

### Casual Steam Player

Result: Can complete a satisfying first loop within a short session.

Request: Make the first successful release feel louder before adding real pixel assets.

### Harsh Steam Reviewer

Result: Prototype finally reads like a game instead of a dashboard.

Complaint: Still needs richer consequences, better balance, and less flat textual feedback.

### Accessibility Tester

Result: Korean labels are readable and locked reasons are explicit.

Request: Recheck small viewport density after menu component split.

## Improvements To Queue

- Add release animation/reward state using CSS before sprite assets.
- Tune second-product costs so first product revenue creates a next decision.
- Add a clearer recovery path when cash goes negative.

## Verdict

The pre-asset prototype loop is playable. Next value comes from balance and feedback polish, not more raw systems.
