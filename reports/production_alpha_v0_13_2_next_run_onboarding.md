# Alpha v0.13.2 제작 보고서 — 새 런 덱 온보딩

작성일: 2026-05-16

## 요약

v0.13.1에서 결과 화면은 “다음 런을 누를 이유”를 보여 주게 됐다. v0.13.2는 그 다음 클릭 직후를 다듬었다. 이제 `통찰 받고 새 런`을 누르면 플레이어는 회사 화면으로 돌아가지 않고 덱 메뉴로 이동해 최근 런 기록, 보유 통찰, 메타 해금 후보, 새 런 첫 행동을 확인한다.

## 구현 내용

- `?scenario=next-run` QA 시나리오 추가.
- 10분 알파 완료 상태에서 `resetRunWithMetaUnlocks`를 적용해 런 2 상태를 생성.
- 새 런 시나리오의 기본 메뉴를 `덱`으로 설정.
- 결과 화면의 `통찰 받고 새 런` 버튼이 새 런 시작 후 덱 메뉴로 이동.
- 덱 메뉴 상단에 `새 런 브리핑` 추가.
- 브리핑에 이전 런 점수, 대표 제품, 대표 카드, 첫 행동 3단계를 표시.

## 에이전트 리뷰

### Executive Producer Agent

- P0/P1 없음.
- 결과 화면과 새 런 시작 사이의 빈 구간을 줄였고, 플레이어의 다음 행동이 더 선명해졌다.

### Game Designer Agent

- P0/P1 없음.
- 로그라이트 반복 플레이의 핵심인 “이전 런을 보고 다음 런을 조정한다”는 감각이 덱 메뉴에서 이어진다.
- P2: 메타 해금 자체가 아직 리스트형이라 보상 선택 장면의 감정은 더 키워야 한다.

### Systems Architect Agent

- P0/P1 없음.
- 새 런 QA는 기존 리셋 함수를 재사용하고, UI는 덱 패널 내부 브리핑으로만 추가했다.

### QA Agent

- P0/P1 없음.
- `next-run` 시나리오 ID, URL 파싱, 덱 메뉴 진입, 런 2 상태, 최근 런 기록을 테스트했다.
- 전체 하네스 게이트와 브라우저 DOM QA에서 새 런 브리핑 렌더링을 확인했다.

### Retention / LTV Agent

- P0/P1 없음.
- 새 런 첫 화면에서 최근 런 기록과 해금 후보를 보게 되어 복귀/재도전 동기가 좋아졌다.

### Solo Dev Scope Agent

- P0/P1 없음.
- 별도 모달이나 새 화면 없이 기존 덱 메뉴를 강화해 범위를 작게 유지했다.

## 검증

- `npm test -- src/game/qa-scenarios.test.ts`: 통과, 16 tests.
- `npm run harness:gate`: 통과, 114 tests.
- Headless Chrome DOM QA: `http://127.0.0.1:5174/?scenario=next-run`에서 `새 런 진입 QA`, 덱 메뉴 활성화, `새 런 브리핑`, 런 2/통찰 13, 최근 런 기록, 메타 해금 후보 확인.

## 다음 액션

1. 메타 해금 선택을 더 보상감 있는 카드 선택 장면처럼 강화한다.
2. 새 런 시작 직후 첫 제품 후보를 덱 브리핑과 연결한다.
3. 그래픽 에셋 전까지 결과/새 런/해금 장면의 정보 밀도를 계속 줄인다.
