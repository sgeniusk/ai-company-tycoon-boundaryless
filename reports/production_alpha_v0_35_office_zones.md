# v0.35-alpha 제작 보고서 — v0.4 회사 운영/공간 성장 구획 시스템

작성일: 2026-05-19

## 목표

오늘 목표는 v0.4의 핵심인 “회사가 실제 공간으로 커지는 감각”을 더 강하게 만드는 것이다. 단순히 고용 한도와 장식 슬롯만 늘어나는 것이 아니라, 차고에서 시작한 회사가 성장할수록 새로운 구역이 열리고, 그 구역이 월간 운영 효과와 다음 목표를 만들어야 한다.

## 이번 구현

- `data/office_zones.json`을 추가했다.
- 사무실 구획 8개를 준비했다.
  - 창업자 지휘 데스크
  - 차고 연산 베이
  - 채용 면접 코너
  - 제품 런칭 무대
  - 복지 라운지
  - 로봇 고용 베이
  - 칩 실험 랩
  - 경계 없는 쇼룸
- `getOfficeZonePlan()`을 추가해 현재 가동 구획, 잠긴 구획, 다음 구획, 월간 총 효과를 계산한다.
- 월간 전략 효과에 사무실 구획 효과를 합산했다.
- 타임라인에 `사무실 구획` 가동 기록을 남긴다.
- 회사 화면에 사무실 구획 패널을 추가했다.
- 상점/인벤토리 화면의 사무실 성장 플래너에도 운영 구획을 표시한다.
- 게임 스테이지 HUD에 가동 구획 수를 표시한다.
- 데이터 검증 하네스가 `office_zones.json`의 리소스, 연구, 산업 조건을 검사하도록 확장했다.

## 하네스 에이전트 검토

- Executive Producer Agent: P1 해결. v0.4의 “회사 운영과 공간 성장”이 단순 확장 버튼을 넘어 다음 공간 목표로 보인다.
- Game Designer Agent: P1 해결. 사무실 단계, 제품 출시, 채용, 연구, 산업 해금이 구획 개방 조건으로 연결됐다.
- Systems Architect Agent: P1 해결. 구획 정의와 밸런스 수치는 JSON 데이터로 분리했고, 시뮬레이션은 `getOfficeZonePlan()`으로 한 번에 계산한다.
- QA Agent: P1 해결. 구획 개방, 월간 효과, 콘텐츠 풀, 레이아웃 계약을 테스트로 고정했다.
- Balance Agent: P2 관찰. 초반 구획 효과는 작은 보너스지만 기존 사무실/시너지 효과와 합산되므로 장기 시뮬레이션에서 조정이 필요하다.
- UX Agent: P2 해결. 회사 화면과 상점 화면 양쪽에서 현재 구획과 다음 구획을 볼 수 있다.
- Retention / LTV Agent: P2 양호. 다음 사무실, 다음 구획, 다음 산업이 연결되어 복귀 목표가 생긴다.
- Shareability Agent: P2 양호. “로봇 고용 베이”, “경계 없는 쇼룸”처럼 스크린샷에 남는 이름을 넣었다.
- Solo Dev Scope Agent: P2 양호. 별도 배치 에디터 없이 계산형 구획으로 구현해 유지 부담을 낮췄다.
