# 로그라이크 콘텐츠·시스템 로드맵 (v0.63+ Codex 파이프라인)

작성일: 2026-05-29
작성자: Claude Code (기획/하네스 트랙)
역할 분담: Codex CLI가 코딩, Claude가 기획·하네스·검증·커밋. 이 문서는 Codex에 끊김 없이 던질 블록 큐의 설계도다.
상위 비전 문서: `reports/v0_62_design_direction.md` (북극성, 결정 A/B/C, 도파민 4-레버)

## 북극성 (재확인)

"계속 반복해서 한 판 더" 하는 로그라이크. 카이로소프트보다 세밀한 픽셀아트, 열린 도전, 풍성한 이벤트와 컨텐츠, 무궁무진한 조합. 긴장·해소는 난이도 조정으로 상당 부분 해결한다(사용자 결정 2026-05-29).

## 현재 시스템 척추 (v0.63에서 구축)

런 모디파이어 엔진은 **순수 데이터 주도**다. 이게 전체 전략의 핵심 — 콘텐츠는 코드 없이 JSON으로 무한 확장된다.
- 블록 #1 — run-modifier foundation. 4축(start_cities / world_lore / market_conditions / founder_traits) + tag_effects + `GameState.runModifiers`(시드 저장, 세이브 마이그레이션). `selectOption`이 시드로 결정론적 선택.
- 블록 #2 — 세계가 월간 경제를 문다. tag_effects를 `getMonthlyStrategicEffects`에 v0.60 시너지 패턴으로 2줄 훅.
- 블록 #3 — 연중 세계 이벤트. campaign_shocks를 미러링한 병렬 레이어, 시드 파생 결정론, `worldEventHistory` 중복 방지. **매년 변수가 계속 들어온다.**
- 블록 #4 (진행 중) — 세계 굴리기 + "세계 뽑기" 리빌 UI. 이게 일반 플레이에서 시스템을 처음으로 살린다(지금까지는 QA 시나리오로만 비표준 세계 진입 가능).

## 리플레이성 가치 사슬 (5개 레버)

시스템 척추만으로는 "한 판 더"가 안 된다. 다섯 레버가 다 채워져야 한다.
1. **다양성(콘텐츠 부피)** — 런마다 다르게 느껴질 만큼 축별 항목이 많아야 한다. → v0.64 콘텐츠 웨이브.
2. **가독성(연출)** — 굴린 세계를 보고 느껴야 한다. → 블록 #4 리빌.
3. **유의미한 차이(전략이 바뀜)** — 세계가 숫자만 바꾸는 게 아니라 플레이 방식을 바꿔야 한다. → 태그 파생 엔진(결정 A) + 더 깊은 효과.
4. **런 간 진행(메타)** — 플레이할수록 새 도시·세계·창업자가 열린다. meta-progression이 이미 있으니 신규 콘텐츠를 언락 사다리에 엮는다.
5. **엔딩** — 세계·창업자·경로에 따라 승리 조건이 달라진다. → 멀티 엔딩(결정 B).

- **v0.67 — 멀티 엔딩 (결정 B)**. 세계/창업자/성장 경로에 따라 승리 조건과 엔딩이 갈린다. campaign 완주 판정(run-simulator의 success/integrity/finale)을 다중 엔딩으로 확장. 10년 완주 게이트는 유지하되 엔딩 분기를 얹는다. xhigh.

## 설계 스케치 (핸드오프 작성용 메모)

### 난이도·도전 축 (v0.65)
- run_modifiers.json에 `challenge_tiers` 또는 각 세계관에 `difficulty_weight` 추가.
- 굴릴 때 강도를 시드로 정하거나 플레이어가 선택(로그라이크는 보통 선택형 — "이 시드 도전?").
- tick 효과는 기존 `getRunModifierMonthlyEffects` 결과에 강도 배수를 곱하는 식(새 시스템 없이 스케일). 표준/쉬움은 1.0 이하라 회귀 없음.
- 보상은 meta-progression 언락 포인트에 배수. "하드 세계 클리어 = 새 도시 언락" 식 동기.
- 검증 — 표준 런 10년 완주 불변, 하드 티어도 (이론상) 완주 가능한지 run-simulator로 확인.

### 태그 파생 엔진 (v0.66, 결정 A)
- 입력 — 런의 활성 태그 집합(4축 + 보유 산업/제품 태그).
- 규칙 — `derivation_rules.json`: `{ requires: [tagA, tagB], yields: { unlock|event|bonus|product } , discovery_id }`.
- 발동 — 조건 충족 시 "발견!" 연출(v0.62 PayoffCelebration/discoveredPayoffIds 재사용) + 효과 적용.
- 결정론 — 시드/상태 파생, RNG 금지(하네스 안정).
- 분해 — #1 규칙 데이터+순수 평가 함수(derive-only), #2 발견 연출, #3 창발 제품/이벤트 연결, #4 밸런스.
- 이게 "무궁무진한 조합"의 엔진 — 9,500 시작 조합 × 파생 규칙 = 사실상 무한.

### 멀티 엔딩 (v0.67, 결정 B)
- 엔딩 정의 — `endings.json`: 조건(세계/창업자/지표 임계/경로) + 엔딩 텍스트/연출.
- 판정 — 기존 campaign finale 시점에 가장 잘 맞는 엔딩 선택(결정론).
- 10년 완주 게이트 유지, 엔딩 분기는 그 위에 얹음.
- 동기 — "이 엔딩 보려고 저 창업자/세계로 또 한 판."

## §5 입장

ROADMAP §5는 "새 대형 시스템 금지"였으나, 로그라이크 슬라이스는 사용자 결정으로 통제된 예외다(v0.60 산업 슬라이스 선례와 동일). 각 블록은 derive-only/additive로 시작하고, tick/세이브 변경은 최소·문서화·게이트 검증한다. 콘텐츠 웨이브는 순수 데이터라 가장 안전하고 부피가 크다 — 우선순위 높음.

## 진행 원칙

1. Codex는 항상 다음 블록을 가지고 있어야 한다(idle 0). Claude는 한 블록을 검증·커밋하는 동안 다음 핸드오프를 미리 써둔다.
2. 블록당 게이트 통과 + §5-안전(diff 확인) 후에만 커밋. 트렁크 기반.
3. 데이터 전용 블록은 fast로 싸게 대량 — 부피는 여기서 나온다.
4. 정확성 치명(tick/세이브/밸런스/엔딩)은 xhigh.
