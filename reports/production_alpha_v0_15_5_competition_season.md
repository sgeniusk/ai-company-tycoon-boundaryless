# v0.15.5-alpha 제작 보고서 — 경쟁사 시즌 브리프

작성일: 2026-05-17

## 목표

이번 단계의 목표는 경쟁사 등장을 단순 타임라인 로그가 아니라 “올해의 시장 시즌”으로 인식하게 만드는 것이다. 플레이어가 10년 캠페인을 진행하면서 매년 어떤 경쟁사가 들어왔고, 다음 파동은 언제 오며, 현재 가장 큰 압박이 누구인지 바로 파악할 수 있어야 한다.

## 구현 내용

- `getCompetitionSeasonBrief`를 추가했다.
- 현재 연차 기준으로 올해 등장한 경쟁사, 다음 예정 경쟁사, 최대 시장 압박 경쟁사를 계산한다.
- 회사 현황 메뉴에 `시장 시즌` 요약 패널을 추가했다.
- 경쟁 메뉴 상단에 `경쟁 시즌` 브리프를 추가했다.
- 경쟁 메뉴의 신규 경쟁사 예고 영역에 올해 등장한 경쟁사 목록도 함께 보여준다.

## 플레이 효과

- 12개월차에 오토노바 모터스, 브루체인 같은 강한 신규 경쟁자가 “올해 등장”으로 묶여 보인다.
- 24개월차 토이클라우드, 아이언오라클 같은 다음 파동이 미리 예고된다.
- 최대 압박 경쟁사가 시장 점유율과 최근 행동으로 표시되어 대응 목표가 뚜렷해진다.

## 하네스 에이전트 검토

### Executive Producer Agent

- P0/P1 없음.
- 경쟁사 시즌 브리프가 10년 캠페인의 장기 목표성을 강화한다.

### Game Designer Agent

- P0/P1 없음.
- 매년 새 경쟁사가 등장한다는 사용자의 요구가 더 게임적인 정보 구조로 연결됐다.
- 다음 단계에서는 시즌 브리프가 실제 보상/위험 이벤트와 연결되어야 한다.

### Systems Architect Agent

- P0/P1 없음.
- 경쟁 시즌 계산은 `competition-signals.ts`에 격리했고, 기존 경쟁사 데이터의 `entry_month`와 `rival_tier`를 재사용한다.

### UX Agent

- P0/P1 없음.
- 경쟁 메뉴 진입 즉시 올해 시장 상황을 볼 수 있다.
- P2: 10년 압축 QA 화면에서는 중앙 보조 패널의 정보량이 여전히 높다. v0.16의 한 화면 UI 정리에서 접기/압축 처리가 필요하다.

### Retention / LTV Agent

- P0/P1 없음.
- “다음 경쟁사 파동 전까지 준비하자”는 단기 복귀 목표가 생겼다.

### Shareability Agent

- P0/P1 없음.
- “로봇 카페 체인 경쟁사가 들어왔다” 같은 스크린샷용 사건성이 더 잘 드러난다.

## 다음 과제

- 시즌 브리프를 연간 심사 결과와 연결해 경쟁사 시즌 보상/패널티를 만든다.
- 신규 경쟁사 등장 시 대응 카드 보너스나 긴급 과제를 제공한다.
- v0.16에서 메인 화면 정보 밀도와 패널 접기 구조를 정리한다.
