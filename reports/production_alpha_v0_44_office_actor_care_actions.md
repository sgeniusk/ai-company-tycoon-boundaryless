# v0.44-alpha 제작 보고서 — 사무실 액터 직접 케어 액션

작성일: 2026-05-19

## 목표

사무실 액터 포커스 패널이 단순히 에이전트 메뉴로 이동하는 버튼에 그치지 않고, 직원 케어 액션을 바로 실행하는 운영 표면이 되게 한다.

## 구현

- `OfficeActorFocusPanel`에 `gameState`와 `onCareAction`을 전달해 현재 액터의 케어 가능 여부를 계산한다.
- 휴식 가능 액터는 `즉시 휴식` 버튼과 비용을 표시하고 `restAgent()`를 직접 호출한다.
- 협상 가능 액터는 `연봉 협상` 버튼과 비용을 표시하고 `negotiateAgentSalary()`를 직접 호출한다.
- 직접 케어 버튼은 기존 인사 시스템의 체크, 비용, 상태 변경, 타임라인 기록을 재사용한다.
- `office-visuals` QA 시나리오에 휴식 가능한 저체력 액터와 협상 가능한 저충성도 액터를 함께 배치했다.

## 에이전트 리뷰

- Executive Producer: P0/P1 없음. v0.42에서 남은 “클릭 후 실제 액션” 공백을 줄였다.
- Game Designer: P1 없음. 사람 관리가 메뉴 이동이 아니라 즉시 운영 선택으로 느껴진다.
- Systems Architect: P1 없음. 새 상태 로직을 만들지 않고 기존 `restAgent`/`negotiateAgentSalary`를 재사용했다.
- QA Agent: P1 없음. 직접 케어 버튼 노출과 QA 시나리오 조건을 테스트로 잠갔다.
- Balance Agent: P1 없음. 비용/효과는 기존 밸런스 함수를 그대로 사용한다.
- UX Agent: P1 없음. 포커스 패널 안에서 케어 비용과 실행이 한 번에 보인다.
- Shareability Agent: P2. 케어 결과 말풍선/플래시는 아직 없다.
- Solo Dev Scope Agent: P1 없음. 작고 재사용 중심의 UI 연결 작업이다.

## 다음 작업

1. 작업 중 액터의 프로젝트 배치 해제/변경을 포커스 패널에서 바로 실행한다.
2. 케어 액션 후 짧은 사무실 말풍선이나 플래시 연출을 추가한다.
3. 카드 사용/제품 출시/경쟁사 사건 연출을 같은 사무실 이벤트 레이어로 묶는다.
