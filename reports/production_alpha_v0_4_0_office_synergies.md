# Alpha v0.4.0 사무실 시너지 제작 보고서

작성일: 2026-05-17  
범위: `v0.4-alpha — 회사 운영과 공간 성장` 1차  
목표: 사무실 확장과 꾸미기가 단순 보유/배치가 아니라 운영 전략과 월간 보너스로 느껴지게 만든다.

## 구현 요약

- `data/office_synergies.json`를 추가해 사무실 장식 조합을 데이터로 분리했다.
- `getOfficeSynergySummary`를 추가해 현재 배치 장식 기준의 활성 시너지, 다음 후보, 월간 효과 합계를 계산한다.
- 월간 진행 시 활성 사무실 시너지 효과가 `strategyEffects`에 합산되도록 했다.
- 월간 타임라인에 `사무실 시너지` 로그가 표시된다.
- 상점 상단에 현재 사무실, 고용 한도, 장식 슬롯, 시너지 상태를 바로 보이게 했다.
- 인벤토리/투자 패널에 활성 시너지와 다음 후보를 보여 주는 `장식 조합` 영역을 추가했다.
- 데이터 검증 하네스가 사무실 시너지의 필수 필드, 요구 효과, 월간 효과를 검사한다.

## 추가된 시너지

- `차고 연산 클러스터`: 초반 GPU, 냉각, 데이터 작업대 조합으로 매월 연산과 데이터 보너스.
- `런칭 중계 코너`: 마케팅/사무실 장식 조합으로 화제성과 이용자 보너스.
- `신뢰 운영 벽`: 안전/신뢰 장식 조합으로 신뢰와 데이터 보너스.
- `에이전트 오케스트라 룸`: 자동화/에이전트 운영 장식 조합으로 자동화와 연산 보너스.

## 에이전트 리뷰

### Executive Producer Agent

- P0/P1 없음.
- 0.4의 첫 목표인 “공간 성장이 운영 선택으로 느껴지는가”에 대한 기반이 생겼다.

### Game Designer Agent

- P1 없음.
- 장식 조합이 월간 보너스로 이어져 꾸미기의 전략성이 올라갔다.
- 다음에는 확장 단계별 고유 선택지와 고용 후보 풀 변화를 더 붙이면 좋다.

### Systems Architect Agent

- P0/P1 없음.
- 시너지 정의는 JSON 데이터로 분리되어 있으며, 검증 하네스에도 연결됐다.
- 월간 효과는 기존 `strategyEffects` 경로에 합산되어 별도 상태를 늘리지 않았다.

### QA Agent

- P0/P1 없음.
- 단위 테스트, 데이터 검증, 빌드, 브라우저 렌더링을 통과했다.
- 남은 P2: 시너지 진행 라벨은 내부 effect/category 키가 섞여 있어 다음 UI 다듬기에서 한국어 라벨로 바꾸는 것이 좋다.

### Balance Agent

- P1 없음.
- 보너스는 월간 소량 자원 보정이라 초반 파산 회복과 성장 보조에 유효하지만 즉시 압도적이지 않다.
- 장기 10년 시뮬레이션에서 시너지 다중 활성의 누적 강도를 추적해야 한다.

### UX Agent

- P1 없음.
- 상점 메뉴 첫 화면에 사무실 상태가 올라와 “상점=회사 공간 관리” 의미가 더 빨리 전달된다.
- 우측 메뉴 전체 밀도는 계속 관리가 필요하다.

## 판정

`v0.4.0-alpha`는 완료로 본다. 사무실은 이제 고용 한도와 장식 슬롯뿐 아니라, 장식 조합을 통해 월간 운영 보너스를 만드는 전략 공간이 됐다.
