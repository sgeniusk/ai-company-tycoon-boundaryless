# v0.31-alpha 제작 보고서 — 덱 시너지와 빌드 보너스

작성일: 2026-05-17

## 목표

이번 단계의 목표는 카드가 “한 번 쓰고 끝나는 즉시 보너스”가 아니라, 이번 런의 회사 성격을 결정하는 빌드 도구처럼 느껴지게 만드는 것이다. 카드 태그가 충분히 모이면 덱 시너지가 켜지고, 월간 운영과 카드 사용 효과에 작지만 읽기 쉬운 보너스를 주도록 구현했다.

## 구현 요약

- `data/deck_synergies.json` 추가
  - 런칭 머신
  - 신뢰 하네스
  - 자동화 복리
  - 연구 플라이휠
  - 라이벌 대응 셀
- `DeckSynergyDefinition`, `DeckSynergyStatus`, `DeckSynergySummary` 타입 추가.
- `getDeckSynergySummary()` 추가.
  - 덱 전체 카드 태그를 집계한다.
  - 강화 카드와 보상으로 얻은 카드는 가중치를 더 받는다.
  - 활성 시너지, 다음 후보, 부족 태그, 월간 효과 합계를 계산한다.
- `getDeckSynergyMonthlyEffects()`를 월간 전략 효과에 연결했다.
- 활성 시너지의 `play_effect_tags`와 맞는 카드는 긍정 효과가 소폭 강화된다.
- 덱 메뉴에 시너지 패널을 추가했다.
  - 활성 개수
  - 월간 효과
  - 카드 보너스 배율
  - 다음 후보 진행률
  - 부족 태그
  - 위험 문구
- `deck-synergy` QA 시나리오를 추가했다.

## 플레이 변화

- 플레이어는 카드 보상 선택 때 “좋은 카드 한 장”보다 “내 덱 태그가 어떤 빌드를 향하는지”를 보게 된다.
- 런칭 카드가 모이면 `런칭 머신`이 켜져 화제성과 이용자 증가가 붙는다.
- 신뢰/안전 카드가 모이면 장기 신뢰 기반 빌드가 보인다.
- 자동화/연구/경쟁 대응도 각각 독립된 덱 방향으로 읽힌다.
- 강화 카드와 출시 보상 카드가 태그 가중치를 더 가져, 런마다 대표 카드가 생긴다.

## 하네스 에이전트 리뷰

### Executive Producer Agent

- 판정: 통과
- 사유: v0.31 목표였던 “덱빌딩 조합 심화”가 한 화면에서 읽히는 플레이 변화로 구현됐다.

### Game Designer Agent

- 판정: 통과
- 사유: 카드 선택이 다음 제품 개발뿐 아니라 월간 운영 방향까지 바꾸게 됐다.

### Systems Architect Agent

- 판정: 통과
- 사유: 시너지 조건과 수치는 JSON 데이터에 있고, 계산은 순수 함수로 분리됐다.

### QA Agent

- 판정: 통과
- 확인: 덱 시너지 계산, 카드 효과 보너스, QA 시나리오, UI 계약, 전체 하네스 통과.

### Balance Agent

- 판정: 조건부 통과
- 사유: 보너스 배율은 1.08~1.12로 낮게 잡았다. 장기 시뮬레이션에서 특정 빌드 독주 여부를 더 봐야 한다.

### UX Agent

- 판정: 조건부 통과
- 사유: 데스크톱 덱 화면에서는 명확하다. 모바일에서는 덱 상세가 내부 스크롤에 들어가므로 후속 압축이 필요하다.

### Retention / LTV Agent

- 판정: 통과
- 사유: 같은 제품을 만들더라도 카드 보상 선택에 따라 다음 런의 회사 색이 달라질 이유가 생겼다.

### Shareability Agent

- 판정: 통과
- 사유: `런칭 머신 활성`, `신뢰 하네스 활성`처럼 스크린샷으로 설명되는 빌드명이 생겼다.

### Solo Dev Scope Agent

- 판정: 통과
- 사유: 새 그래픽 없이 데이터와 UI 패널만으로 반복 플레이 깊이를 올렸다.

### Deck System Engineer

- 판정: 통과
- 사유: 카드 태그, 보상, 강화, 월간 효과가 하나의 덱빌딩 레이어로 묶였다.

### Roguelite Meta Engineer

- 판정: 통과
- 사유: 런마다 대표 태그와 시너지 방향이 달라져 재시작 동기가 강화된다.

### Balance Simulation Engineer

- 판정: 조건부 통과
- 사유: 무한 루프는 없지만 월간 효과가 누적되므로 v0.32에서 장기 빌드별 수익 편차를 측정해야 한다.

## 백로그

- P1: 카드 제거/강화 선택 빈도와 비용을 덱 시너지 목표에 맞게 재조정한다.
- P1: 모바일 덱 메뉴에서 시너지 패널을 더 빨리 보이게 압축한다.
- P1: 10년 시뮬레이션에 빌드별 독주/사장 전략 검사를 추가한다.
- P2: 시너지 활성 순간에 미나가 짧게 리액션하는 튜토리얼을 추가한다.
- P2: 카드 보상 후보에 “이 카드를 고르면 켜지는 시너지” 미리보기를 표시한다.
